# 总览

> 从《项目主 GDD（初版）》拆分而来，便于按专题维护。

## 1. 项目概述

### 1.1 暂定名称
- 项目代号：EntryMap
- 对外包装名暂定：`最后的防线`
- 平台展示名建议：`最后的防线：单英雄守线刷宝`
- 类型定位：单英雄自动战斗 + 波次刷怪 + 局内构筑 + 局外成长的 RPG 肉鸽地图

### 1.2 一句话描述
玩家操控一名驻守在左侧战线的英雄，对抗从右侧持续刷新的敌人。英雄以自动攻击为基础，玩家通过技能选择、装备成长、词条构筑和局外养成，逐步完成一局高压但可滚雪球的商业化 RPG 肉鸽体验。

### 1.3 核心卖点
- 上手门槛低：英雄自动攻击，玩家主要做构筑和决策。
- 成长反馈强：每几分钟都能看到明显战力提升。
- 局内随机性高：技能、装备、羁绊与波次奖励共同构成肉鸽体验。
- 可长期运营：天然适合扩展英雄、流派、敌人、章节与数值池。


## 2. 目标玩家

### 2.1 核心用户
- 喜欢魔兽 RPG、刷刷刷、数值成长、挂机战斗的玩家。
- 喜欢肉鸽构筑，但不追求高强度操作的玩家。
- 喜欢反复开局、测试流派、挑战高压规则和追求成长速度的玩家。

### 2.2 体验目标
- 前 1 分钟理解玩法。
- 前 3 分钟第一次形成“build 成型”的快感。
- 前 10 分钟进入“要不要贪下一轮”的风险博弈。
- 每局结束后都能获得明确的局外收益与下一局动机。


## 3. 设计支柱

### 3.1 决策大于操作
操作输入尽量轻，重点放在“选什么”“升什么”“保什么”“赌不赌”。

### 3.2 成长必须可感知
每次升级、装备替换、羁绊达成、稀有词条触发，都要直接改变输出表现、清怪效率或生存方式。

### 3.3 随机但可控
不能只是纯随机掉落，应该通过保底、刷新、锁定、定向奖励等机制，让玩家能够向目标流派靠拢。

### 3.4 失败也有积累
即便中途暴毙，玩家也能获得资源、解锁或图鉴进度，不会觉得白打一局。


## 4. 核心玩法循环

### 4.1 单局核心循环
1. 开局直接进入战斗，使用当前默认初始英雄。
2. 英雄驻守左侧战线，自动攻击靠近的敌人。
3. 标准模式固定进行 `5 波`，每波怪物阶段持续 `240 秒`，每波只配置 `1 种主怪` 与 `1 名专属 Boss`。
4. 击杀敌人后直接获得经验、金币、木材等奖励，并通过 UI 反馈即时表现入账结果。
5. 玩家在关键节点主动点击对应按钮，使用技能点打开攻击技能强化面板，并消耗木材打开羁绊抽卡面板。
6. 每波怪物阶段结束后进入该波 Boss 战，击败第 `5 波 Boss` 后本局胜利。
7. 单局结束后结算局外资源；通关标准模式后可进入挑战模式，完成挑战获得写入局外存档的额外奖励。

### 4.2 单局时长
- 标准模式固定 `5 波`，每波怪物阶段 `240 秒`，基础战斗时长固定为 `1200 秒`，即 `20 分钟`。
- 每波结束后立刻进入 Boss 战，不再采用 15 到 20 波的长局结构。
- 挑战模式沿用 `5 波通关` 框架，通过挑战规则提高难度与奖励，不追加无限波次。
