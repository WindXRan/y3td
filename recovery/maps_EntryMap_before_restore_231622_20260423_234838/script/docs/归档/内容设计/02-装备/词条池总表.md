# 词条池总表

> 本文用于约束装备词条池结构，不填写具体数值。

## 1. 词条结构

- 装备词条固定分为：
  - `主词条`
  - `副词条`
- 两者都带随机性，但 `主词条` 应更决定方向，`副词条` 应更决定质量。

## 2. 主词条设计原则

- 主词条用于定义装备的大方向，例如：
  - `输出`
  - `生存`
  - `功能`
  - `流派偏好`
- 主词条应按部位限制，不建议所有主词条全局通吃。
- 主词条应优先承载“大方向”，避免副词条反客为主。

## 3. 副词条设计原则

- 副词条用于区分同部位、同品质装备之间的细节差异。
- 副词条应包含：
  - `通用输出`
  - `生存`
  - `资源`
  - `职业 / 流派适配`
- 副词条可随机到：
  - `词条类型`
  - `词条数值档位`

## 4. 建议词条分类

### 4.1 通用输出类
- `攻击`
- `攻速`
- `暴击`
- `暴伤`
- `技能伤害`
- `普攻伤害`

### 4.2 防御生存类
- `生命`
- `护甲`
- `减伤`
- `吸血`
- `生命恢复`

### 4.3 功能节奏类
- `冷却`
- `资源回复`
- `命中`
- `攻击范围`
- `效果持续`

### 4.4 流派适配类
- `持续伤害`
- `召唤强化`
- `受击收益`
- `触发收益`
- `状态切换收益`

## 5. 词条池边界

- 不要让单一词条成为所有职业的绝对最优解。
- 不要让某些副词条的价值高到完全超过主词条方向。
- 流派词条应增强“适配价值”，而不是变成所有职业必抢词条。
