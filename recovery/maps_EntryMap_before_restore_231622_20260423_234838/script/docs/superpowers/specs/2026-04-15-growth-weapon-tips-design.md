# 成长武器 Tips 设计

日期：2026-04-15

## 背景

当前成长武器已经有运行时等级状态，也已经能在开局发放到英雄物品栏，但缺少统一的提示信息展示。玩家无法在 HUD 成长武器格或物品栏内直观看到：

- 当前成长武器等级
- 升级所需金币
- 当前直接属性增幅
- 已获得词缀

这次需求要求：

- HUD 成长武器格和英雄物品栏中的成长武器物品都要显示 tips
- 两处展示的内容保持一致
- “当前属性增幅”只显示当前已有的直接属性，不显示下一级预告

## 目标

新增一套成长武器 tips 数据模型，并接入两个入口：

1. 底部 HUD 的成长武器格鼠标移入
2. 英雄物品栏中成长武器物品鼠标移入

两处共用同一份 runtime payload，避免内容漂移。

## 非目标

本次不包含以下内容：

- 不展示“下一级会增加什么”
- 不展示未来词缀候选或词缀刷新
- 不修改成长武器升级规则或金币消耗规则
- 不新增独立背包页或成长武器专属面板
- 不改现有成长武器物编数值，仅做展示接入

## 方案选择

采用“单一 payload builder，多入口复用”的方案。

理由：

- HUD 和物品栏都需要同一份成长武器信息
- 成长武器的等级、词缀和升级费用属于 runtime 状态，不应分别在两处 UI 里各自拼装
- 后续如果成长武器规则变化，只需要改一处构建逻辑

不采用“各入口各自拼文本”的原因是维护成本高，后续极易出现文案或数值不一致。

## 设计概览

### 1. 新增成长武器 tips payload builder

在 runtime 层新增一个构建函数，输入为当前 `STATE` 和成长武器槽位，输出统一的成长武器 tips payload。

payload 至少包含以下字段：

- `title_text`
  - 成长武器名称
- `subtitle_text`
  - `Lv.X`
- `cost_text`
  - 下一次升级所需金币
- `attr_lines`
  - 当前直接属性增幅列表
- `affix_lines`
  - 已获得词缀列表
- `item_key`
  - 当前成长武器物编 ID
- `slot`
  - 当前装备槽，首版固定为 `weapon`

### 2. 数据来源

#### 2.1 成长武器 runtime

从 `gear_state.items.weapon` 读取：

- 当前等级 `level`
- 已获得词缀 `affixes`
- 当前物品 `item_key`

从 `gear_upgrade_config` 读取：

- 成长武器显示名称
- 最大等级
- 下一次升级费用计算所需配置

#### 2.2 升级所需金币

继续复用 `runtime.gear_upgrades` 现有升级费用逻辑：

- 使用当前等级调用 `get_upgrade_cost('weapon', item.level, config)`
- 已满级时显示 `已满级`

#### 2.3 当前直接属性增幅

从当前成长武器 `item_key` 对应的编辑器物品数据读取“物编附加属性”。

展示范围限定为这件物品当前已经提供的直接属性，例如：

- `attached_attack_phy`
- `attached_attack_speed`
- `attached_critical_chance`
- `attached_hp_max`

读取原则：

- 只展示当前物品本身已经存在的直接属性
- 忽略主动技能说明、被动长描述、未来成长预告
- 忽略值为 0、空值或未配置字段
- 用统一的人类可读名称格式化为文本列表

如果当前物品没有可展示的直接属性，则显示：

- `当前无直接属性增幅`

#### 2.4 已获得词缀

从 `gear_state.items.weapon.affixes` 读取，按获得顺序展示：

- 优先显示 `display_name`
- 如存在 `summary`，展示为附加说明

若没有词缀，则显示：

- `暂无词缀`

## 展示结构

两处入口内容保持一致，展示信息分为 4 段：

1. 标题区
   - 成长武器名
   - 当前等级
2. 升级区
   - 升级所需金币
3. 属性区
   - 当前直接属性增幅
4. 词缀区
   - 当前已获得词缀

推荐文案结构：

- 标题：`洪荒之刃`
- 副标题：`成长武器 Lv.1`
- 升级信息：`升级所需：100 金币`
- 属性标题：`当前属性增幅`
- 词缀标题：`当前词缀`

## 接入点

### 1. HUD 成长武器格

在底部 HUD 新增成长武器格节点绑定逻辑。

行为：

- 鼠标移入时显示成长武器 tips
- 鼠标移出时隐藏 tips
- 左键点击暂不额外打开新面板，本次只负责 tips

如果当前成长武器不存在 payload，则隐藏 tips，不报错。

### 2. 物品栏成长武器物品

在英雄物品栏中识别成长武器 `item_key`。

行为：

- 鼠标移入该物品时，优先展示成长武器 runtime tips
- 不再只依赖物编原始说明作为唯一显示内容

如果鼠标移入的不是成长武器物品，则保持原有物品说明逻辑不变。

## 运行时边界

为避免耦合过深，建议保持以下分层：

- `runtime.gear_upgrades`
  - 负责成长武器运行时状态、升级费用、payload 构建
- `ui.runtime_hud`
  - 负责 HUD 成长武器格 hover 事件与展示
- 物品说明入口
  - 负责识别当前 hovered item 是否为成长武器，并请求 payload

UI 层不自行解析成长武器等级、词缀或属性字段。

## 错误处理

以下情况按兜底方式处理：

- 找不到成长武器 runtime
  - 返回 `nil`，UI 不展示 tips
- 找不到物品 `item_key`
  - 名称退回到 `成长武器`
  - 属性区显示 `当前无直接属性增幅`
- 找不到编辑器物品属性字段
  - 仅展示等级、金币和词缀
- 已满级
  - 升级区显示 `已满级`

## 测试策略

采用最小必要 smoke / static 测试：

1. payload builder 测试
   - 能构造成长武器名称、等级、升级费用、词缀
2. 属性展示测试
   - 只输出当前直接属性增幅
   - 不输出未来升级预告文本
3. HUD 接入测试
   - HUD 可读取成长武器 payload
4. 物品栏接入测试
   - 识别成长武器 `item_key` 时走成长武器 tips 分支

## 实施顺序

1. 在 `runtime.gear_upgrades` 中新增成长武器 tips payload builder
2. 增加编辑器物品直接属性解析与格式化
3. 为 payload builder 添加失败测试并跑红
4. 接入 HUD 成长武器格 hover
5. 接入物品栏成长武器物品 hover
6. 完成 smoke 验证

## 风险与注意事项

### 1. 物品栏 hover 入口可能不在当前脚本主链中

如果当前物品说明是编辑器侧直接驱动，而不是 Lua runtime 驱动，则需要额外查找物品 hover 回调入口。此时优先保持 HUD 入口先可用，再补物品栏接入。

### 2. 编辑器物品属性字段很多，需限制首版范围

首版只格式化“明确的一次性直接属性字段”，不解析复杂被动文本，也不做技能说明重构。

### 3. 当前 HUD 里未必已有成长武器格节点

如果现有 `GameHUD` 已包含成长武器格节点，则直接绑定。
如果没有，则本次需要在 HUD runtime 创建一个最小成长武器格入口，但仍只做 tips，不扩展更多交互。

## 完成标准

满足以下条件即可视为完成：

- HUD 成长武器格移入可看到成长武器 tips
- 物品栏成长武器物品移入可看到同源 tips
- tips 同时包含：
  - 升级所需金币
  - 当前直接属性增幅
  - 当前词缀
- 两处展示内容来自同一份 runtime payload
