# 羁绊系统策划案

## 文档定位

- 本文以当前项目实现为准，事实源优先级为 `data_csv -> data/object_tables -> runtime -> UI`。
- 目标是把当前版本已经落地的 `F` 羁绊系统整理成可供策划、程序、调数共同使用的专题文档。
- 旧目录中的羁绊方案只作为历史参考；若与本文冲突，以当前代码和 CSV 为准。

## 系统定位

- 羁绊是战斗内第二条核心构筑轴，对应热键 `F`。
- 它承担三件事：
  - 提供稳定的属性成长与流派定向。
  - 通过链式解锁把“根链完成 -> 分支成型 -> 上位路线定型”串成一条长期成长线。
  - 把路线标签、自动效果、奖励修正和 `G` 强化 / 宝物权重联动起来。

## 当前版本玩家流程

1. 玩家在局内消耗 `100` 木材进行一次 `F` 抽卡，系统给出 `3` 个候选。
2. 当前配置下，玩家直接从各大类的当前阶段节点开始抽，不经过单独的“分组开启卡”；开局阶段对应各大类的根链整包。
3. 根链阶段会一次性投放该大类的基础链卡牌，玩家先补齐根链，再进入下一阶段。
4. 当某个大类的当前阶段被补齐后，系统会切到该大类的下一阶段整包；更深阶段继续保持锁定。
5. 若再次按 `F` 时仍有未完成的羁绊选择，则直接回到当前轮次，不会重新扣费。
6. 当前轮次支持刷新，费用为 `40 / 80 / 100 / 100...` 木材；当前没有免费刷新。
7. HUD 固定展示 `7` 个羁绊槽位；新节点会进入槽位栏，已完成并被吞噬的根套装会自动退槽。

## 核心规则

| 规则项 | 当前实现 |
| --- | --- |
| 候选数量 | 每次固定 `3` 选 `1` |
| 根节点开放方式 | `bond_pick_rules.csv` 当前配置 `include_group_choices=0`，所以各大类 `tier=1` 的根链整包直接进入候选池 |
| 阶段开放规则 | 代码当前按每条根链的“当前 active tier”整包开放；根链完成前只投放 `tier=1`，完成后切到 `tier=2`，更深 `tier` 继续锁定 |
| 分支开放规则 | 体术/经济/法术/箭术/成长会在根套装完成后开放 `tier=2` 整包；暴击线仍只有单线递进，但也遵守当前阶段整包规则 |
| 候选优先级 | 当前没有“上一次解锁节点的直接后继优先”；候选直接从当前可解锁池随机抽取 |
| 权重规则 | `bond_pick_weights.csv` 当前只有 `node=1 / group=1`，候选基础权重等同 |
| 解锁前置 | 跨阶段仍受当前 active tier 限制；同一 `tier` 内不再要求严格按 `parent_id` 顺序逐张解锁 |
| 根套装完成 | 体术/经济/法术/箭术 `3` 张，暴击 `2` 张，成长 `4` 张 |
| 根套装完成后的处理 | 当前全部采用 `consume_all`：吞噬根链节点、保留已获得的基础/套装收益、释放槽位 |
| 槽位压力处理 | 当前没有“满位自动吞旧卡”或“旧卡替换”逻辑，主要依靠根套装吞噬释放空间 |
| 节点即时奖励 | 节点可配置 `unlock_gold / unlock_wood / unlock_exp`，解锁时立即发放 |

## 链式结构设计

- 羁绊不是平铺卡池，而是按 `parent_id / next_ids` 构成有向树。
- 根链负责定义大类的基础收益与“何时开启分支”。
- 分支节点负责把路线推进到更明确的构筑方向。
- `route_tags` 同时服务于三类系统：
  - 构筑标签：参与攻击技能强化与宝物的路线匹配。
  - 自动效果标签：驱动自动触发效果。
  - 历史兼容标签：已直接并入 `bond_nodes.csv` 的正式 `route_tags`。

### 根链与自动效果的关系

- 根节点自身可以带 `auto_*` 标签，但这类标签不会在“只拿到根节点”时立即生效。
- 当前实现会在根套装完成后，才把根节点上的 `auto_*` 标签纳入生效标签池。
- 分支节点上的普通标签和非根节点标签则在解锁后即时生效。

### 改表口径

- 想改“单张卡给什么”，优先改 `bond_node_attr.csv / bond_node_runtime.csv`。
- 想改“根套装基础效果和成套效果”，优先改 `bond_root_set_*`。
- 想改“F 抽一次多少钱、能刷几次、根节点是否直接开放”，改 `bond_draw_rules.csv / bond_pick_rules.csv / bond_refresh_costs.csv`。
- 想改“自动触发技能”，优先改 `auto_active_effects.csv`，而不是直接改羁绊模板文件。

## 生效层与系统联动

| 联动层 | 当前行为 |
| --- | --- |
| 英雄静态属性 | 节点与根套装可直接提供攻击、力量、敏捷、智力、生命、暴击、暴伤等常驻数值 |
| 每秒成长 / 击杀成长 | `bonds_chain.lua` 在 update 与 kill 回调里处理每秒属性、杀敌属性、杀敌攻击等收益 |

### 自动效果现状

当前由羁绊 route tag 驱动的自动效果有：

- `auto_spell_burst`：法术根套装完成后解锁魔爆术。
- `auto_spell_burst_amp`：魔术分支强化魔爆术次数、范围与冷却。
- `auto_haste_reset`：急速分支给技能冷却重置概率。
- `auto_fighting_spirit`：体术根套装完成后解锁斗气场域。
- `auto_rapid_overdrive`：速攻分支提供攻速爆发。
- `auto_blood_demon_burst`：血魔分支按失血阈值触发回荡。
- `auto_charge_breaker_rally`：陷阵分支按击杀计数触发短时爆发。

### 当前版本的内容特征

- 体术、法术、箭术都已具备“根套装自动效果 + 分支强化”的完整体验。
- 成长是当前节点数最多、路线最长的大类，承担属性流转与多职业终局定型。
- 经济节点里混合了“即时资源奖励”和“长期收益”两类设计。
- 暴击线当前是唯一的纯线性链路，没有横向分叉。

## 与旧方案相比的当前口径

- 当前主实现已切到 `bond_nodes.csv + bond_root_sets.csv + bonds_chain.lua`，不再以旧分散文档为事实源。
- 当前 `F` 抽卡直接抽真实节点，不展示 group placeholder 卡。
- 当前系统没有旧方案里“满位后自动吞掉旧羁绊卡”的逻辑。
- 当前模板文件主要负责节点激活/卸载；多数动态逻辑已经收敛到 `bonds_chain.lua` 和 `auto_active_effects.lua`。
- 当前代码已按“当前阶段整包投放”运行：根链整包先开放，根套装完成后再切到下一阶段整包。

## 当前已知问题与策划风险

1. 暴击线的 `line_id` 与根套装完成阈值存在耦合风险。
   `bond_critical_core` 当前要求 `2/2` 就完成根套装，但整条暴击路线仍共用 `line_critical_core`；如果后续继续强化吞噬逻辑，需要重点验证 `致命 / 大炮` 的后续可达性。

2. 当前候选池还没有额外的品质或流派倾向权重。
   这意味着即便已经切到“当前阶段整包投放”，同阶段节点之间仍接近等概率随机；若后续希望强化 build 定向，需要补充更细的 pick weight 规则。

3. 分组选项能力已存在，但默认关闭。
   若后续想改成“先选大类、再抽节点”的体验，可以直接启用 `include_group_choices`，但要同步检查新手引导、卡面文案和 HUD 提示。

## 结论

- 当前羁绊系统已经是完整可玩的链式构筑系统，而不是占位功能。
- 它的核心价值不是“额外属性卡池”，而是“用根链完成度控制路线开启，用 route tag 把构筑方向传递给战斗系统”。
- 后续若继续扩展，优先建议沿着三条线推进：
  - 统一根节点文案与根套装真实数值。
  - 补足候选权重与流派引导。
  - 逐类检查根链完成后的分支可达性与吞噬表现。
