# 战斗系统

> 从《项目主 GDD（初版）》拆分而来，便于按专题维护。

## 5. 战斗系统

### 5.1 基础战斗框架
- 英雄默认自动攻击最近目标或威胁最高目标。
- 敌人从右向左推进，接近战线后直接对英雄造成压力。
- 玩家主要通过局内构筑影响攻速、范围、穿透、暴击、技能释放频率、生存能力等。

#### 5.1.1 自动索敌与集火规则
- 首版战斗默认采用 `自动索敌 + 可手动集火修正` 的目标选择结构。
- 当玩家没有指定集火目标时，英雄按系统默认索敌逻辑行动，不需要玩家频繁点选目标。
- 当玩家指定了 `集火目标` 后，英雄的普攻与可指定目标的攻击技能优先对该目标进行输出。
- 文案统一使用 `集火`，不再混用 `挑怪`、`锁怪` 等别名。

默认自动索敌优先级：
- 首先攻击当前 `攻击范围` 内可命中的敌人。
- 在范围内目标中，优先级建议为：
  `Boss`
  `挑战目标 / 精英目标`
  `距离战线最近的敌人`
  `威胁最高的敌人`
- 若多个目标优先级相同，则优先选择 `距离英雄最近` 的目标。
- 若当前没有任何合法目标，则英雄保持待机并持续重新检测索敌范围。

集火生效规则：
- 玩家点击底部 `集火` 按钮后，进入 `选择集火目标` 状态。
- 在该状态下，玩家下一次点击敌方单位，会将其设置为当前 `集火目标`。
- 已设置集火目标时，`普攻`、`青木灵矢`、`赤炎箭`、`寒泉箭`、`乙木天雷` 等需要选定目标的攻击行为，优先以该目标作为主要目标。
- 范围型或区域型技能若本身需要选择落点，则优先以 `集火目标` 的当前位置作为施法参考点。
- 纯自身周围生效、纯随机扩散或完全无目标的效果，不因集火而改变结算方式。
- 当集火目标暂时不在攻击范围内时，英雄不会停手发呆，而是暂时按默认索敌逻辑攻击其他合法目标；只要原集火目标重新进入可攻击条件，系统再次优先对其输出。
- 当集火目标死亡、脱离战场、进入不可选中状态或玩家主动清除时，集火状态立即结束。

集火清除规则：
- 玩家再次点击 `集火` 按钮，可直接取消当前集火状态。
- 玩家按下快捷键 `F4` 时，也会立即清除当前集火目标。
- 若当前没有集火目标，`F4` 不产生额外效果。
- 首版不允许同时存在多个集火目标，也不做队伍级分配目标系统。

#### 5.1.2 击杀奖励与 UI 表现规则
- 首版普通资源采用 `怪物死亡后直接入账` 的方式处理，不生成需要玩家手动拾取的地面掉落物。
- 直接入账的基础奖励固定包括：
  `经验`
  `金币`
  `木材`
- 普通敌人死亡后，按规则直接发放少量 `经验` 与 `金币`。
- 精英、Boss、挑战怪物或特殊机制怪死亡后，可以额外直接发放 `木材`、高额 `金币`、高额 `经验` 或章节奖励。
- `宝物候选`、`词缀选择`、`烙印选择`、`羁绊候选` 这类需要玩家做决策的高价值奖励，仍然不直接写成数值入账，而是优先进入对应面板或 `待领取奖励` 流程。

直接入账规则：
- 怪物死亡时，系统立即完成奖励结算并同步刷新顶部资源栏、经验条与相关统计值。
- 若同一时刻有大量怪物连续死亡，允许系统将多次基础奖励在短时间窗口内做 `合并飘字` 或 `批量数字跳动`，避免满屏刷字影响阅读。
- 木材、挑战奖励、精英/Boss 奖励等较高价值收益，应尽量单独显示，不与普通小额金币/经验完全混在同一条提示里。

UI 表现规则：
- 普通敌人死亡后，至少要通过 `经验条增长`、`顶部资源栏数字跳动`、`角色附近或敌人死亡位置的奖励飘字` 三类反馈中的两类，让玩家明确感知自己获得了收益。
- 推荐使用：
  `经验 +X`
  `金币 +X`
  `木材 +X`
  这类短文本飘字或资源跳字，而不是生成可拾取实体。
- 精英、Boss、挑战怪物死亡后的高价值奖励，应额外附带更明显的描边、高亮颜色或短暂停留提示。
- 若某次击杀同时触发多种奖励来源，例如 `基础奖励 + 羁绊奖励 + 挑战奖励`，UI 可以合并展示为一组结算提示，但需要让玩家看得出这是一次高收益击杀。

防丢失规则：
- 因为首版基础资源采用直接入账，所以普通资源不存在“未拾取丢失”问题。
- 当战斗进入 `Boss 战`、完成 `挑战事件`、波次结算、胜利或失败时，所有已经触发但尚未播完的资源反馈只影响 UI 表现，不影响实际入账结果。
- 若因为奖励面板阻塞导致某些高价值奖励暂时无法立即展示，系统也必须先完成逻辑入账或写入 `待领取奖励`，不允许因为弹窗切换导致奖励丢失。

### 5.2 英雄基础属性
这里以后不再用抽象分类做主标准，而是直接以“实际属性面板”的分区作为设计标准。后续 GDD、UI、技能描述、装备词条、数值表和代码字段，都应尽量围绕这套面板语言展开。

属性面板当前分为 5 个主分区：
- 伤害属性
- 防守属性
- 资源属性
- 增幅属性
- 其他属性

#### 5.2.1 面板总原则
- 面板中出现的属性名称，就是玩家认知层面的标准术语。
- 同一个属性不要在设计、UI、脚本里使用多套别名。
- 优先保证“玩家能看懂”而不是“设计上最严谨”。
- 首版先做核心字段，后续再逐步开放边缘字段。

#### 5.2.2 伤害属性
这是角色输出能力的主面板，决定普攻、技能、对群、对精英、对 Boss 的整体表现。

基础攻速与射程类：
- `攻速`：面板显示的攻击速度百分比，决定出手快慢。
- `攻击间隔`：实际基础出手间隔，是攻速最终作用后的结果。
- `攻击范围`：角色的射程范围，决定接敌距离和清怪节奏。
- `命中`：攻击是否稳定命中的概率属性。

暴击与伤害倍率类：
- `物理暴击`
- `魔法暴击`
- `物理暴伤`
- `魔法暴伤`

伤害分类加成类：
- `物理伤害`
- `魔法伤害`
- `普攻伤害`
- `技能伤害`
- `所有伤害`
- `最终伤害`

目标类型加成类：
- `BOSS伤害`
- `精英伤害`
- `挑战伤害`

破防与穿透类：
- `护甲穿透`
- `无视护甲`
- `穿透次数`

投射物扩展类：
- `多重数量`
- `多重伤害`
- `弹射次数`
- `弹射伤害`

设计说明：
- `攻速` 和 `攻击间隔` 应同时存在，但其中一个是展示结果，一个是底层结算关键值。
- `普攻伤害` 与 `技能伤害` 必须严格区分，否则流派边界会变糊。
- `最终伤害` 应视为高价值乘区属性，不能轻易大量发放。
- `多重 / 弹射 / 穿透` 是塔防味最强的一组属性，应作为核心流派驱动器。

#### 5.2.3 防守属性
这是角色生存能力的主面板，决定能否在中后期站住。

- `格挡`：结算时减少固定值伤害。
- `闪避`：有概率规避受到的伤害。
- `生命恢复`：每秒恢复固定生命。
- `杀敌恢复`：击杀敌人时恢复生命。
- `伤害减免`：按比例降低受到的伤害。
- `控制时长`：提高附加控制时长，当前更适合作为攻防混合属性，但在面板中归入防守类展示也成立。

设计说明：
- `生命恢复` 负责稳定续航。
- `杀敌恢复` 负责滚雪球续航。
- `格挡` 和 `闪避` 适合战士型和高风险 build。
- `伤害减免` 是后期高价值防御属性，应控制来源。

#### 5.2.4 资源属性
这部分不是传统意义上的“战斗面板”，但它们直接影响成长节奏，因此必须进入角色面板。

- `杀敌经验`
- `杀敌金币`
- `杀敌木材`
- `杀敌加成`
- `每秒经验`
- `每秒金币`
- `每秒木材`
- `每秒杀敌`

设计说明：
- `杀敌` 系列强调“击杀收益”。
- `每秒` 系列强调“挂机收益”和稳定滚动成长。
- `杀敌加成` 是非常关键的成长资源属性，适合支撑“越杀越强”的肉鸽体验。

#### 5.2.5 增幅属性
这一分区是角色成长系统的核心。它不直接代表一次攻击有多疼，而是决定“白字属性如何被放大”“成长如何滚雪球”。

基础增幅类：
- `力量增幅`
- `敏捷增幅`
- `智力增幅`
- `攻击增幅`
- `生命增幅`
- `护甲增幅`

每秒成长类：
- `每秒力量`
- `每秒敏捷`
- `每秒智力`
- `每秒攻击`

击杀成长类：
- `杀敌力量`
- `杀敌敏捷`
- `杀敌智力`
- `杀敌攻击`
- `杀敌生命`
- `杀敌护甲`

最终成长类：
- `最终力量`
- `最终敏捷`
- `最终智力`
- `最终攻击`
- `最终生命`
- `最终护甲`

设计说明：
- `增幅` 类属性是放大器。
- `每秒` 类属性是时间型收益。
- `杀敌` 类属性是战斗型收益。
- `最终` 类属性是后置乘区。

这 4 组属性叠在一起，正好构成这类刷图 RPG 的成长主循环。

#### 5.2.6 其他属性
这一栏承载五行、流派、异常、回复和构筑辅助属性，是后期内容扩展的主要空间。

五行 / 流派伤害类：
- `金行伤害`
- `木行伤害`
- `水行伤害`
- `火行伤害`
- `土行伤害`
- 以及其他专属流派伤害字段

异常与派生伤害类：
- `燃烧伤害`
- `冻伤伤害`

恢复与功能类：
- `百分比恢复`
- `恢复效果`
- `卡牌增幅`

其他目标类：
- `挑战伤害`

设计说明：
- “其他属性”不是杂项垃圾桶，而是高阶 build 层。
- 首版如果资源不足，可以先只启用其中最关键的 2 到 4 项。
- `卡牌增幅` 明确说明后续玩法里存在羁绊卡牌或类似构筑系统，这要在总玩法设计中保持一致。

#### 5.2.8 面板属性与流派关系
- `攻速` + `攻击间隔` + `普攻伤害`：基础普攻流。
- `攻击范围` + `多重数量` + `多重伤害`：面清箭雨流。
- `攻击范围` + `弹射次数` + `弹射伤害`：弹射连锁流。
- `物理暴击` + `物理暴伤` + `最终伤害`：暴击爆发流。
- `技能伤害` + `魔法暴击` + `魔法暴伤`：技能爆发流。
- `生命恢复` + `伤害减免` + `杀敌恢复`：站场续航流。
- `杀敌金币` + `杀敌经验` + `杀敌加成`：经济滚雪球流。
- `力量增幅 / 敏捷增幅 / 智力增幅` + `杀敌属性`：成长叠层流。
- `火行伤害` + `燃烧伤害`：火行灼烧流。
- `水行伤害` + `冻伤伤害`：水行控制流。

#### 5.2.10 首版属性数值定义表
以下定义表用于指导首版原型、面板展示、升级强化、装备词条和数值平衡。这里的数值不是最终上线值，而是首轮可玩版本的统一基准。

首版默认英雄模板：
- `生命值`：650
- `攻击力`：35
- `护甲`：0
- `攻速`：100%
- `攻击范围`：250
- `命中`：100%
- `物理暴击`：0%
- `物理暴伤`：150%
- `魔法暴击`：0%
- `魔法暴伤`：150%
- `生命恢复`：0
- `伤害减免`：0%

#### 5.2.11 首版结算顺序
首版统一采用以下伤害结算顺序：
1. 读取基础伤害来源：普攻或技能原始伤害。
2. 叠加白字攻击或技能基础值。
3. 结算分类加成：`物理伤害 / 魔法伤害 / 普攻伤害 / 技能伤害`。
4. 结算目标类型加成：`BOSS伤害 / 精英伤害 / 挑战伤害`。
5. 结算破防与穿透：`护甲穿透 / 无视护甲 / 穿透次数`。
6. 结算通用乘区：`所有伤害`。
7. 结算最终乘区：`最终伤害`。
8. 目标侧结算减伤：`格挡 / 伤害减免 / 闪避`。
9. 生成最终伤害结果。

首版统一采用以下成长结算顺序：
1. 先计算白字基础值。
2. 再计算 `力量 / 敏捷 / 智力` 带来的白字成长。
3. 再计算 `攻击增幅 / 生命增幅 / 护甲增幅`。
4. 再计算 `每秒属性 / 杀敌属性`。
5. 最后计算 `最终攻击 / 最终生命 / 最终护甲`。

#### 5.2.12 面板展示规范
为了让 UI 更稳定，首版统一采用以下展示规则：
- 百分比属性统一显示 `%`。
- 固定值属性统一显示整数或保留 1 位小数。
- `攻击间隔` 显示实际结果值，不显示推导过程。
- 默认值为 0 的属性可以显示，但应做弱化处理。
- 未开放的属性不要在首版正式 UI 中展示，避免玩家困惑。

### 5.3 技能类型
当前首发英雄按截图方案，采用“`1 个固定基础位 + 4 个自由攻击技能位 + 各技能独立成长`”的技能系统。`普攻` 是玩家初始自带的攻击技能，英雄总共同时携带 `5 个攻击技能`，并围绕已装配技能展开成长与流派分化。

#### 5.3.1 当前首发英雄定位
当前首发英雄定义为：
- `弓手 / 游侠型英雄`
- 总共拥有 `5 个攻击技能位`
- `1 号位` 固定为初始自带技能 `普攻`
- `2 到 5 号位` 由玩家在局内逐步获得并装配
- 攻击技能采用 `通用池`，不区分英雄专属技能

核心体验：
- 以 `普攻 + 其他 4 个已装配攻击技能` 共同构成输出循环。
- 通过局内成长与技能组合共同决定本局技能构筑。
- 从基础单箭输出，逐步叠加五行箭诀、范围技、召雷、御风术等塔防式清场能力。

#### 5.3.2 核心技能：普攻
根据截图，`普攻` 是角色开局自带技能，并且：
- 默认占用 `1 号攻击技能位（固定基础位）`。
- 是所有箭矢流派的根技能。
- 首版中拥有最完整的一套技能树与卡牌分支。

`普攻` 定义为：
- 射出 1 支箭矢，对命中目标造成 `金行箭罡伤害`。
- 伤害系数基础值为 `100%攻击`。
- 拥有独立的 `攻击范围`、`攻击间隔`、`伤害增幅` 成长字段。

面板展示：
- 当前局外等级
- 伤害增幅
- 伤害系数
- 攻击范围
- 攻击间隔
- 已解锁卡牌列表

#### 5.3.3 攻击技能装配规则
当前技能装配系统固定采用以下规则：
- 英雄总共同时拥有 `5 个攻击技能位`。
- `普攻` 为初始自带技能，默认装配在 `1 号位`。
- 其中 `1 号位` 为固定基础位，默认由 `普攻` 占用且不参与普通替换。
- 其余 `4 个攻击技能位` 由玩家在局内逐步获得的通用攻击技能填充。
- 首版通用攻击技能池固定包含：
  `青木灵矢`
  `赤炎箭`
  `寒泉箭`
  `乙木天雷`
- 玩家在首版单局内最多只能从以上 `4 个通用攻击技能` 中装配 `4 个`。
- 每个攻击技能独立显示 `技能等级`、`技能详情面板` 与 `已解锁卡牌列表`。
- 英雄在 `10 / 20 / 30 / 40 级` 时各进入 `1 次烙印节点`。
- `烙印` 不属于攻击技能，不占用攻击技能格子。
- `烙印` 不可成长，只提供固定效果。
- 所有已获得 `烙印` 统一在 `吞噬界面` 查看。
- `攻击技能` 本身不会被吞噬。
- 首版中 `普攻`、`青木灵矢`、`赤炎箭`、`寒泉箭`、`乙木天雷` 全部采用独立技能树成长。

首版烙印获取规则：
- `10 / 20 / 30 / 40 级` 四个节点各弹出 `1 次烙印选择面板`。
- 每个烙印节点固定刷新 `3 个烙印候选`，玩家必须 `3 选 1`。
- 候选烙印按统一品质概率随机生成：`普通 65%`、`稀有 25%`、`史诗 10%`。
- 每个烙印节点固定保底 `1 个稀有或史诗烙印`；若前两张均为 `普通`，则第 `3 张` 强制从 `稀有 / 史诗` 池中抽取。
- 同一节点内不允许出现重复烙印。
- 同一局内已经获得的烙印不会再次进入后续节点候选池。
- 四个烙印节点共用同一套刷新概率与同一烙印池，不采用“等级越高烙印越强”的固定阶梯。
- 烙印不需要技能点，也不进入攻击技能强化池。
- 烙印不参与羁绊计数，不与羁绊卡系统混用。

首版烙印定案：
`普通烙印`
- `战痕烙印`：`所有攻击技能伤害 +12%`
- `迅节烙印`：`所有攻击技能冷却缩减 8%`
- `猎王烙印`：`对精英与 Boss 伤害 +15%`
- `追风烙印`：`普攻间隔 -12%`，`普攻伤害 +15%`
- `余震烙印`：每次释放攻击技能后的首次命中会触发 `余震`，对目标周围 `180` 范围造成 `40%攻击` 的 `自适应伤害`
- `汲能烙印`：每次释放攻击技能时，恢复 `1%最大生命值`，并使下一次 `普攻伤害 +20%`

`稀有烙印`
- `共鸣烙印`：每次释放攻击技能后，下一次 `普攻` 额外附加 `120%攻击` 的 `自适应伤害`；若命中敌人，则为最近释放的攻击技能返还 `0.8 秒` 冷却
- `裂界烙印`：每累计释放 `4 次攻击技能`，自动触发 `裂界冲击`；`裂界冲击` 对当前目标周围 `260` 范围内敌人造成 `220%攻击` 的 `自适应伤害`，命中精英或 Boss 时额外附加 `10%易伤`，持续 `5 秒`
- `追猎烙印`：同一目标在 `2 秒` 内同时受到 `普攻` 与 `攻击技能` 命中时，追加 `1 次追猎打击`，造成 `180%攻击` 的 `自适应伤害`；每个目标 `2 秒` 最多触发 `1 次`
- `压制烙印`：被 `减速 / 冻结 / 麻痹 / 束缚` 的敌人额外承受 `18%攻击技能伤害`；若目标为精英或 Boss，则改为额外承受 `12%攻击技能伤害`

`史诗烙印`
- `终焉烙印`：每次释放攻击技能时有 `15%` 概率立即再触发一次 `70%威力` 的同源追击
- `星蚀烙印`：每 `12 秒` 使下一次释放的攻击技能获得 `星蚀施放`；该次伤害 `+60%`，并在命中后对范围 `220` 内敌人附加 `8 秒星蚀`，`星蚀` 状态下受到的所有伤害 `+12%`
- `时轮烙印`：每累计释放 `6 次攻击技能`，立刻重置 `1 个当前处于冷却中的随机攻击技能`，并使其下一次释放伤害 `+35%`

#### 5.3.4 普攻成长字段
普攻拥有以下独立成长项：
- `伤害增幅`
- `伤害系数`
- `攻击范围`
- `攻击间隔`
- `多重数量`
- `弹射次数`
- 以及后续的五行附矢、爆炸、连锁等附加模块

- `伤害增幅` 为线性成长项，由局外等级成长与局内卡牌强化共同提升。
- `伤害系数` 为高价值卡牌奖励。
- `攻击间隔` 是高感知成长项，应谨慎发放但每次都让玩家明显感觉变强。
- `多重`、`弹射`、`五行化` 属于“形态变化”，应比单纯白字成长更稀有。

#### 5.3.5 普攻技能树卡牌类型
根据截图，当前首版优先定案 `普攻` 技能树，其卡牌池拆成 4 类：

基础强化卡：
- 直接强化普攻数值，不改变攻击形态。
- 示例：
  `强化箭矢`
  `攻速提升`

形态解锁卡：
- 让普攻获得新的攻击行为。
- 示例：
  `多重箭`
  `弹射箭`
  `奥能附矢`
  `烈焰附矢`
  `霜寒附矢`
  `雷霆附矢`
  `疾风附矢`

形态强化卡：
- 只能在已经拥有某个形态后出现，强化该流派。
- 示例：
  `多重增伤`
  `弹射增伤`
  `急速射击`
  `超速射击`

高阶派生卡：
- 在某条流派成型后开放，承担 build 质变效果。
- 根据截图中的已解锁条目，高阶派生卡包含：
  `奥能喷发`
  `火焰爆炸`
  `贯穿冰箭`
  `连环闪电`
  `狂风之力`
  `神箭手`
  `破甲强弩`
  `真实分裂`

- 前置：`普攻`

`多重箭`
- 效果：`多重数量 +1`
- 前置：`普攻`

#### 5.3.7 技能系统规则
当前英雄技能系统固定采用以下规则：
- `普攻` 永远存在，默认占用 `1 号攻击技能位`，并作为固定基础位存在。
- 英雄总共同时拥有 `5 个攻击技能`。
- 卡牌分为“可重复强化”和“唯一解锁”两种。
- 数值成长卡可重复抽取。
- 路线解锁卡通常唯一。
- 五行路线之间默认互斥，避免同一把同时叠满多种箭系流派。
- 高阶派生卡必须建立在前置路线已成型的前提下。
- 所有 `解锁等级` 字段一律定义为 `局外等级`，用于决定该卡是否已被养成系统开放。
- 局内攻击技能强化池不设置等级门槛，只校验 `前置`、`排斥`、`唯一性` 与 `最大叠层次数`。
- 当前首版局外开放顺序固定为：10 级开放 `奥能喷发`、`火焰爆炸`、`贯穿冰箭`、`连环闪电`、`狂风之力`
- 当前首版局外开放顺序固定为：13 级开放 `神箭手`
- 当前首版局外开放顺序固定为：20 级开放 `破甲强弩`
- 当前首版局外开放顺序固定为：23 级开放 `真实分裂`

#### 5.3.8 技能详情 UI 展示
参考截图，技能详情面板固定包含：
- 顶部：当前已装配 `5 个攻击技能位` 的页签或切换条
- 左上：技能图标与名称
- 中部：当前局外等级、伤害类型、伤害系数、攻击范围、攻击间隔
- 中下：已解锁卡牌列表
- 底部：卡牌查看按钮、关闭按钮

技能入口分工规则：
- `G` 与角色操作区中的 `技能` 按钮，统一用于打开 `攻击技能强化 3 选 1 面板`，不直接进入某个单独技能页签。
- 底部角色操作区中的 `5 个攻击技能位` 与详情面板顶部页签，统一用于查看当前已装配攻击技能的成长详情、已解锁卡牌与数值变化。
- 当玩家主动点击某个攻击技能位进入详情模式时，优先进入该技能对应页签。
- 若当前没有明确选中的攻击技能位，则技能详情页默认进入 `1 号位普攻页签`。
- 玩家在技能详情面板内可以通过顶部技能页签切换到其他已装配攻击技能。
- 技能详情面板只承担“查看成长结果”的职责，不提供 `升级按钮` 与 `重置按钮`；所有技能成长都统一通过 `G` 的 `攻击技能强化 3 选 1` 完成。

设计上要强调：
- 玩家看到的不是“技能描述文本堆叠”，而是“这把 build 已经往哪个方向长了”。

#### 5.3.13 第二批通用攻击技能卡牌树
第二批通用攻击技能统一采用以下规则：
- 每个技能固定采用 `9 张强化牌 + 1 张终局牌` 模板。
- 品质固定为 `普通 3 / 优秀 3 / 稀有 3 / 传奇 1`。
- `普通` 只负责基础伤害、频率、功能，`优秀` 负责范围、数量、形态，`稀有` 负责状态、对高威胁目标与触发特化，`传奇` 负责终局定型。
- 除 `传奇` 外，单张卡默认只承载一类核心成长方向，不允许把多类核心职责堆叠在同一张卡上。
- 第二批技能默认仍受 `前置`、`排斥`、`唯一性` 约束，但这些约束服务于构筑分支，而不是用于堆砌多层级卡树。
- 第二批技能的详细文案、终局命名与结构化蓝图，统一维护在 [第二批通用攻击技能与进化卡设计.md](/C:/Y3TD/Y3GPT/ProjectName002/maps/EntryMap/script/docs/归档/内容设计/05-技能/第二批通用攻击技能与进化卡设计.md) 与 [second_batch_skills.lua](/C:/Y3TD/Y3GPT/ProjectName002/maps/EntryMap/script/entry_objects/attack_skill_blueprints/second_batch_skills.lua)。

第二批技能统一采用以下卡槽顺序：
- `普通 1`：基础伤害成长
- `普通 2`：基础频率成长
- `普通 3`：基础功能成长
- `优秀 1`：范围成长
- `优秀 2`：数量成长
- `优秀 3`：形态成长
- `稀有 1`：状态特化
- `稀有 2`：对单或高威胁目标特化
- `稀有 3`：特殊触发或联动特化
- `传奇 1`：终局形态

### 5.4 敌人设计原则
- 普通怪：提供基础资源与节奏压力。
- 快速怪：高移速，测试单体点杀和减速能力。
- 厚血怪：测试持续输出与破甲能力。
- 远程怪：迫使玩家考虑防御、治疗、优先级击杀。
- 精英怪：携带额外词条，如护盾、召唤、复活、光环。
- Boss：阶段式战斗，要求 build 检定和短时爆发。
