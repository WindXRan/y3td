# 中央决策面板结构标注示例（MOBA端游习惯版）

> 这是 [UI结构标注图模板](./16-UI结构标注图模板.md) 的轻量示例。目标不是做复杂施工图，而是让 AI 和程序快速理解：`中央决策面板` 应该像端游强化/商店弹窗那样稳定、可读、低干扰。

## 1. 一句话结构
- 中间是一块大弹窗，上方标题栏，左侧构筑摘要，中间三列候选，下方固定操作栏，外面是一层轻度遮罩。

## 6. 程序接法

```md
程序接法：
- 聚焦遮罩铺满全屏
- 中央面板外框用 percent 居中
- 面板内部固定分三层：标题栏 / 内容区 / 底部栏
- 内容区固定再分两列：左摘要栏 + 中央候选舞台
- 候选舞台固定 3 列，不因文案长度改变列结构
- 刷新/稍后/关闭都放在稳定位置，不随系统变化漂移
```

## 7. MOBA 习惯化优化点
- 中央决策面板要更像 PC 端游的 `强化弹窗 / 商店弹窗 / 天赋面板`，而不是“覆盖战场的整屏卡牌页”。
- 标题栏固定在上、操作栏固定在下，能建立稳定肌肉记忆。
- 左摘要栏是端游习惯里很重要的“当前状态参考区”，能减少玩家做选择时来回切界面的负担。
- 候选卡保持工整 3 列，强调“对比选择”，而不是强调“卡面展示表演”。
- 遮罩只做轻度压暗，不要把战场完全抹黑，保留玩家对战局的连续感。

## 8. 给 AI 的使用方式
- 先把这份结构清单发给 AI，让它出 `低保真中央弹窗草图`。
- 再让 AI 基于这份结构输出：
  `通用资源拆分`
  `G/F/宝物/烙印 四种变体建议`
  `端游风格中央弹窗 prompt`
