# 底部角色操作区资源清单示例（MOBA端游习惯版）

> 这是 [UI资源清单表模板](./15-UI资源清单表模板.md) 的轻量示例，目标是让 `底部角色操作区` 更符合 `MOBA 端游用户习惯`，让玩家第一眼就知道英雄信息、技能热键区和功能入口分别在哪。

## 1. 页面说明

```md
页面：底部角色操作区
目标：让玩家在战斗中快速读取英雄核心信息、技能热区、构筑入口和功能按钮
风格词：moba_pc、功能热键栏、稳重金属、清晰高对比
复用基底：bottom_hud_shell
```

## 2. 模块

```md
模块：
- 英雄核心区 | 左侧 | 头像、等级、主属性摘要、成长武器格 | 动态
- 技能热键区 | 中间 | 4 个攻击技能位，强调战斗热区 | 动态
- 主功能簇 | 右侧上排 | 技能(G)、羁绊(F) 两个主按钮 | 动态
- 次功能簇 | 右侧下排 | 宝物、集火、已吞噬 三个次级按钮 | 动态
- 经验轨 | 底部横向 | 当前经验和升级进度 | 动态
```

## 4. 文本

```md
文本：
- 英雄名 | 游侠 | 1 行 | 左对齐
- 等级 | 12 | 1 行 | 居中
- 属性摘要 | 攻击 120 / 暴击 18% | 1 行 | 左对齐
- 武器等级 | 武器 Lv.8 | 1 行 | 居中
- 按钮主文案 | 技能 / 羁绊 / 宝物 / 集火 / 已吞噬 | 1 行 | 居中

- 宝物数量 | 2/3 | 1 行 | 居中
```

## 5. 程序备注

```md
程序备注：
- 整个底栏：bottom-center 吸附
- 英雄核心区放左，技能热键区保持几何中心
- G/F 主按钮比右下三个次级按钮更大，更靠中间
- 成长武器格放在英雄区和技能区之间，像 MOBA 的特殊装备槽
- 经验轨做整条底边细轨，不做厚经验框
- 所有文本独立，不烘进图
- 可拉伸底板全部走九宫
```

## 6. MOBA 习惯化优化点
- 左侧保留 `英雄头像 + 等级 + 角色摘要`，符合 PC MOBA 老玩家的第一定位习惯。
- 中间把 `4 个攻击技能位` 做成明确热区，保证战斗操作第一眼就在屏幕底部中心。
- `技能(G)` 和 `羁绊(F)` 虽然不是传统 QWER 技能，但仍按“主操作按钮”处理，放在中心右侧最近手眼焦点的位置。
- `宝物 / 集火 / 已吞噬` 作为半战斗功能，降到次一级视觉权重，避免抢中间技能区注意力。
- 整体更像 `英雄面板 + 热键栏 + 右侧功能簇`，而不是一排同权重按钮。
