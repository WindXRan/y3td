# UI资源清单表模板（AI快速版）

> 给 `AI / 程序 / 美术` 共用的轻量模板。目标是 `5 分钟写完`，能直接拿去出第一版结构、资源和接入骨架。

## 1. 使用原则
- 当前默认走 `AI 快速协作模式`，不是传统大交付单。
- 一页只写 `一个界面`。
- 能一句话说清，就不要拆成复杂表格。
- 先写 `逻辑名`，后面再补正式资源名。
- 先让 AI 和程序跑起来，再做精修。
- 若你已经使用 [UI单文件YAML规格模板](./19-UI单文件YAML规格模板.md)，这里可以只当作 `展开版查看方式`。

## 2. 只需要写 5 块内容
- `页面说明`
- `模块列表`
- `资源列表`
- `文本列表`
- `程序备注`

## 3. 推荐写法

```md
页面：顶部战斗HUD
目标：战斗中快速看到波次、Boss、资源
风格词：奇幻、克制、清晰、压迫感
复用基底：无

模块：
- 顶部主条 | 顶部居中 | 波次/Boss/倒计时 | 动态
- 资源栏 | 右上 | 金币/木材/杀敌 3 格 | 动态
- 玩家统计 | 资源栏下方 | 1-4 行列表 | 动态

资源：
- top_bar_bg | 底板 | 顶部主条 | 可拉伸
- boss_info_bg | 底板 | Boss区 | 预警/战斗 两态
- resource_cell_bg | 底板 | 资源栏 | 复用 3 次
- icon_gold | 图标 | 资源栏 | 金币

文本：
- 波次标题 | 第 1 / 5 波 | 1 行 | 居中
- Boss名称 | 深渊吞噬者 | 1 行 | 左对齐
- 资源值 | 999999 | 1 行 | 居中

程序备注：
- 顶部主条：percent 居中
- 资源栏：顶部 + 右侧吸附
- 可拉伸底板：九宫
- 文本独立，不烘进图
```

## 4. 更适合喂给 AI 的紧凑格式

```yaml
page: 顶部战斗HUD
goal: 战斗中快速看到波次、Boss、资源
style: [奇幻, 克制, 清晰, 压迫感]
base: 无

modules:
  - name: 顶部主条
    pos: top-center
    size: 920x112
    dynamic: [wave, boss, timer]
  - name: 资源栏
    pos: top-right
    size: 420x116
    dynamic: [gold, wood, kill]

assets:
  - name: top_bar_bg
    type: 底板
    module: 顶部主条
    stretch: true
  - name: boss_info_bg
    type: 底板
    module: Boss区
    state: [prepare, fight]

texts:
  - use: 波次标题
    sample: 第 1 / 5 波
    lines: 1
    align: center
  - use: Boss名称
    sample: 深渊吞噬者
    lines: 1
    align: left

runtime:
  - 顶部主条用 percent 居中
  - 资源栏用 right/top 吸附
  - 所有文本独立
```

## 6. AI 快速检查
- AI 能不能直接知道 `要画什么`
- 程序能不能直接知道 `放哪里`
- 文本有没有和背景焊死
- 底板是否一眼能看出 `要不要拉伸`
