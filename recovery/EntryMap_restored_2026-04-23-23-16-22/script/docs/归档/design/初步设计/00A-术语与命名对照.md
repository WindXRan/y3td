# 术语与命名对照

> 用于统一当前项目在 `文档 / CSV / runtime / 玩家可见文案` 四层里的命名口径。若不同文档出现冲突，以本页和现行数据表为准。

## 1. 当前总原则

- 玩家可见包装统一采用 `修仙 / 五行` 口径。
- 运行时伤害结构统一采用 `damage_type + damage_form + element + damage_label`。
- `damage_type` 目前仍保留兼容职责，主判定口径优先看 `damage_form` 与 `element`。
- 当前没有 `五行相克`、也没有敌方五行抗性系统。

## 4. 伤害字段口径

| 字段 | 当前含义 | 示例 |
| --- | --- | --- |
| `damage_type` | 旧兼容字段，主要保留 `物理 / 法术` 的老口径 | `物理` |
| `damage_form` | 运行时伤害形态 | `weapon / spell / dot / summon` |
| `element` | 五行字段 | `metal / wood / water / fire / earth / none` |
| `damage_label` | 玩家可见伤害名 | `金行箭罡` |

## 5. 属性旧新对照

| 旧属性名 | 当前属性名 |
| --- | --- |
| `物系伤害` | `金行伤害` |
| `火系伤害` | `火行伤害` |
| `电系伤害` | `木行伤害` |
| `冰系伤害` | `水行伤害` |
| `风系伤害` | `木行伤害` |
| `能量伤害` | `金行伤害` |

说明：
- 上表是当前项目里的兼容映射，不代表严格世界观等价。
- `风` 相关旧包装当前并入 `木行` 侧术法表现。
- `能量` 旧包装当前按金行术法侧收口；个别技能若已在 CSV 中落成其他五行，以实际数据为准。
