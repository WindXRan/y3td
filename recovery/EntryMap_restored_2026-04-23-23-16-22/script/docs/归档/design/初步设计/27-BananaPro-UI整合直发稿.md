# BananaPro UI整合直发稿

> 本文件可直接发给 BananaPro。目标是为本项目产出一套 `更符合 MOBA / PC 端游用户习惯` 的局内核心 UI 方案。请把它当成完整需求，不依赖其他文档。

## 1. 你的任务
- 你是一名擅长 `PC MOBA / ARPG / RTS HUD` 的游戏 UI 设计师。
- 你要为一个 `单英雄守线 + 肉鸽构筑 + 奇幻战场` 项目设计局内核心 UI。
- 这次只做 `局内核心界面`，不做登录、局外养成、商城、社交。
- 重点不是做“好看的海报 UI”，而是做：
  `战斗中读得快`
  `符合 PC 端游习惯`
  `方便程序拆资源接入`
  `后续可继续扩展`

## 2. 项目背景
- 游戏类型：
  `单英雄守线`
  `局内肉鸽构筑`
  `5 波标准模式`
  `有 Boss、挑战、宝物、羁绊、攻击技能强化、烙印`
- 目标用户：
  熟悉 `MOBA 端游`、`动作端游`、`魔兽 / Dota / LoL` 式阅读习惯的玩家。
- 用户预期：
  `顶部看战斗节奏`
  `底部看英雄和技能`
  `中央做关键选择`
  不希望像移动端整屏卡牌页，也不希望像花哨的 RPG 全屏菜单。

## 3. 总体设计方向

### 3.1 核心关键词
- `moba_pc`
- `奇幻金属`
- `稳重`
- `清晰高对比`
- `战术桌面感`
- `功能优先`

### 3.2 必须符合的习惯
- 顶部承担：
  `战斗节奏`
  `波次`
  `Boss`
  `资源`
- 底部承担：
  `英雄归属`
  `技能热区`
  `功能按钮`
  `经验进度`
- 中央承担：
  `关键决策弹窗`
  `3 选 1`
  `刷新 / 稍后 / 返回`
- 读图顺序优先：
  `战斗节奏信息`
  `技能热区`
  `资源`
  `构筑摘要`
  `统计`

### 3.3 明确避免
- 不要做成 `移动端整屏卡牌页`
- 不要做成 `一堆漂浮胶囊和海报卡片`
- 不要大面积 `紫白默认风`
- 不要强依赖 `满屏发光、重粒子、夸张浮层`
- 不要把标题、正文、数字长期烘焙进大底图
- 不要把所有按钮做成同尺寸、同权重平铺

## 4. 整体布局原则

### 4.1 顶部 HUD
- 顶部中间是一条 `扁平状态带`，承担：
  `关卡标签`
  `波次主信息`
  `Boss 信息`
  `倒计时 / 时间轨`
- 顶部右侧是一组 `紧凑资源区`，显示：
  `金币`
  `木材`
  `杀敌`
- 资源区下方是 `轻量玩家统计卡`
- 重点是：
  `中上方主信息最显眼`
  `右上资源次一级`
  `统计最弱`

### 4.2 底部角色操作区
- 左边是 `英雄核心区`：
  `头像`
  `等级`
  `英雄名`
  `属性摘要`
- 中间是 `技能热键区`：
  `4 个攻击技能位`
  要明显像 PC MOBA 的技能热区
- 技能区右边是 `G / F 主功能按钮`
- 最右下是 `宝物 / 集火 / 已吞噬` 次级功能按钮
- 底边是一条 `细经验轨`
- 重点是：
  `英雄在左`
  `技能在中`
  `功能在右`
  `经验在底边`

### 4.3 中央决策面板
- 中央是一块 `端游强化/商店/天赋风格` 的大弹窗
- 上方固定：
  `主标题`
  `副标题`
  `关闭 / 返回`
- 左侧固定：
  `当前构筑摘要`
  `当前羁绊`
  `当前宝物`
  `当前技能路线`
- 中间固定：
  `3 张候选`
  或 `3 个候选槽位`
- 下方固定：
  `刷新`
  `稍后选择`
  `提示文案`
  `费用信息`
- 外围只做 `轻度遮罩`，不要把战场完全抹黑

## 5. 三大核心区详细要求

### 5.1 顶部 HUD 详细要求
- 顶部主信息带要扁平、横向、稳定。
- `第 X / 5 波` 必须是一眼可见的主信息。
- `Boss 来临倒计时 / Boss 血量信息` 必须和波次同级或次一级。
- `关卡节点` 可以更轻，只作为上下文标签存在。
- 资源区不要做成笨重表格，更像 `紧凑资源簇`。
- 玩家统计卡只作为辅助信息，不要抢波次和 Boss 的注意力。

建议结构：
- 左：关卡标签
- 中：波次主信息
- 右：Boss 信息胶囊
- 下：细时间轨
- 右上：资源窄卡 3 连
- 右下：轻量统计卡

### 5.2 底部角色操作区详细要求
- 头像和等级要给玩家稳定的“这是我的英雄”的归属感。
- 技能热区必须是底栏视觉中心，不能被右侧功能按钮挤偏。
- `技能(G)` 与 `羁绊(F)` 应像“主操作按钮”，而不是菜单按钮。
- `宝物 / 集火 / 已吞噬` 是次级入口，视觉弱于 G/F。
- `成长武器格` 适合放在英雄区和技能区之间，像一个特殊装备槽。
- 经验轨做成长条细轨，贴底边，不做厚重大框。

建议结构：
- 左：头像、等级、属性摘要
- 中左桥接：成长武器格
- 中：4 技能位
- 中右：G / F 主按钮
- 右下：宝物 / 集火 / 已吞噬
- 最底：经验轨

### 5.3 中央决策面板详细要求
- 中央面板要更像 `PC 端游弹窗`，不是整屏卡牌页。
- 同一套框架要能服务：
  `攻击技能强化`
  `羁绊抽卡`
  `宝物候选`
  `烙印选择`
  `成长武器词缀`
- 玩家做决策时必须同时看到：
  `当前 build 摘要`
  `当前 3 个候选`
  `刷新/返回/稍后选择`
- 候选区应保持稳定三列，便于比较，不做散落式漂浮构图。
- 标题栏和底部操作栏位置必须稳定，形成肌肉记忆。

建议结构：
- 上：标题栏
- 左：构筑摘要栏
- 中：三列候选
- 下：操作栏

## 6. 视觉风格要求

### 6.1 材质与形态
- 材质建议：
  `深色金属`
  `皮革`
  `轻魔法纹`
- 面板和按钮应有统一语言：
  `硬朗边框`
  `轻度磨损`
  `清晰内框`
- 不要走：
  `纯科技蓝屏`
  `纯现代扁平白卡`
  `Q 萌手游弹窗`

### 6.2 色彩层级
- 背景底板：
  深灰、铁黑、暗金属
- 主要高亮：
  金色、暖白、冷青
- 品质色：
  `普通 = 深灰`
  `稀有 = 蓝`
  `史诗 = 紫`
- 危险提示：
  `橙红`
- 成功 / 可用：
  `亮金 / 亮青`

### 6.3 文本与数字
- 数字必须比装饰更重要。
- 标题清楚、正文简短、标签短促。
- 不允许为了视觉好看牺牲可读性。
- 技能点、木材消耗、倒计时、Boss 血量都要优先读清。

## 7. 资源拆分规则
- 请按 `可拼装资源` 的思路出，不要出单张整屏成品图。
- 所有文字默认独立，不烘进背景图。
- 所有长底板、按钮底、面板底尽量支持 `九宫 / 拉伸`。
- 图标、底板、标签、描边、角标尽量拆开。
- 候选卡请至少拆：
  `卡底`
  `品质框`
  `标签`
  `状态条`
  `图标位`

## 8. 交付给程序的约束
- 程序会优先按：
  `顶部 / 底部 / 左侧 / 右侧吸附`
  `center / percent`
  `局部坐标`
  来搭结构。
- 程序会优先使用：
  `可拉伸底板`
  `独立文本`
  `占位容器`
  的方式接入。
- 因此你的设计必须尽量支持：
  `换资源不改结构`
  `换品质不改布局`
  `换文案不重画底图`

## 9. 输出要求

### 9.1 先输出这 4 部分
- `整体视觉方向`
  一段话说明这套 UI 的核心视觉语言。
- `三大核心区低保真结构稿`
  顶部 HUD、底部角色操作区、中央决策面板。
- `每个区域的资源拆分建议`
  底板、图标、标签、按钮、文本区如何拆。
- `程序友好的适配说明`
  哪些区域固定，哪些可拉伸，哪些文本独立。

### 9.2 如果你可以继续深化
- 再给出：
  `中保真视觉稿方向`
  `统一按钮体系`
  `统一卡牌体系`
  `统一标题栏体系`

## 10. 输出格式建议
- 先给 `整体方向说明`
- 再分 3 段输出：
  `顶部 HUD`
  `底部角色操作区`
  `中央决策面板`
- 每段固定包含：
  `布局`
  `视觉重点`
  `资源拆分`
  `适配要点`

## 11. 额外提示
- 这是一个偏战斗、偏构筑、偏端游习惯的项目。
- 你的目标不是做炫技 UI，而是做：
  `一眼会用`
  `读得快`
  `程序容易接`
  `可持续扩展`
- 如果你觉得某块布局太像手游，请主动把它拉回 `MOBA / PC 端游习惯`。
