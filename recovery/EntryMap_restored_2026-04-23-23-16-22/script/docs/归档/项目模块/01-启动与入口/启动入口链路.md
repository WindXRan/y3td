# 启动入口链路

## 模块职责

说明项目如何从地图初始化进入业务运行时，以及入口层与玩法层的边界。

## Source Of Truth

- 启动文件：`maps/EntryMap/script/main.lua`
- 运行时入口：`maps/EntryMap/script/entry_runtime.lua`

## 关键状态与数据流

- `main.lua` 维护 `runtime`、`runtime_load_attempted`、`bootstrapped`
- `bootstrap_once()` 通过 `游戏-初始化` 事件与 `ltimer.wait(0)` 双重触发，避免漏启动
- `entry_runtime.bootstrap()` 负责真正的会话态初始化、局外入口和运行时注册

## 当前已实现行为

- `main.lua` 会根据调试模式设置日志级别
- `main.lua` 用 `xpcall(require 'entry_runtime')` 做入口加载兜底
- `require 'entry_runtime'` 时会先把配置、静态对象和运行时子模块装载进内存
- `entry_runtime.bootstrap()` 当前执行顺序稳定：
  - `validate_config()`
  - `reset_session_state()`
  - `register_runtime_events()`
  - `register_dev_commands()`
  - `start_runtime_loops()`
  - `ensure_gm_panel()`
  - `outgame_system.load_profile()`
  - `outgame_system.enter_outgame()`
- 启动后先显示局外选关页，不再自动创建英雄或自动起波
- 真正开局发生在局外页点击开始之后：
  - `start_selected_stage()`
  - `reset_battle_state()`
  - `create_hero()`
  - `initialize_hero_progression()`
  - `setup_basic_attack_ability()`
  - `ensure_runtime_hud()`
  - `start_wave(1)`

## 未实现或占位项

- 当前入口层已经接通局外选关、章节/模式选择和表型存档骨架
- 仍未实现正式 Loading、多英雄入口、地图切换式大厅和长期局外养成页

## Agent 修改建议与风险点

- 想改“游戏开始时做什么”，先改 `entry_runtime.bootstrap()`，不是改 `main.lua` 的外围壳
- 想新增真实启动分支时，优先在 `bootstrap()` 内拆模块，不要把业务规则堆回 `main.lua`
- 想加新的启动前校验、预加载或 HUD 初始化，也优先放在 `bootstrap()` 的明确阶段
- `main.lua` 底部的大段 `if false then` 旧原型代码不是当前主线，不要误改
