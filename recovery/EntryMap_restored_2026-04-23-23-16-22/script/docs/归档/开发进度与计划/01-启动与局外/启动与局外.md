# 启动与局外

## 文档职责

说明当前真实启动链路、局外页、章节/模式解锁和局外存档骨架的完成度。

## Source Of Truth

- `maps/EntryMap/script/main.lua`
- `maps/EntryMap/script/entry_runtime.lua`
- `maps/EntryMap/script/entry_runtime_outgame.lua`
- `maps/EntryMap/script/entry_config.lua`
- `maps/EntryMap/script/entry_objects/stages`
- `maps/EntryMap/script/entry_objects/stage_modes`

## 已实现

- `main.lua -> require 'entry_runtime' -> bootstrap()` 的真实启动链路已经切到局外页，不再地图启动后自动起波。
- `entry_runtime.lua` 通过 `STATE.session_phase` 区分 `outgame` / `battle`，并在开始新局时重建战斗态。
- 局外页已动态挂到 `GameHUD`，提供 `1-1`、`1-2`、`1-3` 三张章节卡和 `standard` / `challenge` 两种模式按钮。
- 新档默认解锁 `1-1 standard`；通关某章节 `standard` 后解锁本章 `challenge` 和下一章 `standard`。
- `outgame_profile` 已保存 `selected_stage_id`、`selected_mode_id`、`stage_progress`、`last_result`，并带有默认值修正与非法数据回退。
- 战斗结束后会回到局外页，并把最近结果、章节解锁与模式解锁写回当前存档骨架。

## 部分实现

- `challenge` 当前主要只有模式身份、解锁逻辑、独立结果记录和局外入口；没有开局挑战词条选择，也没有单独的规则覆写闭环。
- `1-2`、`1-3` 当前虽然拥有独立章节卡、独立解锁进度和独立最近结果，但 `content_source_stage_id` 仍指向 `1-1`，实际战斗内容复用 `1-1`。
- 当前存档只覆盖章节/模式解锁和最近结果，不包含局外货币、挑战奖励、英雄解锁、图鉴或正式成长数据。
- 当前局外页是 Lua 动态挂载的骨架页面，不是完整大厅、Loading、章节详情或长期养成界面。

## 未实现

- 挑战模式词条选择、规则覆写、奖励档位与挑战奖励写回局外存档。
- 局外货币、长期成长、英雄解锁、攻击技能 / 羁绊 / 烙印内容解锁、图鉴、任务、成就、签到。
- 正式结算奖励页、章节详情页、地图切换式大厅和多英雄局外入口。

## 后续开发建议

1. 先把 `challenge` 从“可选模式名”补成真正的挑战模式：开局词条选择、规则覆写、奖励写回。
2. 扩展 `outgame_profile` 时保持默认值修正和非法值回退，避免旧存档直接损坏。
3. 在长期局外成长落地前，继续把“章节/模式解锁骨架”和“长期养成数据”分层维护，不要混到一个状态表里。
