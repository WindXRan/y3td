## 状态定义

- `已实现`
  - 已接入真实 runtime，玩家可以在当前主链路中实际触发并完成闭环。
- `部分实现`
  - 已有 runtime 状态、静态对象、入口或占位字段，但还没有形成设计要求的完整玩家体验。
  - 典型例子：
    - `1-2`、`1-3` 当前可进入，但战斗内容仍复用 `1-1`
    - 宝物、烙印已有轮次与队列，但没有正式中央决策面板
    - 波次对象已有 `boss_timeline_profile_id`，runtime 仍未消费
- `未实现`
  - 仍停留在设计文档、字段预留或资源占位，未进入真实 runtime 主链路。

## 当前阶段结论

- 已跑通的主闭环：
  - 局外选关骨架
  - 单英雄 `5 波` 战斗
  - Boss 切波
  - `G` 强化
  - `F` 羁绊
  - 四类挑战
  - 宝物
  - 烙印
  - 奖励队列
  - 运行时 HUD
  - GM 调试
- 当前高价值决策主要仍由文本消息、HUD 按钮和待选轮次恢复逻辑驱动，不是正式中央决策面板体验。
- 局外当前只有章节/模式解锁和最近结果回流，长期养成、局外货币与挑战奖励写回没有落地。
- 设计中已明确但仍未接入 runtime 的高优先级缺口包括：
  - 成长武器与词缀
  - 正式中央决策面板 / 背包 / 奖励记录 / 结算页
  - Boss 时间轴与低血机制
  - 挑战词条与局外写回
  - 多英雄
  - 章节专属内容
- `G / F / 宝物` 当前已经从“首版原型”扩展到“可追路线的构筑闭环”：羁绊支持配方进化、技能支持路线质变、宝物支持临时战术 Buff。

## 当前推荐开发顺序

1. 先补正式中央决策面板、背包、奖励记录与正式结算 UI，把现有待选轮次从文本/HUD 驱动升级为正式交互。
2. 接入成长武器、金币强化、词缀节点与对应入口，补齐 `金币 -> 构筑成长` 主循环。
3. 消化现有波次对象中的 Boss 预留字段，补 Boss 时间轴、低血强化和更正式的 Boss 机制。
4. 把挑战模式做成真正的模式差异：挑战词条选择、规则覆写、挑战奖励写回局外存档。
5. 最后扩展长期局外成长、多英雄、章节专属内容和更多正式敌人与 Boss 资源。

## 维护规则

- 更新本目录前，先核对 `maps/EntryMap/script/*.lua` 与 `entry_objects`，不要只对着 `docs/design` 下结论。
- 新功能落地后，至少同步修改对应模块页和本索引中的阶段结论。
- 如果某项功能仍依赖临时物编、复用章节或仅有调试入口，应优先归类为 `部分实现`，而不是 `已实现`。
