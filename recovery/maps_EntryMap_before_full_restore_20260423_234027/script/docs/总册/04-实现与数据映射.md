# 实现与数据映射

## 数据优先级

1. `script/data_csv/*`
2. `script/data/object_tables/*`
3. `script/runtime/*`
4. `script/entry_objects/*`

## 文档目标

- 说明当前现行数据事实源与装配落点
- 说明已经纳入统一校验入口的 CSV 模块
- 说明新增 CSV 模块时应遵守的最低维护规范

## 作用范围

- 包含：现行 CSV、object table、compat 规则表、统一校验入口
- 不包含：具体玩法规则、详细数值平衡、历史拆表过程

### 宝物 CSV 现行约束

- `treasures.csv`、`treasure_sets.csv`、`treasure_set_members.csv`、`treasure_set_effects.csv`、`treasure_effects.csv` 全部使用显式 `order_index`
- 不再依赖 CSV 行顺序表达显示顺序、成员顺序或效果执行顺序
- 套装件宝物必须在 `treasures.csv` 上声明 `is_set_item=true` 与 `set_id`
- 非套装宝物不得填写 `set_id`
- `treasure_sets.csv` 的 `piece_count` 必须与 `treasure_set_members.csv` 中实际成员数一致
- compat 规则表统一使用 `match_* / output_*` 风格字段，避免同义列名并存

### 羁绊 CSV 现行约束

- `bond_nodes.csv` 中的 `parent_id / next_ids` 必须能指向真实节点
- root 节点必须同时存在于 `bond_root_sets.csv`
- 节点 `group_id` 必须能在 `bond_group_choices.csv` 中找到对应分组元数据
- `bond_group_choice_paths.csv` 只负责描述同组解锁路径文本，不应引用不存在的 group
- object table 层负责把节点、根套装、抽取配置装配成稳定结构，运行时直接消费装配结果

## 需要注意的旧口径

- `damage_type` 不是唯一伤害事实源
- 旧 `treasure_id` 示例大多已经废弃
- 旧 `element_type` 已替换为 `damage_form + element + damage_label`

## CSV 维护通用规则

- 主表与子表一旦存在“顺序有意义”的语义，优先使用显式顺序字段，不依赖 CSV 行顺序
- object table 层负责把 CSV 装配为稳定结构，不在 runtime 层重复拼装静态关系
- 每新增一组核心 CSV，至少补一条 loader smoke 和一条关系一致性 smoke
- 关系一致性 smoke 应接入 `script/tools/verify_csv_object_tables.py`，避免变成只能手动记忆的额外命令
- compat 规则表优先使用 `match_* / output_*` 风格字段命名，减少同义字段并存
- 当前统一入口已覆盖：宝物、烙印、章节与模式、波次与挑战、羁绊
