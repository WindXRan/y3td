# 局外选关与存档骨架

## 模块职责

说明 `EntryMap` 当前已落地的局外页、章节/模式解锁、最近结果回流和首版存档结构。

## Source Of Truth

- 局外运行时：`maps/EntryMap/script/entry_runtime_outgame.lua`
- 总协调器：`maps/EntryMap/script/entry_runtime.lua`
- 局外配置汇总：`maps/EntryMap/script/entry_config.lua`
- 章节定义：`maps/EntryMap/script/entry_objects/stages`
- 模式定义：`maps/EntryMap/script/entry_objects/stage_modes`

## 当前状态归属

- 会话态保留在 `entry_runtime.lua` 的 `STATE`
  - `STATE.session_phase`
  - `STATE.outgame_profile`
  - `STATE.selected_stage_id`
  - `STATE.selected_mode_id`
  - `STATE.current_stage_def`
  - `STATE.current_mode_def`
  - `STATE.last_battle_result`
  - `STATE.outgame_ui`
- 战斗态仍按局内生命周期重建
  - 英雄、资源、波次、挑战、羁绊、技能、宝物、烙印、奖励队列、战斗 HUD runtime

## 章节与模式结构

- 当前固定展示 3 个章节：
  - `1-1`
  - `1-2`
  - `1-3`
- 当前固定提供 2 个模式：
  - `standard`
  - `challenge`
- `1-2` 和 `1-3` 当前允许进入，但 `content_source_stage_id` 暂时都指向 `1-1`
- 因此第一版的章节差异先体现为：
  - UI 独立卡片
  - HUD 独立章节名
  - 存档独立通关/解锁记录
  - 战斗内容暂复用 `1-1`

## 解锁规则

- 新档默认只解锁 `1-1 standard`
- 通关 `N standard`
  - 标记该章 `standard_cleared = true`
  - 解锁该章 `challenge`
  - 解锁下一章 `standard`
- 通关 `N challenge`
  - 只记录 `challenge_cleared = true`
  - 不额外解锁下一章
- 失败
  - 不解锁新内容
  - 只更新最近结果

## 存档结构

- 存档槽位由 `entry_config.lua` 暴露：
  - `save_slots.outgame_profile = 1`
- 当前使用的表型存档骨架包含：
  - `version`
  - `selected_stage_id`
  - `selected_mode_id`
  - `stage_progress`
  - `last_result`
- `stage_progress[stage_id]` 当前固定记录：
  - `standard_unlocked`
  - `standard_cleared`
  - `challenge_unlocked`
  - `challenge_cleared`
- `last_result` 当前固定记录：
  - `stage_id`
  - `mode_id`
  - `is_win`
  - `reached_wave_index`

## 加载与修正策略

- 地图启动时不再直接创建英雄或自动起波
- `bootstrap()` 会先加载局外存档，再进入局外页
- 如果存档中的选中章节/模式非法，或章节/模式当前未解锁：
  - 自动回退到“最高已解锁章节的 `standard`”
  - 最差回退到 `1-1 standard`
- 如果表字段缺失，`entry_runtime_outgame.lua` 会补齐默认值

## UI 策略

- 第一版局外页不新增独立 JSON 资产
- UI 直接动态挂在 `GameHUD` 根节点
- 战斗 HUD 在 `outgame` 阶段隐藏，在 `battle` 阶段显示
- 局外页当前至少包含：
  - 左侧章节卡
  - 右侧模式切换
  - 开始按钮
  - 最近结果区域

## 战斗回流流程

```text
bootstrap()
  -> load_profile()
  -> enter_outgame()

局外点击开始
  -> start_selected_stage()
  -> reset_battle_state()
  -> create_hero()
  -> initialize_hero_progression()
  -> setup_basic_attack_ability()
  -> start_wave(1)

finish_game()
  -> 生成 battle_result
  -> cleanup_battle_units()
  -> apply_battle_result()
  -> enter_outgame(result)
  -> 刷新局外页、最近结果和解锁状态
```

## 当前实现边界

- 已落地的是“局外选关与存档骨架”，不是完整长期养成
- 当前没有局外货币、图鉴、英雄解锁和正式结算奖励页
- `challenge` 第一版只有独立模式身份、独立解锁和独立结果记录
- 当前没有挑战词条选择器，也没有章节级大规模数值覆写
