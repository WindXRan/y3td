# UI结构标注图模板（AI快速版）

> 这不是正式施工图，而是给 `AI / 程序 / 美术` 共用的 `结构草图模板`。能把布局和动态区说清就够了，不追求传统标注图的完整度。

## 1. 使用原则
- 低保真即可，不需要精美。
- 不强制像素级精确。
- 不强制复杂编号。
- 若你已经使用 [UI单文件YAML规格模板](./19-UI单文件YAML规格模板.md)，这里可以只保留 `草图` 部分。
- 可以是：
  `框图`
  `截图上画框`
  `ASCII 草图`
  `Markdown 结构清单`
- 重点只看：
  `大模块在哪`
  `大概多大`
  `哪里会动态变化`
  `哪些底板要拉伸`

## 2. 最少要标的 4 类信息
- `模块位置`
- `大小关系`
- `动态文本区`
- `九宫 / 拉伸底板`

## 3. 推荐写法

```md
页面：顶部战斗HUD
画布：动态宽 x 1080

布局：
- 顶部主条 | top-center | 920x112 | percent(50,48)
- 资源栏 | top-right | 420x116 | top=24 right=32
- 玩家统计 | under(资源栏) | 420x132 | top=148 right=32

动态区：
- 波次标题 | 220x48 | 1 行 | 中
- Boss名称 | 150x24 | 1 行 | 左
- 资源值 | 84x30 | 1 行 | 中

九宫：
- top_bar_bg | 80/80/28/28
- resource_group_bg | 32/32/24/24
- player_stats_bg | 24/24/18/18

备注：
- 文本全部独立
- 顶部主条和资源栏不共享定位逻辑
```

## 4. ASCII 草图模板

```text
┌──────────────────────── 顶部安全区 ────────────────────────┐
│                                                            │
│                    [ 顶部主条 / top-center ]               │
│                                              [ 资源栏 ]    │
│                                              [ 统计板 ]    │
└────────────────────────────────────────────────────────────┘
```

## 5. 空白模板

```md
页面：
画布：

布局：
- {模块名} | {位置} | {大概尺寸} | {布局方式}
- {模块名} | {位置} | {大概尺寸} | {布局方式}

动态区：
- {用途} | {大概尺寸} | {最大行数} | {对齐}
- {用途} | {大概尺寸} | {最大行数} | {对齐}

九宫：
- {资源名} | {left/right/top/bottom}
- {资源名} | {left/right/top/bottom}

备注：
- {写 1}
- {写 2}
```

## 6. 给 AI 的直接用法
- 可以把 `布局 + 动态区 + 九宫` 三块直接发给 AI，让它先出：
  `低保真框图`
  `资源拆分建议`
  `第一版出图 prompt`
- 如果 AI 先出的是成品图，也要求它同时回写：
  `模块列表`
  `拉伸底板`
  `文本独立区`

## 7. 最小检查
- 程序能不能一眼看出 `top / bottom / left / right / center`
- AI 能不能一眼看出 `哪些是底板、哪些是图标、哪些是文本`
- 哪些资源需要九宫，是否已经明确
