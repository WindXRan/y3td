# 遗物总表

> 本文用于管理遗物内容池；遗物系统原则以 `docs/design/初步设计/03-局外成长系统.md` 为准。

## 1. 当前定案摘要

- `遗物` 主要通过 `卡池抽取` 获取。
- `遗物` 使用六档品质体系，但强度职责明确分层：
  - `紫色及以下`：只提供数值加成
  - `橙色`：提供轻度机制加成
  - `金色`：提供更强但仍可控的机制加成
- 遗物建议采用 `少量固定槽位`。
- 遗物重复获取后，应转化为长期养成资源，服务升阶、升星或保底。

## 2. 建议记录字段

| 字段 | 含义 | 说明 |
| --- | --- | --- |
| `relic_id` | 遗物 ID | 配置与存档唯一标识 |
| `display_name` | 遗物名 | 玩家可见名称 |
| `quality` | 品质 | 六档品质 |
| `stat_bonus` | 数值加成 | 低稀有遗物主要填写这里 |
| `mechanic_bonus` | 机制加成 | 主要用于橙金遗物 |
| `target_tags` | 适配标签 | 例如 `暴击`、`技能流`、`站场` |
| `obtain_pool` | 获取池 | 常驻池、限定池、活动池等 |
| `duplicate_value` | 重复转化价值 | 用于升星材料或兑换资源 |
| `status` | 当前状态 | `设计中 / 未实现 / 部分实现 / 已实现` |

## 3. 设计边界

- 橙金遗物只能 `强化已有玩法`，不能单件遗物直接创造一套独立职业。
- 低稀有遗物要保证基础成长平滑，不做过度拉开差距的数值墙。
- 高稀有遗物可以成为长期追求点，但不能压过职业和装备系统的存在感。
