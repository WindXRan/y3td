## 已实现

- 章节 / 模式解锁、最近结果回流和最小 `outgame_profile` 已经形成长期进度骨架。
- 当前 `stages`、`stage_modes`、`attack_skills`、`bonds`、`treasures`、`marks` 等静态对象已拆成可扩展目录，具备继续扩内容的工程结构。

## 部分实现

- `1-2`、`1-3` 已有章节占位和独立进度记录，但仍复用 `1-1` 内容。
- 设计中很多扩展系统已经有文档、命名和部分字段预留，例如 `unlock_level_outgame`、章节模式结构、挑战模式骨架，但未形成真实长期循环。
- 当前长期进度更多是“记录已通关什么”，不是“记录长期养成拿到了什么”。

## 未实现

- 局外货币、长期成长、英雄解锁、图鉴、任务、成就、签到。
- 多英雄与多英雄差异化 build 规划落地。
- 章节专属敌人、Boss、奖励和更多地图 / 章节内容。
- 挑战模式长期奖励、解锁链路和更完整的章节 / 模式扩展系统。

## 后续开发建议

1. 在核心局内循环和正式 UI 补齐前，不要过早把长期系统做成重数值养成，优先做“选择宽度”的扩展。
2. 长期系统落地时，优先沿用现有数据驱动目录和存档默认值修正机制，不要为单个功能再开一套孤立存档结构。
3. 扩章节和多英雄时，把“内容存在”与“内容真正可玩”分开记录，避免再次出现章节卡已开但内容仍复用的状态漂移。
