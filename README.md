# Y3TD — AI 驱动的 Y3 游戏全流程开发体系

基于 Claude Code 的 Y3（魔兽争霸3复刻编辑器）游戏全流程 AI 开发工具链，驱动一款 **Hero Auto-Battle Wave Defense Roguelike** 游戏从设计到实现的完整开发。

## 项目概况

| 项目 | 说明 |
|------|------|
| **游戏类型** | 英雄自动战斗 + 波次防守 + Roguelike 构筑 |
| **引擎** | Y3 Editor (魔兽争霸3复刻编辑器) |
| **语言** | Lua 5.4 |
| **项目规模** | 169 个业务 Lua 文件 / 41,572 行业务代码 |
| **开发方式** | Claude Code + .y3maker AI 工具链全流程驱动 |

## 游戏核心系统

```
选择英雄 → 波次推进(5波→Boss) → 波间构筑选择 → 抵挡怪物浪潮
                ↑                    ↑
          自动战斗+技能           羁绊/装备/属性/进化 四选一
```

| 系统层 | 模块 | 说明 |
|--------|------|------|
| **战场层** | 波次推进、Boss 波、挑战试炼、敌人类型、减速区域 | 多波怪物推进，Boss 登场制造高压节点 |
| **构筑层** | 羁绊抽卡(F)、成长武器、属性四选一、猎手专精、攻击技能 | 四层协同构筑，形成独特战斗流派 |
| **英雄层** | 属性系统、等级成长(最高60级)、自动战斗 | 9 维属性 + 自动寻敌 + 技能释放 |
| **局外层** | 关卡选择、英雄图鉴、存档读档、排行榜、宝箱抽奖 | 跨局进度保留与长线养成 |
| **表现层** | 战斗HUD、弹道/粒子特效、音频、消息提示 | 完整视听反馈 |

## .y3maker — AI 开发工具链

项目核心创新在于构建了一套完整的 AI 驱动开发基础设施，让 Claude Code 能够可靠地编写 Y3 游戏代码。

```

│  │ (rules/) │ │(knowledge/)│ │(memory/lua-)│  │
│  └────┬─────┘ └────┬─────┘ └──────┬──────┘  │
│       │             │              │          │

│  │           9 个专用技能 (skills/)         │  │
│  │  y3-lua-pipeline │ y3-ui-pipeline       │  │
│  │  y3-obj-gen      │ y3-auto-test         │  │
```

### 核心痛点解决

Y3 编辑器 Lua API 文档严重缺失，开发者只能通过反编译源码和试错摸索 API 用法。传统开发中查一个 API 要翻几十个演示文件，写错一个方法名就会运行时崩溃且无有效错误提示。

### 知识沉淀层

- **知识库** (knowledge/)：9 大系统、30+ 参考文档，覆盖 UI 系统、核心系统、物编系统
- **错题集** (memory/lua-issues/)：12 条已验证的 API 错误→正确映射，如 `is_playing`→`is_alive`、`revive`→`reborn`
- **会话记忆** (memory/sessions/)：每次 AI 开发会话的目标、完成内容、修复 Bug 归档

### 规则约束层

将 API 安全规则、UI 坐标系规范、MCP 熔断规则等编码为规则文件，Agent 在写代码前强制执行规则读取，从源头消灭 API 臆造。

### 多 Agent 协作

9 个专用技能各司其职，复杂需求由多个 Agent 并行或串行协作：

| 技能 | 职责 |
|------|------|
| `y3-lua-pipeline` | 游戏逻辑编码，强制 API 白名单校验 |
| `y3-ui-pipeline` | UI 交互代码，自动校验 UI 树路径 |
| `y3-obj-gen` | 物编生成（单位/技能/魔法效果/投射物） |
| `y3-auto-test` | MCP 自动化回归测试 |
| `y3-lua-review` | Lua 代码审查 |
| `y3-game-spec` | 从自然语言设计文档生成技术规格 |
| `y3-ui-generator` | UI 模板渲染与动态布局 |
| `y3-env-setup` | 开发环境配置 |
| `y3-gen-terrain-from-image` | 从图片生成地形 |

### MCP 闭环验证

通过自建的 MCP Bridge Server 打通 Claude Code 与 Y3 编辑器：
- **代码热更**：写完代码直接热更到运行中的编辑器
- **截图验证**：自动截图 UI 面板核对渲染结果
- **物编同步**：通过 MCP API 批量创建/修改物编资源
- **自动化测试**：UI 控件定位、点击操作、结果校验

### 开发效率

- API 错误率下降约 **90%**（错题集 + 规则约束消除 API 臆造）
- 开发效率提升约 **3-5 倍**（知识库即时检索 + Agent 自动编码 + 热更验证）
- 从零到 41,572 行业务代码，覆盖 20+ 核心系统，实现于 50 次 git 提交中

## 项目结构

```
maps/EntryMap/
├── script/
│   ├── main.lua                  # 游戏启动入口
│   ├── config/                   # 配置汇总
│   ├── core/                     # 核心框架(DI/事件/配置加载)
│   ├── data/tables/              # 数据表(技能/羁绊/英雄/商店/存档)
│   ├── runtime/                  # 运行时系统(70个模块)
│   │   ├── boot*.lua             # 启动与总协调
│   │   ├── battlefield/          # 战场与刷怪
│   │   ├── skill_system/         # 技能框架(激活/修改器)
│   │   ├── bonds_chain.lua       # 羁绊链系统
│   │   ├── attack_skills.lua     # 攻击技能运行时
│   │   ├── progression.lua       # 英雄成长
│   │   └── rewards.lua           # 奖励队列与英雄进化
│   ├── ui/                       # UI系统(HUD/局外/结果面板)
│   ├── tools/                    # 开发工具与冒烟测试
│   └── tests/                    # 测试用例
├── .y3maker/                     # AI开发工具链
│   ├── rules/                    # 规则约束(.mdc)
│   ├── knowledge/                # Y3引擎知识库
│   ├── memory/                   # 错题集与会话记忆
│   └── skills/                   # 9个专用AI技能
├── ui/                           # UI JSON布局文件
├── unit/                         # 单位物编资源
├── ability/                      # 技能物编资源
├── item/                         # 物品物编资源
├── modifier/                     # 魔法效果物编资源
└── projectile/                   # 投射物物编资源
```

## 快速开始

1. 用 Y3 Editor 打开 `maps/EntryMap/` 目录
2. 游戏入口：`maps/EntryMap/script/main.lua`
3. AI 开发规则见 `maps/EntryMap/script/CLAUDE.md`
4. 开发前必读：`.y3maker/memory/lua-issues/api_issues.md`（API 错题集）
