# 项目概览

## 1. 项目定位

这是一个基于 Y3 编辑器的塔防地图工程。仓库同时包含以下几类内容：

- 地图本体与地图级脚本：`maps/EntryMap`
- 项目级全局脚本保留位：`global_script`
- 项目级全局触发器保留位：根目录 `global_trigger`
- 自定义资源仓库：`custom`
- 编辑器插件与对象库：`plugins`
- 项目级规则与元配置：根目录各类 `*.json`

当前实际会驱动游戏运行的核心逻辑，集中在 `maps/EntryMap/script`。

## 2. 顶层目录职责

### `maps/EntryMap`

地图主目录，包含一张地图运行所需的主要内容：

- `script`：Lua 业务脚本，当前游戏主链路在这里
- `ability`、`unit`、`item`、`modifier`、`projectile`：地图物编对象数据
- `ui`：地图 UI 面板与预制界面
- `editor_table`：编辑器导出的对象总表/索引表
- `global_trigger`：地图内可视化触发器资产，本项目中主要是 UI 相关触发器
- `editor`：编辑器分组、树结构、文件夹信息等元数据
- `custom_eca`：地图级自定义 ECA 扩展位，当前几乎为空

### `global_script`

项目级全局脚本目录。当前仓库中仅看到 `global_main.lua` 的占位内容，没有接入当前地图的主运行链路。

### 根目录 `global_trigger`

项目级全局触发器目录。当前 `function/index.txt` 与 `trigger/index.txt` 都是空对象，说明这里暂未承担业务逻辑。

### `custom`

自定义资源与导入仓库：

- `CustomImportRepo.local/resource.repository`：资源仓库索引
- `UIScript`、`Fonts`、`OriginalRes` 等：UI、字体、原始资源、音频、模型等

这部分主要为地图提供素材，不直接承载业务状态机。

### `plugins`

编辑器插件与对象库，包含单位、投射物、修饰器、音效等基础资源来源。更多是资源库/编辑器层依赖，而不是运行时主逻辑。

### 根目录各类 `*.json`

包括 `match.json`、`ruledata.json`、`setting.json`、`game_shortcut.json` 等项目级配置。它们影响编辑器或地图工程行为，但当前核心玩法逻辑仍由 `maps/EntryMap/script` 主导。

## 3. 当前真实运行主线

当前实际运行主线如下：

```text
maps/EntryMap/script/main.lua
  -> require 'entry_runtime'
  -> entry_runtime.bootstrap()
  -> reset_state()
  -> create_hero()
  -> register_runtime_events()
  -> start_runtime_loops()
  -> start_wave(1)
```

这条链路是当前游戏逻辑的第一入口。

## 4. 需要特别区分的“保留模块”

仓库里有几处容易误判为主入口的内容，需要明确区分：

- `maps/EntryMap/script/可重载的代码.lua`
  这是热重载示例文件，通过 `include` 引入，但当前只注册了一个 `R` 键打印示例。
- `global_script/global_main.lua`
  当前内容是 `--Hello, world!`，没有接入地图主循环。
- 根目录 `global_trigger`
  当前是空索引。
- `maps/EntryMap/script/design`
  这是策划/设计文档，不是运行时代码来源。

## 5. 建议的阅读顺序

如果你要从“先理解项目，再读代码”的角度进入本地图，推荐顺序是：

1. 先看 `maps/EntryMap/script/main.lua`
2. 再看 `maps/EntryMap/script/entry_runtime.lua`
3. 再看 `maps/EntryMap/script/entry_config.lua`
4. 再看 `maps/EntryMap/script/runtime_bonds.lua`
5. 最后回头看 `maps/EntryMap/ui`、`maps/EntryMap/global_trigger`、`custom`

这样可以先掌握“谁在驱动游戏”，再回头理解“运行时依赖了哪些数据和资产”。
