# GM面板与开发命令

## 1. 调试系统入口

当前调试能力主要来自两部分：

- `require 'y3.develop.command'`
- `entry_runtime.lua` 中的 GM 面板与快捷键

这套系统明显是为了单机压测、流程验收和坐标校准服务的。

## 2. 开发命令

`register_dev_commands()` 会注册一批开发命令。根据当前代码与提示文案，重点包括：

- `.epos`
- `.eset hero`
- `.eset defense`
- `.earea <area_id> <width> <height>`
- `.edump`

这些命令主要用于：

- 查看当前位置
- 重设英雄点与防线点
- 调整区域中心与尺寸
- 导出当前校准结果

它们直接服务 `entry_config.lua` 中点位与区域的迭代调整。

## 3. GM 面板挂载方式

GM 面板不是静态 UI 文件直接渲染出来的，而是由 `entry_runtime.lua` 动态创建：

- 先获取 `GameHUD`
- 创建右上角切换按钮
- 创建面板背景、标题、状态文本
- 创建一组调试按钮

因此它是“代码生成型调试界面”。

## 4. 当前 GM 面板支持的功能

从按钮定义可以看到，当前面板支持：

- 帮助 / F1
- 加资源 / F2
- 升 3 级 / F3
- 解锁技能 / F4
- 开强化 / F5
- 抽羁绊 / F6
- 满挑战 / F7
- 刷 Boss / F8
- 清全场 / F9
- 打印状态

这些按钮基本覆盖了成长、资源、波次、挑战、清场几个核心验证方向。

## 5. 键盘调试热键

在调试模式下，系统还注册了一组 `Ctrl + Fx` 热键。

其作用与面板按钮大体一致，例如：

- `Ctrl + F1`：帮助
- `Ctrl + F2`：加资源
- `Ctrl + F3`：升 3 级
- `Ctrl + F4`：解锁全部攻击技能
- `Ctrl + F5`：打开强化面板
- `Ctrl + F6`：触发羁绊抽卡
- `Ctrl + F7`：补满挑战次数
- `Ctrl + F8`：强制刷 Boss
- `Ctrl + F9`：清空场上敌人
- `Ctrl + F10`：展开/折叠 GM 面板

## 6. 调试系统服务哪些业务模块

当前调试系统覆盖面很广，几乎直连所有核心模块：

- 波次系统
- 挑战系统
- 升级三选一
- 羁绊抽卡
- 资源系统
- 坐标与区域配置

这说明该项目已经从“仅能靠正常游玩验证”进化到了“能用 GM 工具快速压测核心流程”的阶段。

## 7. 为什么调试系统很重要

对这种有波次、成长、羁绊、挑战并行的塔防地图来说，如果没有快速调试能力，验证成本会非常高：

- 升级要反复打怪
- 抽卡要反复攒木材
- 挑战要等次数恢复
- 波次要等到指定时间

当前这套 GM 面板与开发命令，本质上是为开发效率服务的“内建测试工具”。
