# 地图级触发器说明

## 1. 目录位置

地图级触发器位于：

- `maps/EntryMap/global_trigger`

这和根目录的 `global_trigger` 不是一回事。

## 2. 当前目录结构

地图级触发器当前至少包含两类内容：

- `function`
- `trigger`

其中又包含一个明显的 `ui` / `UI` 分类。

## 3. 当前已存在的 UI 触发器资产

从索引文件看，地图内 `global_trigger` 当前主要围绕 UI 构建了一套触发器资产，例如：

### function/ui

- `Select Unit`
- `Select Unit Group`
- `Select Item`
- `Select Destructible`
- `Set Primary Unit`
- `Bind Unit's Ability Button`
- `Bind Unit's Inventory`
- `Mover Tooltip to Mouse Coordinates`
- `Refresh Tooltip`
- `Open Shop`
- `Refresh Shop Tab`
- `Refresh Titles of Shop Tabs`
- `Display Notice`
- `Refresh Unit Attribute`

### trigger/UI

- `Init Data`
- `Init Main UI Event Handler`
- `Init UIEvent`
- `Init Update`

这些名字已经能说明：地图内触发器更偏“UI 框架层”和“界面交互层”。

## 4. 与 Lua 主逻辑的关系

当前 Lua 主逻辑主线是：

- `main.lua`
- `entry_runtime.lua`
- `runtime_bonds.lua`

地图级 `global_trigger` 并不是这条主线的第一入口，但它与 Lua 主逻辑是并行存在的：

- Lua 管主循环、战斗、成长、挑战、调试
- 地图级触发器更像 UI 与编辑器可视化逻辑资产

因此它们不是“互相替代”，而是“不同层次共同构成地图”。

## 5. 为什么要单独写这一章

如果不把地图级触发器单独拎出来，读项目时很容易出现两种误判：

- 误以为全项目只有 Lua，没有可视化触发器
- 误以为 `global_trigger` 都是空的

实际上当前事实是：

- 根目录 `global_trigger` 为空
- 地图内 `maps/EntryMap/global_trigger` 不为空，而且内容不少

## 6. 当前应如何理解它的定位

最稳妥的理解方式是：

- 它是地图级 UI 触发器资产层
- 不是当前业务主循环的第一入口
- 但很可能仍影响地图内现有 UI 体验和编辑器搭建流程

因此做项目盘点时，不能忽略它，但也不应把它写成“当前游戏主线脚本入口”。
