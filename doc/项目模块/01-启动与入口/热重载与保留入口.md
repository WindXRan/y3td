# 热重载与保留入口

## 1. 热重载入口是什么

`maps/EntryMap/script/main.lua` 在底部执行了：

```lua
include '可重载的代码'
```

这里使用的是 `include`，不是 `require`。在 Y3 体系里：

- `require`：普通模块加载，默认不参与热重载
- `include`：适合需要热重载的逻辑

所以 `可重载的代码.lua` 是一个“可在开发期快速替换”的模块挂载点。

## 2. `可重载的代码.lua` 当前做了什么

当前文件只注册了一个键盘事件：

- 监听 `R` 键
- 打印热重载提示
- 告诉开发者可以用 `.rd`、VSCode 重载按钮、或 `y3.reload.reload()`

因此它目前的定位是：

- 热重载演示文件
- 开发流程教学文件
- 不是核心玩法代码

## 3. 与主流程中 `R` 热键的关系

这里要注意一个容易混淆的点：

- `可重载的代码.lua` 中也监听了 `R`
- `entry_runtime.lua` 中也把 `R` 绑定为“宝物挑战”

这意味着当前仓库里，`R` 键存在两套注册来源：

- 一套用于热重载示例提示
- 一套用于主玩法挑战入口

从项目语义上看，当前真正的玩法意义以 `entry_runtime.lua` 的挑战入口为主；`可重载的代码.lua` 更像开发示例，不应被当成主玩法控制层。

## 4. 当前保留但未接入主链路的入口

### `global_script/global_main.lua`

当前文件内容只有：

```lua
--Hello, world!
```

它没有被 `main.lua`、`entry_runtime.lua` 或地图配置调用，因此目前只是项目级保留脚本位。

### 根目录 `global_trigger`

根目录 `global_trigger/function/index.txt` 与 `global_trigger/trigger/index.txt` 都是空对象：

- 没有项目级函数触发器
- 没有项目级事件触发器

这说明根级触发器目前不参与当前地图业务。

## 5. 不能和“地图内触发器”混淆

虽然根目录 `global_trigger` 目前为空，但 `maps/EntryMap/global_trigger` 不是空的。

地图内 `global_trigger` 当前包含大量 UI 相关触发器资产，例如：

- `Init Data`
- `Init Main UI Event Handler`
- `Init UIEvent`
- `Init Update`
- `Open Shop`
- `Refresh Tooltip`
- `Refresh Unit Attribute`

因此要区分两层概念：

- 根目录 `global_trigger`：项目级保留位，当前空
- `maps/EntryMap/global_trigger`：地图级触发器资产，当前非空

## 6. 热重载和保留入口的正确理解

当前项目的入口优先级应理解为：

```text
主入口：
main.lua -> entry_runtime.lua

开发辅助入口：
可重载的代码.lua

当前未承载主流程的保留入口：
global_script/global_main.lua
根目录 global_trigger
```

这样理解后，阅读项目时就不会把“开发示例”和“业务主链路”混在一起。
