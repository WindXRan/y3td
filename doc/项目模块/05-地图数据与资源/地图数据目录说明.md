# 地图数据目录说明

## 1. 这部分文档看什么

本章讲的是 `maps/EntryMap` 下“不是 Lua 代码，但会影响运行时表现”的数据目录。重点是理解职责边界，不逐个枚举数百个物编文件。

## 2. 主要数据目录

### `ability`

地图内技能物编数据目录。当前数量很多，用于承载地图中定义过的技能对象。

对 Lua 来说，这一层最常见的作用是：

- 作为技能、普攻、效果、弹道绑定的资源来源
- 被单位对象或运行时能力对象引用

### `unit`

地图单位物编数据目录。

当前 `entry_config.lua` 中大量 `unit_id` 都指向这里定义的对象，例如：

- 英雄单位
- 每波主怪
- 每波 Boss
- 挑战用怪

### `item`

地图物品物编数据目录。当前主线 Lua 逻辑没有大量直接使用，但它仍属于地图资产的一部分。

### `modifier`

地图修饰器/Buff 相关物编数据目录。虽然当前核心战斗效果很多由 Lua 直接控制，但底层表现或其他系统仍可能依赖这些对象。

### `projectile`

地图投射物数据目录。攻击技能系统中的部分表现层，会依赖投射物物编 ID。

### `ui`

地图 UI 面板目录，承载：

- `GameHUD`
- `LoadingPanel`
- `LogoPanel`
- `CommonTip`
- 胜负界面等

它是地图静态 UI 的主要来源。

### `editor_table`

编辑器导出的总表/索引资产。它本质上更偏“对象总览和编辑器表层输出”，不是直接手写维护的业务模块。

### `editor`

编辑器分组树、文件夹组织、资源目录结构等元数据，用于描述编辑器里的组织方式。

### `global_trigger`

地图内可视化触发器资产目录。当前主要集中在 UI 相关逻辑。

### `custom_eca`

地图级自定义 ECA 扩展位。当前内容非常少，基本可视为空位。

## 3. 关键配置文件

### `logicres.json`

记录地图逻辑资源集合，例如初始镜头等逻辑资源信息。

### `userresource.json`

记录地图自定义编辑器资源集合。当前内容较轻，更多像资源登记表。

### `desc.json`

记录地图描述性配置，例如地图名、是否懒加载触发器等。

### `userdata.json`

记录一些地图级功能开关与行为设置，例如是否拆分保存触发器、输入同步、布局配置等。

## 4. 运行时代码如何依赖这些数据

当前 Lua 逻辑并不会手动去遍历这些目录，而是通过“引用 ID 或名称”的方式间接依赖：

- `entry_config.lua` 提供单位物编 ID、区域、挑战配置
- `entry_runtime.lua` 通过这些 ID 创建单位、播放弹道、挂 UI
- 引擎从对应目录中找到实际对象定义

所以可以把这层理解为：

- Lua 负责“调用”
- 物编目录负责“提供对象实体”

## 5. 需要避免的误解

不要把以下两件事混为一谈：

- 目录中文件多，不代表都在当前主流程里被显式使用
- Lua 没直接读取 json，不代表这些资产对运行时没影响

当前项目里，大量地图数据是通过“物编 ID”间接参与运行时的。

## 6. 当前最重要的地图数据依赖

从当前真实主链路看，最关键的数据依赖是：

- `unit`：英雄、主怪、Boss、挑战怪
- `projectile`：攻击技能弹道
- `ui`：HUD 与界面预制
- `logicres.json`：逻辑资源
- `custom` 资源仓库：UI 图片等素材来源

这些是文档和排错时最值得优先关注的数据层。
