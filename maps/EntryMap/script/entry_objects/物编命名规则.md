# 物编命名规则（当前项目）

本文用于统一 `maps/EntryMap/editor_table` 的物编命名口径，避免“目录名 / 逻辑名 / 显示名”混用。

## 1. 命名分层

- 引擎目录名（不可改）：
  - `editorunit`、`abilityall`、`modifierall`、`projectileall`、`editoritem`
- 逻辑类型名（代码/文档统一使用）：
  - `unit`、`ability`、`modifier`、`projectile`、`item`
- 文件名（物编 JSON）：
  - 固定使用纯数字 ID，例如 `100001.json`
- 显示名（编辑器看到的名称）：
  - 通过 `name` TID 映射到 `zhlanguage.json`

## 2. 目录名与逻辑名映射（唯一口径）

- `editorunit` -> `unit`
- `abilityall` -> `ability`
- `modifierall` -> `modifier`
- `projectileall` -> `projectile`
- `editoritem` -> `item`

禁止在同一文档/代码中交替使用“技能/abilityall/ability”而不声明含义。推荐优先写逻辑名，首次出现时附目录名。

## 3. ID 与文件命名规则

- 文件名必须等于 `key`，且 `uid` 必须是同值字符串。
- 示例：
  - `editorunit/100001.json` -> `key = 100001`，`uid = "100001"`
- 不使用中文文件名、不使用语义化文件名（如 `火球术.json`）。

## 4. 显示名（name/description）规则

- `name`、`description` 一律存 TID（整数），不直接存中文文本。
- 中文文案只维护在 `zhlanguage.json`。
- 临时占位名必须带前缀 `runtime_placeholder_`，上线前清理。

## 5. 推荐语义前缀（用于显示名，不用于文件名）

- 单位：`UNIT_`
- 技能：`ABILITY_`
- Buff：`MOD_`
- 投射物：`PROJ_`
- 物品：`ITEM_`

示例：`ABILITY_chain_lightning`、`PROJ_chain_lightning_main`

## 6. 代码常量命名建议（Lua）

- 单位 ID 常量：`UNIT_ID_xxx`
- 技能 ID 常量：`ABILITY_ID_xxx`
- Buff ID 常量：`MOD_ID_xxx`
- 投射物 ID 常量：`PROJ_ID_xxx`
- 物品 ID 常量：`ITEM_ID_xxx`

示例：

```lua
local ABILITY_ID_CHAIN_LIGHTNING = 201390009
local PROJ_ID_CHAIN_LIGHTNING = 201364746
```

## 7. 迁移与兼容策略

- 历史目录名不做重命名（引擎约束）。
- 新增文档、脚本、注释统一使用“逻辑名 + 首次附目录名”写法。
- 发现旧数据存在“纯数字回显名”“placeholder 文案”时，按批次修复 `zhlanguage.json` 与对应 TID。
