# 运行时与代码结构

## 代码主链

- 启动入口：`script/main.lua`
- 总协调器：`script/entry_runtime.lua`
- 局外：`script/entry_runtime_outgame.lua`
- 运行时子模块：`script/runtime/*.lua`
- 静态对象：`script/entry_objects/*`

## runtime 关键模块

| 文件 | 职责 |
| --- | --- |
| `runtime/boot.lua` | 子系统装配与战斗入口协调 |
| `runtime/battlefield.lua` | 波次、挑战、战场、胜负 |
| `runtime/progression.lua` | 经验与等级成长 |
| `runtime/attack_skills.lua` | 攻击技能施法与命中处理 |
| `runtime/bonds.lua` | `F` 羁绊候选、结算、奖励修正 |
| `runtime/bonds_chain.lua` | 链式羁绊与进化逻辑 |
| `runtime/rewards.lua` | 宝物、烙印、奖励队列 |

## 运行时状态

- 总状态在 `entry_runtime.lua` 的 `STATE`
- 高价值轮次互斥由总协调层管理
- UI 只读状态，不保存第二份真状态

## 目录边界

- `entry_objects`：稳定入口与对象装配层
- `data_csv`：静态事实数据
- `data/tables`：CSV 到 Lua 的装配层
- `runtime`：行为与结算
- `ui`：表现层与布局

## CSV 校验入口

- 统一批量入口：`script/tools/verify_csv_object_tables.py`
- 当前已接入的一致性烟测：
  - 宝物：`test_treasure_catalog_consistency_smoke.lua`
  - 烙印：`test_marks_catalog_consistency_smoke.lua`
  - 章节与模式：`test_stages_catalog_consistency_smoke.lua`
  - 波次与挑战：`test_waves_challenges_catalog_consistency_smoke.lua`
  - 羁绊：`test_bond_catalog_consistency_smoke.lua`
- 目标不是替代全部运行时测试，而是尽早拦截 CSV 关系错误、顺序错误和孤儿引用

## 当前结构结论

- 主运行时已经不再是“一切都塞进总文件”
- 文档里旧的 `entry_runtime_*` 路径多数已经过时
- 后续开发统一按 `runtime/*.lua` 理解子模块分工
