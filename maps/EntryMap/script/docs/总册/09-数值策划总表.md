# 数值策划总表

> 本文是当前版本的数值整理总表，用于把运行中的事实源、配表入口和消费代码收敛到一个入口。
> 不替代原有专题策划文档；原始设计与历史方案继续保留在 `script/docs/总册/*` 与 `script/docs/归档/*`。

## 文档目标

- 把当前项目里和数值策划直接相关的 `CSV / object table / runtime 消费点 / 旧策划文档` 收敛到一个入口。
- 优先回答两个问题：
  - 现在有哪些数值系统正在生效？
  - 想改某一类数值，应该先看哪张表、哪层装配、哪段运行时代码？

## 事实源优先级

1. `script/data_csv/*`
2. `script/data/object_tables/*`
3. `script/config/entry_config.lua`
4. `script/runtime/*`
5. `script/docs/总册/*`
6. `script/docs/归档/*`

## 2. 英雄属性、初始化与等级成长

### 2.1 事实源

- `script/data_csv/hero_attr_config.csv`
- `script/data_csv/hero_init_stats.csv`
- `script/data_csv/hero_level_progression.csv`
- `script/runtime/hero_attr_defs.lua`
- `script/runtime/hero_attr_system.lua`

### 2.2 当前初始化属性

`hero_init_stats.csv` 当前直接给了 `7` 条开局属性：

- `攻击白字 = 46`
- `攻击速度 = 95`
- `物理吸血 = 5`
- `物理暴伤 = 25`
- `物理暴击 = 10`
- `生命白字 = 900`
- `攻击范围 = 1400`

### 2.3 当前等级成长

- 总等级数：`60`
- 表头：`level / order_index / exp_to_next / all_attr_bonus / all_element_damage_bonus`
- 示例：
  - `1级 -> 升级经验 350`
  - `2级 -> 全属性加成 1`
  - `5级 -> 全属性加成 9，元素伤害加成 0.1`

### 2.4 属性定义层职责

`hero_attr_defs.lua` 负责：

- 定义属性分类：`伤害 / 防守 / 资源 / 增幅 / 其他`
- 定义属性显示顺序、格式、是否百分比、是否持久化
- 维护属性别名映射，例如：
  - `物理攻击 -> 攻击`
  - `最大生命 -> 生命`
  - `物系伤害 -> 金行伤害`
  - `BOSS伤害 -> 挑战伤害`

### 2.5 系统职责划分

- `hero_attr_config.csv`
  - 负责面板默认值、调试附加值等静态配置。
- `hero_init_stats.csv`
  - 负责初始开局属性包。
- `hero_level_progression.csv`
  - 负责等级曲线。
- `hero_attr_system.lua`
  - 负责规范属性名、保存 KV、计算白字/绿字/结算值、同步引擎属性。

## 10. 当前维护建议

- 想做“纯数值调整”，优先改 `data_csv/*`，尽量不要先改 runtime 常量。
- 想给多个系统都能消费的奖励效果，优先考虑落到 `attreffect.csv`，不要在多个 runtime 文件里重复硬编码。
- 想查“某个系统到底是不是已经数据化”，先看对应的 `object_tables/*.lua` 是否已经把 CSV 装配成稳定结构。
- 想做大改版前，先把本文件里的对应模块和旧策划文档一起过一遍，再决定是改事实源还是改运行时口径。
