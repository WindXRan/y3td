# UI设计与资产规范

## 当前 UI 现状

- 已有战斗 HUD
- 已有局外选关与模式选择
- 高价值选择仍大量依赖文本消息与 HUD 按钮
- 正式中央决策面板尚未完全取代原型交互

## UI 主体分层

- 常驻 HUD
- 底部角色操作区
- 三选一决策面板
- 背包与奖励记录
- 结算页

## 当前已确定规范

- UI 资源索引走 `script/ui/res.lua`
- 运行时 UI 主要由 Lua 动态创建或控制
- 业务状态不在 UI 层重复保存

## 当前高优先级 UI 缺口

- 正式三选一面板
- 背包页
- 奖励记录
- 正式结算页
- 更完整的局外长期成长界面

## 文档压缩后的维护规则

- UI 规范、模板、示例统一并入本册
- 不再拆出大量 HUD/面板/YAML 样例文件作为日常入口
- 需要保留的模板与样例，只在旧目录中归档
