# UI设计与资产规范

## 当前视觉方向

局内 UI 统一采用“暗色金属框游戏 HUD”风格：厚外框、强模块分区、高饱和功能按钮、暗纹空槽、蓝色标题牌和金色奖励强调。

详细规范见：

- [14-局内UI视觉风格规范.md](/C:/Y3TD/Y3GPT/y3td/maps/EntryMap/script/docs/总册/14-局内UI视觉风格规范.md)

## 当前 UI 现状

- 已有战斗 HUD
- 已有局外选关与模式选择
- 高价值选择仍大量依赖文本消息与 HUD 按钮
- 正式中央决策面板尚未完全取代原型交互

## UI 主体分层

- 常驻 HUD
- 底部角色操作区
- 三选一决策面板
- 背包与奖励记录
- 结算页

## 当前已确定规范

- UI 资源索引走 `script/ui/res.lua`
- 运行时 UI 主要由 Lua 动态创建或控制
- 业务状态不在 UI 层重复保存
- 局内 HUD 不再使用大面积纯黑调试面板风格
- 底栏改造必须保留既有运行时节点名，避免破坏 Lua 绑定

## 当前高优先级 UI 缺口

- BattleBottomHUD 按暗色金属框风格重包装
- 正式三选一 / 二选一面板视觉统一
- 背包页
- 奖励记录
- 正式结算页
- 更完整的局外长期成长界面

## 文档压缩后的维护规则

- UI 总入口仍为本册
- 局内 HUD 视觉细则以 `14-局内UI视觉风格规范.md` 为准
- 不再拆出大量 HUD/面板/YAML 样例文件作为日常入口；确需沉淀的风格规范进入总册
- 需要保留的模板与样例，只在旧目录中归档
