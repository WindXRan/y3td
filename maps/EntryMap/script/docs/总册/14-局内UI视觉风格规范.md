# 局内UI视觉风格规范

> 本文用于统一后续局内 HUD、底栏、弹窗、选择面板的视觉方向。当前目标不是继续做“黑底上摆控件”，而是统一改成参考图式的暗色金属框游戏 HUD：强外轮廓、强模块分区、高饱和功能图标、明确可点击状态。

## 风格定位

- 关键词：暗色金属、厚边框、斜角、压纹槽位、蓝色标题牌、金色奖励感、绿色成长反馈。
- 观感目标：像完整游戏 HUD，而不是调试面板。
- 信息目标：玩家一眼能分清“资源 / 角色 / 被动技能 / 物品 / 卡牌按钮 / 提示面板”。
- 交互目标：可点击控件必须比展示信息更亮、更厚、更有按钮态。

## 参考图可复用结论

参考图好看的原因不是单个按钮，而是整体结构稳定：

- 顶部和底部都有连续外壳，边缘是厚金属框。
- 每个功能模块都有独立框体，框体之间用竖向隔板分开。
- 重要按钮使用高饱和蓝、绿、橙、金，普通信息留在深灰底上。
- 空槽不是纯黑，而是暗纹、凹槽、压印图案。
- 标题条清楚，例如“物品栏”“快捷键 C”“章节 / 波次 / Boss”。
- 图标尺寸大，文字短，按钮不依赖长文案撑信息。

## 总体布局规范

局内 HUD 分三层：

1. 顶部状态栏
   - 放资源、章节、波次、计时、排行榜、快捷入口。
   - 使用窄高条，黑蓝金属底，资源数字使用高亮色。

2. 底部主操作栏
   - 放小地图、属性、英雄、被动技能、经验、物品、卡牌功能。
   - 必须是连续底座，但内部模块要有明确隔板。

3. 右侧任务/提示区
   - 放主线任务、自动挑战、活动提示。
   - 使用独立悬浮面板，不要和底栏混成一块。

## 底部主操作栏规范

底栏必须按以下模块从左到右排列：

| 模块 | 内容 | 视觉要求 |
| --- | --- | --- |
| 小地图 | 小地图、辅助按钮 | 大地图框，金属边，地图不可贴边 |
| 属性 | 战力、攻击、护甲、力量、智力、敏捷 | 每行一个圆形/盾形图标，数值黄白，增幅绿 |
| 英雄 | 模型/头像、名字、生命条 | 英雄框要最亮，生命条必须有清晰外框 |
| 被动技能 | 5 个被动图标 | 只显示图标，不显示热键、冷却、技能名 |
| 经验/进化 | 等级、经验条、进化入口 | 经验条常驻；待进化时显示金色按钮态 |
| 物品栏 | 装备/物品格、消耗品 | 空槽用暗纹压印，已装备槽高亮 |
| 卡牌功能 | 抽卡、卡册、杀敌抽奖、摆烂钓鱼、卡槽 | 功能按钮用大图标+短标题，避免长按钮文字 |

### 底栏尺寸

- 底栏整体高度控制在 240-280 像素。
- 窗口模式必须留安全边距，不允许被 Windows 任务栏裁切。
- 不要把所有控件压到最底部，底栏底部至少留 10-18 像素内边距。
- 各模块之间必须有 8-16 像素视觉隔板。

### 底栏外壳

必须有三层：

1. 最外层黑色/深蓝金属底，透明度接近不透明。
2. 内层轻微蓝灰底纹，避免纯黑。
3. 顶部高亮线和底部暗线，形成厚度。

禁止：

- 大面积纯黑矩形。
- 只靠 1px 线条分区。
- 模块没有标题、没有边框、没有压纹。

## 色彩规范

| 用途 | 推荐色 |
| --- | --- |
| 主背景 | `#0b0d14`, `#111722`, `#1a202d` |
| 金属边框 | `#2b3342`, `#465064`, `#6a7488` |
| 蓝色标题牌 | `#1f5f9e`, `#2d8bff`, `#69b8ff` |
| 金色强调 | `#ffcc45`, `#f6a622`, `#8c5a19` |
| 绿色收益 | `#30f06f`, `#6dff91` |
| 红色危险 | `#ff3b25`, `#d5201b` |
| 普通文字 | `#f2f5ff` |
| 次级文字 | `#aeb8c8` |

要求：

- 功能按钮优先使用蓝、绿、橙、金，不要全用黑灰。
- 增益数字使用绿色，战斗警告使用红色。
- 金色只用于奖励、进化、重要按钮，不要铺满全 UI。

## 字体与文字

- 标题：14-18 号，白字或黄字，有深色底牌。
- 数值：12-16 号，数字优先黄/白，增幅绿。
- 按钮文字：尽量 2-4 个字，例如“抽卡”“卡册”“杀敌”“钓鱼”。
- 长说明进入 tooltip，不放在底栏按钮里。
- 被动技能槽不显示热键、冷却、技能名。

禁止：

- 按钮里塞“杀敌抽奖”“摆烂钓鱼”这类长文案导致挤压。
- 用纯文本解释按钮功能。
- 字号过小且颜色接近背景。

## 槽位规范

所有槽位分三种状态：

1. 空槽
   - 暗纹底、压印图案、低亮边框。
   - 不能是纯黑方块。

2. 已装配
   - 有图标、亮边框、轻微内发光。
   - 稀有度可以用边框颜色区分。

3. 可交互/可领取
   - 金色或蓝色外发光。
   - 鼠标悬浮显示 tooltip。

被动技能槽：

- 固定 5 个。
- 只显示被动图标。
- 不显示数字热键。
- 不显示冷却。
- 不显示“未装配”等文字。

物品/卡牌槽：

- 空槽要使用统一暗纹框。
- 卡牌槽至少 2 行，间距一致。
- 卡册/抽卡按钮和卡牌槽必须视觉分离。

## 按钮规范

按钮必须有四态：

| 状态 | 视觉 |
| --- | --- |
| 普通 | 高饱和底图，清晰边框 |
| 悬浮 | 亮度提升，边缘发光 |
| 按下 | 轻微变暗/内陷 |
| 禁用 | 降饱和、低透明度 |

底栏按钮优先做成“图标按钮 + 短标签”：

- 抽卡：卡牌/印箓图标 + “抽卡”
- 卡册：书页/卷轴图标 + “卡册”
- 杀敌抽奖：奖章/宝箱图标 + “杀敌”
- 摆烂钓鱼：鱼/钓竿图标 + “钓鱼”
- 进化：金色按钮态 + “进化”

长说明放到 hover tooltip：

- 抽卡：按 F 抽取战术卡卡牌。
- 卡册：查看已经收录的卡牌与战术卡进度。
- 杀敌：消耗击杀奖励，抽取一次奖励。
- 钓鱼：进入摆烂钓鱼玩法。

## Tooltip 规范

Tooltip 用于承载长说明，底栏不直接写说明。

结构：

- 顶部：图标 + 名称 + 品质/类型
- 中部：当前效果
- 底部：操作提示或消耗

视觉：

- 深蓝黑底。
- 顶部有蓝色或金色标题条。
- 正文用白字，数值用绿/黄。
- 宽度固定，避免随文字跳动。

## 选择面板规范

三选一、二选一、四选一面板使用同一套卡牌风格：

- 背景遮罩不能太黑，保留战场环境。
- 卡牌有品质边框。
- 卡牌标题短而大，效果描述分块。
- 选择按钮必须在卡牌底部，颜色清楚。
- 可刷新按钮放面板下方，不和卡牌混在一起。

进化二选一：

- 使用 `BondChoice2` 面板。
- 标题显示“专精进阶 / 10级进化 / 20级进化”等。
- 两张卡必须居中，不允许只发消息不弹面板。

## 当前 BattleBottomHUD 改造要求

下一轮底栏改造按以下优先级执行：

1. 重做底栏外壳：厚金属边、斜角、顶部高亮线、底部阴影。
2. 重做模块框：每块加标题牌和竖向隔板。
3. 重做空槽：所有空槽换暗纹压印，不再纯黑。

## 实现约束

- UI JSON 变更后必须重新生成 `ui_tree`。
- Lua 只绑定节点树中确认存在的路径。
- 视觉重做优先改生成脚本，不要手改生成后的 JSON。
- 重做过程中保留节点名，避免破坏运行时绑定。
- 每轮改完至少跑底栏 UI 结构测试和运行时静态测试。

## 验收标准

一版 UI 改造完成后，用以下标准判断是否合格：

- 3 秒内能看懂每个模块用途。
- 可点击按钮比展示信息更醒目。
- 空槽不是纯黑方块。
- 底栏窗口模式下不被裁切。
- 被动技能槽没有热键、冷却、技能名。
- 进化待选时点击底栏能弹出二选一面板。
- 右侧按钮没有文字挤压。
- 整体观感接近参考图的暗色金属游戏 HUD，而不是调试面板。
