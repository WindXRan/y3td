### 1.2 宝物系统已下线

- `runtime/rewards.lua`：`TREASURE_SYSTEM_ENABLED = false`
- `runtime/boot.lua` 多处输出"宝物功能已下线"提示
- 运行时不再发放/生效宝物，但仍保留数据结构接口兼容
- 原"宝物与烙印奖励"应修正为"烙印（进化/专精）奖励"

### 1.3 未完成

| 模块 | 说明 |
|------|------|
| 正式结算页 UI | 战斗结束后只有 `STATE.last_battle_result` 和 console 消息，无专门结算面板 |
| 正式背包 UI | 无专门背包面板入口 |
| 局外长期成长写回 | 挑战奖励局外写回链路未完全落地 |
| 多英雄与章节专属内容 | 仍在设计阶段 |

### 1.4 技术债与风险

- `runtime/boot.lua` 体量较大（3000+ 行），耦合面广，继续叠加功能会提高回归成本
- UI 根节点显隐由运行时强控（`enforce_runtime_ui_phase`），新增面板时容易遗漏同步
- 文档已迁移到 `docs/总册`，旧目录仅作归档查阅

## 2. 当前推荐开发顺序

### P0：正式结算页 UI
- 战斗结束后弹出结算面板（胜利/失败）
- 面板展示当局数据：波次、金币、木材、英雄剩余 HP
- 面板可点击返回局外
- 触发点：`runtime/battlefield.lua` 的 `finish_game()` → 调用 UI 面板

### P1：正式背包面板 UI
- 背包界面：查看当前成长武器、已装备烙印
- 可结合结算页一同开发

### P2：局外长期成长与挑战奖励写回
- 挑战完成后写回局外成长数据
- 下局可看到成长结果

### P3：多英雄与章节专属内容
