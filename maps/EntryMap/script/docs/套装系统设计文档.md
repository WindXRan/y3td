# 套装系统设计文档

## 一、系统概述

套装系统是游戏中的核心养成系统之一，玩家可以通过收集和升阶套装装备来获得强大的属性加成。套装系统包含以下核心机制：

- **5个套装**：每个套装有独特的定位和属性倾向
- **6个装备槽**：武器、护甲、头盔、靴子、饰品1、饰品2
- **10星升阶**：每个装备可升阶至10星
- **套装阶级**：由所有装备的最低阶级决定
- **套装效果**：根据套装阶级解锁不同的属性加成

### 2.3 套装阶级规则

**核心逻辑**：套装阶级 = min(所有装备阶级)

```
示例：
- 武器：5星
- 护甲：3星
- 头盔：5星
- 靴子：4星
- 饰品1：5星
- 饰品2：5星

→ 套装阶级 = 3星（取最小值）
```

## 四、系统API

### 4.1 核心接口

| 函数名 | 功能说明 | 参数 | 返回值 |
|--------|----------|------|--------|
| `get_player_suit_data(player)` | 获取玩家所有套装数据 | player: Player对象 | 套装数据table |
| `get_equipment_star_level(player, suit_id, slot)` | 获取装备星级 | player, suit_id, slot | 星级(0-10) |
| `set_equipment_star_level(player, suit_id, slot, level)` | 设置装备星级 | player, suit_id, slot, level | 无 |
| `calculate_suit_star_level(player, suit_id)` | 计算套装阶级 | player, suit_id | 阶级(0-10) |
| `get_upgrade_cost(player, suit_id, slot)` | 获取升阶消耗 | player, suit_id, slot | 消耗配置或nil |
| `can_upgrade_equipment(player, suit_id, slot)` | 检查是否可升阶 | player, suit_id, slot | bool, 错误信息 |
| `upgrade_equipment(player, suit_id, slot)` | 升阶装备 | player, suit_id, slot | bool, 消息, 新等级 |
| `apply_suit_effects(player, suit_id, level)` | 应用套装效果 | player, suit_id, level | 无 |
| `get_suit_current_effects(player, suit_id)` | 获取当前套装效果 | player, suit_id | 效果配置或nil |
| `get_suit_next_effects(player, suit_id)` | 获取下一阶效果 | player, suit_id | 效果配置或nil |
| `initialize_player_suit_data(player)` | 初始化玩家数据 | player | 无 |
| `get_player_consumable_amount(player, key)` | 获取消耗品数量 | player, key | 数量 |
| `add_player_consumable(player, key, amount)` | 添加消耗品 | player, key, amount | 无 |
| `remove_player_consumable(player, key, amount)` | 扣除消耗品 | player, key, amount | 无 |

### 4.2 使用示例

```lua
local SuitSystem = require 'y3.game.suit_system'

-- 初始化玩家套装数据
SuitSystem.initialize_player_suit_data(player)

-- 检查是否可升阶
local can_upgrade, err = SuitSystem.can_upgrade_equipment(player, 'suit_dragon', 'weapon')
if not can_upgrade then
    print('无法升阶:', err)
    return
end

-- 执行升阶
local success, msg, new_level = SuitSystem.upgrade_equipment(player, 'suit_dragon', 'weapon')
print(msg, '新等级:', new_level)

-- 获取套装阶级
local suit_level = SuitSystem.calculate_suit_star_level(player, 'suit_dragon')
print('套装阶级:', suit_level)

-- 获取当前效果
local effects = SuitSystem.get_suit_current_effects(player, 'suit_dragon')
if effects then
    print('当前效果:', effects.description)
end

-- 添加消耗品
SuitSystem.add_player_consumable(player, 'consumable_star_stone_common', 10)
```

## 六、系统流程图

```
玩家选择装备 → 检查消耗品是否足够 → 扣除消耗品 → 随机判定成功率
    ↓
成功 → 提升装备星级 → 计算套装阶级 → 应用新效果
    ↓
失败 → 仅扣除消耗品，星级不变
```

## 七、扩展说明

### 7.1 新增套装

如需新增套装，需要：
1. 在 `suit_catalog.lua` 中添加套装配置
2. 在 `suit_upgrade_cost.lua` 中添加升阶消耗（或使用通用规则）
3. 在 `suit_effects.lua` 中添加10个星级的效果配置

### 7.2 调整数值

- 升阶消耗：修改 `suit_upgrade_cost.lua`
- 套装效果：修改 `suit_effects.lua`
- 消耗品配置：修改 `consumables.lua`

### 7.3 注意事项

1. 套装阶级取所有装备的最小值，需要提示玩家均衡升级
2. 升阶有失败概率，高星级失败概率较高
3. 消耗品不足时无法升阶
4. 最高星级为10星，达到后无法继续升阶
