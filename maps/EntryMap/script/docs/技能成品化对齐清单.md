# 技能成品化对齐清单（第一版）

## 目标

将当前羁绊技能从“可用”提升到“成品手感”，统一按照以下维度验收：

1. 命中一致性（视觉轨迹=判定轨迹=伤害结果）
2. 反馈层级（前摇/飞行/命中/余波）
3. 技能识别度（不同技能一眼可分）
4. AOE 边界可预期（边缘命中稳定）
5. 触发节奏与强度曲线（不空放、不突兀）
6. 性能预算（并发可控、掉帧可控）

---

## 高优先级技能对齐（先做这 3 条）

| 技能 | 当前问题 | 目标标准（成品） | 验收方式 |
|---|---|---|---|
| 龙骑士（火龙） | 偶发空伤、手感轻、范围曾偏小 | 固定距离直线命中，0 空伤；明显前摇+飞行压迫+命中爆裂；AOE 不限命中数 | 30 次触发抽样：空伤=0；边缘怪命中稳定；体感主观评分≥8/10 |
| 冰霜法师（冰法） | 曾出现无特效/存在感弱 | 每次触发必有投射物+命中特效；持续伤害节奏清晰（每跳有反馈） | 20 次触发抽样：特效缺失=0；每跳均有可见反馈 |
| 雷电法王（电法） | 曾出现无特效/电击辨识弱 | 每次触发必有电击视觉；多目标链击层次分明 | 20 次触发抽样：特效缺失=0；目标切换反馈清晰 |

---

## 参数与实现规范

1. AOE 技能默认不设置最大命中数（除非设计文档明确要求）。
2. 所有关键技能必须有“视觉兜底配置”（projectile/particle 强制可用）。
3. 判定逻辑优先“固定轨迹”而不是“追踪单位状态”。
4. 每个技能必须有独立弹道视觉，不与其它核心技能共用。
5. 高并发循环内禁止文件系统扫描、禁止高成本动态查询。

---

## 性能预算（首版阈值）

1. 单技能触发时：新增粒子对象数量应可控（避免瞬时堆积）。
2. 周期技能每秒触发次数可配上限，防止异常叠加。
3. UI/逻辑刷新与技能计算解耦，避免技能触发牵连 UI 卡顿。

---

## 执行顺序（建议）

1. 火龙：命中一致性 + 反馈层级 + 性能采样  
2. 冰法：持续伤害跳点反馈 + 视觉稳定性  
3. 电法：链击反馈层次 + 多目标稳定性  
4. 其余羁绊按同模板逐条迁移
