# 启动入口链路

## 模块职责

说明项目如何从地图初始化进入业务运行时，以及入口层与玩法层的边界。

## Source Of Truth

- 启动文件：`maps/EntryMap/script/main.lua`
- 运行时入口：`maps/EntryMap/script/entry_runtime.lua`

## 关键状态与数据流

- `main.lua` 维护 `runtime`、`runtime_load_attempted`、`bootstrapped`
- `bootstrap_once()` 通过 `游戏-初始化` 事件与 `ltimer.wait(0)` 双重触发，避免漏启动
- `entry_runtime.bootstrap()` 负责真正的局内状态初始化

## 当前已实现行为

- `main.lua` 会根据调试模式设置日志级别
- `main.lua` 用 `xpcall(require 'entry_runtime')` 做入口加载兜底
- `entry_runtime.bootstrap()` 当前执行顺序稳定：
  - `validate_config()`
  - `reset_state()`
  - `create_hero()`
  - `register_runtime_events()`
  - `register_dev_commands()`
  - `start_runtime_loops()`
  - `ensure_gm_panel()`
  - `start_wave(1)`

## 未实现或占位项

- 入口层没有接多模式选择、多英雄选择、局外存档读写
- 启动后没有正式 Loading/模式选择流程，当前直接进局

## Agent 修改建议与风险点

- 想改“游戏开始时做什么”，先改 `entry_runtime.bootstrap()`，不是改 `main.lua` 的外围壳
- 想新增真实启动分支时，优先在 `bootstrap()` 内拆模块，不要把业务规则堆回 `main.lua`
- `main.lua` 底部的大段 `if false then` 旧原型代码不是当前主线，不要误改
