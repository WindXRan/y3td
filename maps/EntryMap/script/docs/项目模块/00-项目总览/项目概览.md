# 项目概览

## 模块职责

本目录不是策划案，也不是 Lua API 教程，而是面向开发 agent 的工程手册。  
`maps/EntryMap/script` 是当前项目唯一需要优先理解和维护的运行时主线。

## Source Of Truth

- 运行入口：`maps/EntryMap/script/main.lua`
- 运行时总协调器：`maps/EntryMap/script/entry_runtime.lua`
- 局外运行时：`maps/EntryMap/script/entry_runtime_outgame.lua`
- 运行时配置汇总：`maps/EntryMap/script/entry_config.lua`
- 静态对象定义：`maps/EntryMap/script/entry_objects`
- 羁绊运行时：`maps/EntryMap/script/runtime_bonds.lua`
- 运行时子模块：`maps/EntryMap/script/entry_runtime_*.lua`
- 资源索引：`maps/EntryMap/script/ui_res.lua`
- 策划设计源：`maps/EntryMap/script/docs/design`

## 关键状态与数据流

- 地图业务代码集中在 `maps/EntryMap/script`
- 静态玩法对象集中在 `maps/EntryMap/script/entry_objects`
- 全局运行时状态集中在 `entry_runtime.lua` 的 `STATE`
- 地图 UI 资产在 `maps/EntryMap/ui`
- 地图级可视化触发器在 `maps/EntryMap/global_trigger`
- 单位、技能、投射物等物编数据在 `maps/EntryMap/unit`、`maps/EntryMap/ability`、`maps/EntryMap/projectile` 等目录
- 根目录 `global_script`、根目录 `global_trigger` 当前都不是主链路

## 当前已实现行为

- `main.lua -> require 'entry_runtime' -> bootstrap()` 的真实启动链路已经跑通
- 启动后先进入 `EntryMap` 内局外页，不再自动创建英雄和直接开第 1 波
- 已有 `1-1`、`1-2`、`1-3` 三张章节卡与 `standard/challenge` 模式切换
- 单英雄自动战斗、5 波主线、Boss 登场、挑战并行与基础胜负结算已实现
- `G` 三选一强化、`F` 羁绊抽卡、`Q/W/E/R` 四类挑战、GM 面板与调试热键已接入
- 宝物 3 选 1、烙印 3 选 1、奖励队列与待选轮次互斥已经进入真实 runtime
- `entry_runtime_hud.lua` 已接入顶部/底部 HUD、技能按钮、羁绊按钮与挑战按钮
- 羁绊不再只是文档占位，`runtime_bonds.lua` 已经负责候选、结算、动态效果与奖励加成
- 局外存档骨架已经落地，可记录章节/模式解锁、最近结果与当前选择

## 部分实现与未实现

- 当前高价值决策仍主要通过文本消息加 HUD 按钮驱动，不是正式中央决策面板
- 地图已有 HUD 挂载与运行时 HUD，但背包、奖励记录、正式结算页还未落地
- 当前不少主怪/Boss/挑战怪仍是临时替身物编，不是最终怪物资源
- 当前局外系统只完成选关、解锁和基础存档骨架，长期养成与局外奖励未接入
- 成长武器、词缀节点、多英雄、章节专属战斗内容仍未正式接入运行时

## Agent 修改建议与风险点

- 先确认改动是否属于 `maps/EntryMap/script` 主链路，再决定是否碰 UI、触发器或物编
- 如果改的是波次、挑战、攻击技能、宝物、烙印、羁绊静态定义，优先看 `entry_objects`
- 如果改的是运行时行为，优先看 `entry_runtime.lua` 指向的子模块，而不是把逻辑继续塞回总文件
- 不要把 `docs/design` 当成“已经实现”的事实来源；实现事实必须以 `script/*.lua` 为准
- 不要误把根目录 `global_script`、根目录 `global_trigger`、`可重载的代码.lua` 当成真实玩法入口
