# 项目概览

## 模块职责

本目录不是策划案，也不是 Lua API 教程，而是面向开发 agent 的工程手册。  
`maps/EntryMap/script` 是当前项目唯一需要优先理解和维护的运行时主线。

## Source Of Truth

- 运行入口：`maps/EntryMap/script/main.lua`
- 运行时主模块：`maps/EntryMap/script/entry_runtime.lua`
- 运行时配置：`maps/EntryMap/script/entry_config.lua`
- 羁绊运行时：`maps/EntryMap/script/runtime_bonds.lua`
- 资源索引：`maps/EntryMap/script/ui_res.lua`
- 策划设计源：`maps/EntryMap/script/docs/design`

## 关键状态与数据流

- 地图业务代码集中在 `maps/EntryMap/script`
- 地图 UI 资产在 `maps/EntryMap/ui`
- 地图级可视化触发器在 `maps/EntryMap/global_trigger`
- 单位、技能、投射物等物编数据在 `maps/EntryMap/unit`、`maps/EntryMap/ability`、`maps/EntryMap/projectile` 等目录
- 根目录 `global_script`、根目录 `global_trigger` 当前都不是主链路

## 当前已实现行为

- `main.lua -> require 'entry_runtime' -> bootstrap()` 的真实启动链路已经跑通
- 单英雄自动战斗、5 波主线、Boss 登场、基础胜负结算已实现
- `G` 三选一强化、`F` 羁绊抽卡、`Q/W/E/R` 四类挑战、GM 面板与调试热键已接入
- 羁绊不再只是文档占位，`runtime_bonds.lua` 已经负责候选、结算、动态效果与奖励加成

## 部分实现与未实现

- 宝物挑战奖励中的“宝物候选”仍是文本占位，没有正式宝物系统和奖励队列
- 中央决策面板、成长武器、词缀节点、烙印系统、局外成长/存档、多英雄/章节仍未正式接入运行时
- `entry_runtime.lua` 仍保留部分 legacy 包装函数，真实效果已经委托给 `runtime_bonds.lua`

## Agent 修改建议与风险点

- 先确认改动是否属于 `maps/EntryMap/script` 主链路，再决定是否碰 UI、触发器或物编
- 不要把 `docs/design` 当成“已经实现”的事实来源；实现事实必须以 `script/*.lua` 为准
- 不要误把根目录 `global_script`、根目录 `global_trigger`、`可重载的代码.lua` 当成真实玩法入口
