# 挑战系统

## 模块职责

说明 `Q/W/E/R` 四类挑战如何解锁、消耗次数、并行运行和结算奖励。

## Source Of Truth

- 挑战配置：`maps/EntryMap/script/entry_config.lua`
- 挑战运行时：`maps/EntryMap/script/entry_runtime.lua`

## 关键状态与数据流

- 静态配置：`CONFIG.challenges`、`CONFIG.challenge_rules`
- 当前实例：`STATE.active_challenges`
- 次数资源：`STATE.challenge_charges`
- 启动入口：`try_start_challenge(challenge_id)`
- 更新入口：`update_challenges(dt)`、`update_challenge_charges(dt)`

## 当前已实现行为

- 已有四类挑战：
  - `Q` 金币挑战
  - `W` 木材挑战
  - `E` 经验挑战
  - `R` 宝物挑战
- 每个挑战都有解锁条件、持续时间、批次配置、奖励定义
- 挑战和主线并行运行，不中断主线刷怪
- 挑战次数随时间恢复

## 当前实现边界

- 解锁规则已有字段，且 `entry_runtime.lua` 已有 `evaluate_unlock()`
- `ignore_unlock_requirements = true` 使当前调试/原型阶段可以绕过正式解锁约束
- 宝物挑战当前只有“奖励文字占位”，没有正式宝物选择轮次

## 未实现或占位项

- 没有拆成独立 challenge 表体系
- 没有挑战模式专用 stage mode
- 没有挑战追踪 UI 与中央奖励队列

## Agent 修改建议与风险点

- 改挑战优先改 `entry_config.lua`，再改 `entry_runtime.lua` 的运行时判定
- 不要把宝物挑战误写成“已具备正式宝物系统”
- 若要把挑战扩展成长期系统，应先补状态机与奖励队列，再补内容量
