# 挑战系统

## 模块职责

说明 `Q/W/E/R` 四类挑战如何解锁、消耗次数、并行运行和结算奖励。

## Source Of Truth

- 挑战对象：`maps/EntryMap/script/entry_objects/challenges`
- 挑战配置汇总：`maps/EntryMap/script/entry_config.lua`
- 挑战运行时：`maps/EntryMap/script/entry_runtime_battlefield.lua`
- 挑战按钮/HUD 状态：`maps/EntryMap/script/entry_runtime_hud.lua`

## 关键状态与数据流

- 静态配置：`CONFIG.challenges`、`CONFIG.challenge_rules`
- 当前实例：`STATE.active_challenges`
- 次数资源：`STATE.challenge_charges`
- 启动入口：`try_start_challenge(challenge_id)`
- 更新入口：`update_challenges(dt)`、`update_challenge_charges(dt)`

## 当前已实现行为

- 已有四类挑战：
  - `Q` 金币挑战
  - `W` 木材挑战
  - `E` 经验挑战
  - `R` 宝物挑战
- 每个挑战都有解锁条件、持续时间、批次配置、奖励定义
- 挑战和主线并行运行，不中断主线刷怪
- 挑战次数随时间恢复
- HUD 和底部按钮会显示挑战可用、进行中、未解锁、缺次数等状态
- 宝物挑战成功后会发放基础资源，并额外排入一次宝物 3 选 1

## 当前实现边界

- 解锁规则已有字段，`entry_runtime_battlefield.lua` 与 `entry_runtime_hud.lua` 都会评估
- `ignore_unlock_requirements = true` 使当前调试/原型阶段可以绕过正式解锁约束
- 当前没有独立挑战详情面板或挑战历史面板

## 未实现或占位项

- 没有挑战模式专用 stage mode
- 没有更完整的挑战追踪 UI 和结算页

## Agent 修改建议与风险点

- 改挑战优先改 `entry_objects/challenges/*.lua`，再看 `entry_runtime_battlefield.lua` 的运行时判定
- 不要再把宝物挑战写成“只有文本占位”；它已经会进入正式宝物奖励轮次
- 若要加新的高价值挑战奖励，优先走奖励队列，不要直接弹散乱文本
