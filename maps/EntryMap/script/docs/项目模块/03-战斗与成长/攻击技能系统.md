# 攻击技能系统

## 模块职责

说明英雄自动战斗、普攻技能、攻击技能槽位和施放更新由谁负责。

## Source Of Truth

- 攻击技能定义：`maps/EntryMap/script/entry_objects/attack_skills`
- 攻击技能运行时：`maps/EntryMap/script/entry_runtime_attack_skills.lua`
- 攻击技能强化：`maps/EntryMap/script/entry_runtime_attack_upgrades.lua`
- 技能强化设计参考：`maps/EntryMap/script/docs/design/初步设计/11-攻击技能与羁绊配表与实现映射.md`

## 关键状态与数据流

- 技能定义：`ATTACK_SKILL_DEFS`
- 表现定义：`ATTACK_SKILL_VFX`
- 技能运行时：`STATE.attack_skill_state`
- 普攻技能同步：`setup_basic_attack_ability()`、`sync_basic_attack_ability()`

## 当前已实现行为

- 英雄会自动索敌并自动执行普攻
- 已内置 `basic_attack`、`arcane_arrow`、`flame_arrow`、`frost_arrow`、`thunder`
- `basic_attack` 固定在 1 号位，2-4 号位通过 `G` 三选一逐步解锁
- 技能有槽位、冷却、目标选择、VFX、投射物、控制和命中后处理
- 普攻被单独绑定到引擎技能按钮与描述同步逻辑

## 当前实现特点

- 技能静态定义和表现资源已经拆进 `entry_objects/attack_skills/*.lua`
- 技能实例、自动施法与伤害处理在 `entry_runtime_attack_skills.lua`
- `G` 强化是加权 3 选 1，并带新技能解锁概率、失败保底与次数上限
- 文档中的更完整卡牌 UI 仍是未来方向，不是当前实现现实

## 未实现或占位项

- 没有正式中央决策面板承载技能候选 UI
- 没有多英雄差异化技能树

## Agent 修改建议与风险点

- 改技能数值或描述时先改 `entry_objects/attack_skills/*.lua`
- 改施法、索敌、投射物或控制时改 `entry_runtime_attack_skills.lua`
- 改候选生成、权重、解锁逻辑时改 `entry_runtime_attack_upgrades.lua`
- 改普攻逻辑时同时检查 `basic_attack` 与引擎技能描述同步
