# 攻击技能系统

## 模块职责

说明英雄自动战斗、普攻技能、攻击技能槽位和施放更新由谁负责。

## Source Of Truth

- 攻击技能定义：`maps/EntryMap/script/entry_runtime.lua`
- 技能强化设计参考：`maps/EntryMap/script/docs/design/初步设计/11-攻击技能与羁绊配表与实现映射.md`

## 关键状态与数据流

- 技能定义：`ATTACK_SKILL_DEFS`
- 表现定义：`ATTACK_SKILL_VFX`
- 技能运行时：`STATE.attack_skill_state`
- 普攻技能同步：`setup_basic_attack_ability()`、`sync_basic_attack_ability()`

## 当前已实现行为

- 英雄会自动索敌并自动执行普攻
- 已内置 `basic_attack`、`arcane_arrow`、`flame_arrow`、`frost_arrow`、`thunder`
- 技能有槽位、冷却、目标选择、VFX 与命中后处理
- 普攻被单独绑定到引擎技能按钮与描述同步逻辑

## 当前实现特点

- 技能本体、表现资源、升级效果当前都还在 `entry_runtime.lua` 内部
- 技能系统已经可跑，但还没有拆成独立模块/独立配表
- 文档中的“攻击技能卡池工程化方案”仍是未来拆表方向，不是当前代码现实

## 未实现或占位项

- 没有独立 `attack_skill_base`、`attack_skill_card` 等外部表
- 没有正式中央决策面板承载技能候选 UI
- 没有多英雄差异化技能树

## Agent 修改建议与风险点

- 改技能数值或行为时，先确认是定义层、VFX 层还是强化层
- 如果要继续扩展技能数量，应优先抽离定义与施法器，而不是继续把所有技能塞进一个文件块
- 改普攻逻辑时同时检查 `basic_attack` 与引擎技能描述同步
