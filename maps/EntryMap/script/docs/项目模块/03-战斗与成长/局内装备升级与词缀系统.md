# 局内装备升级与词缀系统

## 模块职责

说明局内三件初始装备如何通过金币升级、在固定等级节点获得词缀三选一，并与现有羁绊三选一共同构成局内成长系统。

## 设计目标

- 不做传统掉落装备、背包、比较、替换流程
- 通过极轻交互保留“养装备”和“抽词缀”的 build 成长感
- 让金币成为稳定的局内强化资源
- 与木材抽羁绊形成明确分工
- 让普攻流、技能流、双修流都能成立

## 系统定位

- 木材：负责拿路线，主要投入 `F` 链式羁绊
- 金币：负责养装备，主要投入局内装备升级与词缀
- 两套系统并存，但资源用途清晰，不互相替代

## 核心规则

- 开局固定给予玩家三件初始装备：
  - 武器
  - 法器
  - 战徽
- 每件装备独立升级
- 左键：升级 `1` 级
- 右键：尽可能连续升级 `10` 级
- 每 `10` 级获得一次词缀选择机会
- 词缀节点固定为：
  - `10`
  - `20`
  - `30`
  - `40`
  - `50`
  - `60`
- 装备等级上限为 `80`
- `70` 与 `80` 只提供毕业数值强化，不新增词缀
- 当连续升级跨过词缀节点时，必须先弹出词缀选择并完成选择，再继续后续升级

## 三件装备职责

### 武器

- 面向普攻侧输出
- 负责持续输出、清怪效率、普攻触发
- 与箭术、暴击、体术近战型打法兼容

建议基础成长：

- `1-60` 级：攻击 `+8`，普攻伤害 `+0.6%`
- `61-80` 级：攻击 `+12`，普攻伤害 `+0.8%`

### 法器

- 面向技能侧输出
- 负责技能爆发、元素联动、施法循环
- 与法术、成长智力线、元素类技能兼容

建议基础成长：

- `1-60` 级：技能伤害 `+0.8%`，技能急速 `+0.5`
- `61-80` 级：技能伤害 `+1%`，技能急速 `+0.8`

### 战徽

- 面向通用输出放大
- 负责把武器与法器的收益一起放大
- 负责击杀、精英/Boss、羁绊联动类增益

建议基础成长：

- `1-60` 级：所有伤害 `+0.35%`，对精英/Boss伤害 `+0.5%`
- `61-80` 级：所有伤害 `+0.5%`，对精英/Boss伤害 `+0.8%`

## 输出叠加原则

- 武器与法器不是二选一
- 武器侧普攻输出与法器侧技能输出可以同时成立
- 战徽默认同时增益这两条输出线
- 系统应支持三类 build：
  - 主武器流
  - 主法器流
  - 武器法器双修流

## 词缀节点规则

- 每件装备最多获得 `6` 个词缀
- 词缀只在 `10-60` 级节点发放
- `70/80` 作为纯毕业强化区间
- 词缀选择使用统一 `3 选 1`
- 词缀一旦选定，本局内固定生效，不做替换或洗词条

## 词缀池方向

### 武器词缀池

用于普攻、暴击、清怪、连击类成长。

首批建议词缀：

- `分裂箭锋`：普攻额外命中 `1` 个目标
- `穿透箭锋`：普攻穿透 `+1`
- `猎杀节奏`：攻击速度 `+25%`
- `残暴打击`：暴击率 `+8%`
- `断骨重击`：暴击伤害 `+20%`
- `爆裂弹头`：普攻命中时造成一次小范围爆炸
- `收割狂热`：每击杀 `30` 名敌人，攻速 `+40%`，持续 `6` 秒
- `破军`：对精英和 Boss 的普攻伤害 `+25%`

### 法器词缀池

用于技能、五行、冷却、触发类成长。

首批建议词缀：

- `施法回响`：技能有 `15%` 概率额外施放一次
- `极速施法`：技能急速 `+15`
- `扩散术式`：技能范围 `+20%`
- `烈焰灌注`：火行伤害 `+20%`
- `寒霜灌注`：水行伤害 `+20%`
- `雷暴灌注`：木行伤害 `+20%`
- `燃烧附加`：技能命中附加燃烧
- `终幕法击`：对精英和 Boss 的技能伤害 `+25%`

### 战徽词缀池

用于总增伤、混合增伤、羁绊联动类成长。

首批建议词缀：

- `战斗狂热`：所有伤害 `+12%`
- `精英猎手`：对精英和 Boss 伤害 `+20%`
- `屠戮奖赏`：每击杀 `50` 名敌人，所有伤害 `+8%`，持续 `8` 秒
- `双修印记`：普攻伤害和技能伤害各 `+10%`
- `箭术共鸣`：每拥有 `1` 张箭术羁绊卡，普攻伤害 `+4%`
- `法术共鸣`：每拥有 `1` 张法术羁绊卡，技能伤害 `+4%`
- `暴击共鸣`：每拥有 `1` 张暴击羁绊卡，暴击伤害 `+8%`
- `终局号令`：波次越高，所有伤害越高

## 交互流程

### 开局

- 开局自动给三件“破烂装备”
- 三件装备默认可见，不需要拾取
- 装备面板中显示：
  - 装备名
  - 当前等级
  - 当前基础加成
  - 下一个词缀节点
  - 已获得词缀数量

### 正常升级

- 玩家有足够金币时，可点击装备升级
- 左键执行单级升级
- 右键执行尽可能多的十连升级
- 升级成功后立即写回装备等级与加成

### 跨越词缀节点

- 若本次升级触发 `10/20/30/40/50/60` 节点
- 系统暂停本次连续升级
- 弹出一轮 `装备词缀三选一`
- 玩家选完后再继续剩余升级

### 词缀弹窗

- 复用现有三选一 UI 框架
- 标题改为 `装备词缀选择`
- 副标题显示触发来源，如：
  - `武器 Lv10`
  - `法器 Lv30`
  - `战徽 Lv50`
- 三个选项全部来自对应装备类型的词缀池

## 与现有三选一系统的关系

- 羁绊三选一继续使用现有 `F` 系统
- 装备词缀三选一复用同一套选择面板
- 两者都属于 pending 轮次系统，必须统一轮次互斥
- 同一时刻只允许一个待选轮次存在

建议优先级：

- 若有待处理的装备词缀节点，先完成装备词缀选择
- 若有待处理的羁绊三选一，继续当前羁绊轮次
- 不允许两种三选一同时压栈

## 金币消耗建议

建议先用分段递增，保持简单可控：

- `1-20` 级：每级 `100` 金币
- `21-40` 级：每级 `180` 金币
- `41-60` 级：每级 `300` 金币
- `61-80` 级：每级 `500` 金币

右键十连升级时：

- 不要求玩家必须拥有完整十级总费用
- 只要还能继续支付下一级费用，就继续升级
- 钱不够时自动停在当前可达等级

## UI 建议

- HUD 或构筑面板中增加三件装备区
- 每件装备按钮直接标明：
  - `Lv x/80`
  - `下个词缀：Lv y`
  - `左键+1`
  - `右键+10`
- 每到 `10` 的倍数时，装备名称可以变化一次，增强“养装备”的感受

命名示例：

- 武器：`破损长弓` -> `硬木长弓` -> `猎杀号角弓` -> `灾厄风暴弓`
- 法器：`裂纹法珠` -> `元素法珠` -> `共鸣法印` -> `灾厄秘核`
- 战徽：`磨损战徽` -> `军团战徽` -> `征服徽记` -> `终局圣徽`

## 当前版本刻意不做的内容

- 不做装备掉落
- 不做装备背包
- 不做装备替换
- 不做装备洗练
- 不做词缀重铸
- 不做词缀拆卸

以上内容都可以在后续版本再考虑，但不应进入第一版。

## 实现建议

建议拆成以下几个部分：

- 装备定义与词缀池定义
- 装备运行时状态
- 升级与费用计算
- 词缀节点检测
- 装备词缀三选一轮次
- HUD/三选一面板适配

建议运行时状态挂在：

- `STATE.gear_state`

建议包含：

- 三件装备当前等级
- 三件装备已获得词缀列表
- 当前待处理的装备词缀轮次
- 装备基础成长累计值

## 风险点

- 不能让装备词缀轮次和羁绊轮次同时存在
- 不能让右键十连跳过中间词缀节点
- 不能让金币系统失控，导致装备升级完全无脑点满
- 不能让武器与法器变成互斥成长路线
- 第一版不要把装备系统做成掉落系统，否则交互复杂度会急剧上升

## 第一版落地建议

- 先做三件固定初始装备
- 先做固定上限 `80`
- 先做固定词缀节点 `10-60`
- 先做每件装备 `8` 条左右首批词缀
- 先复用现有三选一 UI
- 先不做任何背包与替换逻辑

这样可以最小成本把“金币养装备 + 阶段词缀 build”跑通。
