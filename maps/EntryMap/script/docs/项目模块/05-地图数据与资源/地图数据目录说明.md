# 地图数据目录说明

## 模块职责

说明地图侧哪些目录承载“资源与编辑器数据”，哪些目录才承载“运行时玩法逻辑”。

## Source Of Truth

- 业务代码：`maps/EntryMap/script`
- 地图 UI：`maps/EntryMap/ui`
- 地图级触发器：`maps/EntryMap/global_trigger`
- 物编目录：`maps/EntryMap/unit`、`maps/EntryMap/ability`、`maps/EntryMap/item`、`maps/EntryMap/modifier`、`maps/EntryMap/projectile`

## 目录职责

- `script`
  - 当前唯一主逻辑目录
- `ui`
  - 面板 JSON、预制体、HUD 外壳
- `global_trigger`
  - 地图内 UI 绑定与编辑器触发器资产
- `unit` / `ability` / `projectile` / `modifier` / `item`
  - 实际物编资源库
- `editor_table`
  - 编辑器导出的对象索引与总表
- `editor`
  - 编辑器树结构、分组与元数据
- `custom_eca`
  - 地图级自定义 ECA 扩展位，当前不是主线

## 关键状态与数据流

- `entry_config.lua` 中的单位 ID 必须能在地图物编数据中找到
- `entry_runtime.lua` 的 UI 挂载依赖 `maps/EntryMap/ui/GameHUD.json`
- `ui_res.lua` 维护部分资源 ID，但不替代 UI JSON 本体

## 当前已实现行为

- 当前项目已经从“临时英雄替身怪”过渡到正式波次单位 ID，但设计稿中的很多表结构还没工程化
- 编辑器目录和物编目录是数据依赖，不是 agent 的首要阅读入口

## 未实现或占位项

- 物编、编辑器表与运行时之间还没有自动校验或生成流程
- 设计稿中的外部配表体系还没有从 Markdown 真正落成地图可消费的数据表

## Agent 修改建议与风险点

- 如果只是改玩法规则，不要先钻物编目录
- 如果改单位、技能、投射物 ID，必须同步核对地图资源是否存在
- 文档里引用设计稿时统一使用 `maps/EntryMap/script/docs/design` 路径
