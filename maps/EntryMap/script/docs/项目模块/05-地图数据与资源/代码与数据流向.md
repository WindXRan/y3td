# 代码与数据流向

## 模块职责

说明 `script`、配置、物编、UI 资产、触发器之间的数据边界，帮助 agent 判断“应该改哪一层”。

## Source Of Truth

- 配置层：`maps/EntryMap/script/entry_config.lua`
- 运行时层：`maps/EntryMap/script/entry_runtime.lua`
- 羁绊层：`maps/EntryMap/script/runtime_bonds.lua`
- UI 资产层：`maps/EntryMap/ui`
- 物编层：`maps/EntryMap/unit`、`maps/EntryMap/ability`、`maps/EntryMap/projectile`

## 数据流总图

```text
docs/design
  -> 设计目标与未来模块

entry_config.lua
  -> 波次 / 区域 / 挑战 / 资源规则

entry_runtime.lua
  -> STATE / 主循环 / 战斗 / 结算 / 调试

runtime_bonds.lua
  -> F抽卡 / 羁绊效果 / 奖励修正

maps/EntryMap/ui + ui_res.lua
  -> HUD资产 / 挂载点 / 资源ID

maps/EntryMap/unit|ability|projectile
  -> 实际单位/技能/投射物物编资源
```

## 当前已实现闭环

- `entry_config.lua` 提供 5 波、区域、挑战与资源规则
- `entry_runtime.lua` 使用这些配置生成单位、推进波次、发放奖励、处理胜负
- `runtime_bonds.lua` 在击杀、奖励、动态效果刷新中回流到主循环
- UI 当前主要承担挂载点与开发辅助展示

## 当前缺口

- 设计稿中的成长武器、宝物、烙印、奖励队列还没有进入上述闭环
- 这些系统当前停留在设计层，尚未拥有自己的 runtime state 与 UI 接口

## Agent 修改建议与风险点

- 改规则先看 `entry_config.lua`，改行为先看 `entry_runtime.lua`
- 改羁绊必须检查 `runtime_bonds.lua` 是否也要同步
- 改物编 ID 时要核对真实资源目录，不能只改文档或 Lua 常量
