# 代码与数据流向

## 模块职责

说明 `script`、配置、物编、UI 资产、触发器之间的数据边界，帮助 agent 判断“应该改哪一层”。

## Source Of Truth

- 配置层：`maps/EntryMap/script/entry_config.lua`
- 对象层：`maps/EntryMap/script/entry_objects`
- 运行时协调层：`maps/EntryMap/script/entry_runtime.lua`
- 运行时子模块层：`maps/EntryMap/script/entry_runtime_*.lua`
- 羁绊层：`maps/EntryMap/script/runtime_bonds.lua`
- UI 资产层：`maps/EntryMap/ui`
- 物编层：`maps/EntryMap/unit`、`maps/EntryMap/ability`、`maps/EntryMap/projectile`

## 数据流总图

```text
docs/design
  -> 设计目标与未来模块

entry_objects/*
  -> 波次 / 挑战 / 攻击技能 / 羁绊 / 宝物 / 烙印 的静态对象

entry_config.lua
  -> 点位 / 区域 / 资源规则 / 挑战规则 / 对象汇总

entry_runtime.lua
  -> STATE / 奖励队列 / 轮次互斥 / 子系统装配

entry_runtime_battlefield.lua
  -> 主线波次 / 挑战 / 战场 / 结算

entry_runtime_progression.lua
  -> 经验 / 等级 / 技能点

entry_runtime_attack_skills.lua + entry_runtime_attack_upgrades.lua
  -> 攻击技能实例 / G三选一

runtime_bonds.lua
  -> F抽卡 / 羁绊效果 / 奖励修正 / 动态触发

entry_runtime_hud.lua
  -> 运行时 HUD 与按钮状态

maps/EntryMap/ui + ui_res.lua
  -> HUD资产 / 挂载点 / 资源ID

maps/EntryMap/unit|ability|projectile
  -> 实际单位/技能/投射物物编资源
```

## 当前已实现闭环

- `entry_objects/*` 提供波次、挑战、攻击技能、羁绊、宝物、烙印等静态对象
- `entry_config.lua` 汇总点位、区域、资源规则与挑战规则
- `entry_runtime.lua` 协调战场、成长、攻击技能、羁绊、宝物、烙印和奖励队列
- `runtime_bonds.lua` 在击杀、奖励、动态效果刷新中回流到主循环
- `entry_runtime_hud.lua` 读取 `STATE` 展示波次、Boss、资源、技能点、挑战状态与待领奖励

## 当前缺口

- 成长武器、词缀节点和长期局外成长还没有进入上述闭环
- 局外选关、章节/模式解锁和最近结果存档已经接通，但还不是完整长期资源闭环
- 宝物、烙印、奖励队列已经进入 runtime，但仍缺正式中央面板和背包页

## Agent 修改建议与风险点

- 改规则先看 `entry_config.lua` 与 `entry_objects/*`
- 改行为先看 `entry_runtime.lua` 指向的子模块
- 改羁绊必须检查 `runtime_bonds.lua` 是否也要同步
- 改物编 ID 时要核对真实资源目录，不能只改文档或 Lua 常量
