# 整体游戏流程图

## 模块职责

用 agent 可执行视角展示启动链路、单局循环和模块关系。

## Source Of Truth

- 启动与循环：`maps/EntryMap/script/main.lua`、`maps/EntryMap/script/entry_runtime.lua`
- 配置与系统分层：`maps/EntryMap/script/entry_config.lua`、`maps/EntryMap/script/runtime_bonds.lua`

## 关键状态与数据流

- 启动流：`main.lua -> entry_runtime.bootstrap()`
- 主循环流：`STATE + timer update`
- 成长回流：击杀奖励 -> 经验/资源 -> `G/F` -> 战斗力回写
- 系统分层流：设计 -> 配置 -> 运行时 -> UI/资源

## 启动链路

```mermaid
flowchart TD
    A[游戏初始化] --> B[main.lua]
    B --> C[require entry_runtime]
    C --> D[bootstrap]
    D --> E[validate_config]
    E --> F[reset_state]
    F --> G[create_hero]
    G --> H[register_runtime_events]
    H --> I[register_dev_commands]
    I --> J[start_runtime_loops]
    J --> K[ensure_gm_panel]
    K --> L[start_wave 1]
```

## 单局主循环

```mermaid
flowchart TD
    A[0.25秒循环] --> B[update_wave]

    A --> D[update_challenge_charges]
    A --> E[update_bond_effects]
    A --> F[update_attack_skills]
    A --> G[update_basic_attack]
    A --> H[被动资源与GM刷新]
```

## 单局推进

```mermaid
flowchart TD
    A[开始第N波] --> B[主怪批次刷新]
    B --> C[英雄自动战斗]
    C --> D[击杀发奖励与经验]
    D --> E[升级后获得技能点]
    E --> F[G 三选一强化]
    D --> G[F 羁绊抽卡与效果回流]
    B --> H[Boss到时登场]
    H --> I{是否击败Boss}
    I -- 否 --> C
    I -- 是且N小于5 --> J[进入下一波]
    I -- 是且N等于5 --> K[胜利结算]
```

## 阅读重点

- 看主循环先读 `entry_runtime.lua`
- 看配置先读 `entry_config.lua`
- 看 `F` 系统先读 `runtime_bonds.lua`
- 看未来规划与缺口，回到 `maps/EntryMap/script/docs/项目模块/07-实现状态与路线图/实现状态与路线图.md`
