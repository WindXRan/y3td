# 资源经验与结算

## 模块职责

说明金币、木材、经验、技能点、挑战次数以及胜负结算的归属与处理流程。

## Source Of Truth

- 资源规则：`maps/EntryMap/script/entry_config.lua`
- 资源与结算逻辑：`maps/EntryMap/script/entry_runtime.lua`
- 羁绊奖励修正：`maps/EntryMap/script/runtime_bonds.lua`

## 关键状态与数据流

- 资源规则：`CONFIG.resource_rules`
- 英雄成长：`STATE.hero_progress`
- 技能点：`STATE.skill_points`
- 挑战次数：`STATE.challenge_charges`
- 奖励入账统一入口：`award_rewards()`

## 当前已实现行为

- 每秒有基础金币/木材收入
- 击杀主怪和 Boss 会给经验、金币、木材
- 羁绊奖励修正通过 `BondSystem.build_reward_with_bonus()` 生效
- 英雄升级会增加技能点，并触发 `G` 系统输入入口
- `finish_game()` 会输出胜利/失败文本和本局基础结算摘要

## 当前实现边界

- 资源是局内资源，没有正式局外资源写回
- 经验已兼容引擎等级上限后的自定义进度计算
- 挑战次数是一种随时间恢复的特殊资源

## 部分实现与未实现

- 宝物挑战的 `special = '宝物候选(占位)'` 还没有进入正式奖励队列
- 没有局外货币、图鉴、成就、章节通关奖励写盘逻辑
- 没有结算页 UI 面板，只是基础文本与已有 UI 资源占位

## Agent 修改建议与风险点

- 任何新奖励都优先经过统一奖励入口，避免直接散改金币/木材/经验
- 若新增高价值奖励，先设计“待领取状态”而不是直接发文本
- 不要把局内资源和未来局外资源共用一套字段
