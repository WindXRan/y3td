# 资源经验与结算

## 模块职责

说明金币、木材、经验、技能点、挑战次数以及胜负结算的归属与处理流程。

## Source Of Truth

- 资源规则：`maps/EntryMap/script/entry_config.lua`
- 成长逻辑：`maps/EntryMap/script/entry_runtime_progression.lua`
- 资源与奖励接入：`maps/EntryMap/script/entry_runtime.lua`
- 局外结果写回：`maps/EntryMap/script/entry_runtime_outgame.lua`
- 羁绊奖励修正：`maps/EntryMap/script/runtime_bonds.lua`

## 关键状态与数据流

- 资源规则：`CONFIG.resource_rules`
- 局内资源：`STATE.resources`
- 英雄成长：`STATE.hero_progress`
- 技能点：`STATE.skill_points`
- 挑战次数：`STATE.challenge_charges`
- 奖励队列：`STATE.reward_queue`
- 奖励入账统一入口：`award_rewards()`

## 当前已实现行为

- 每秒有基础金币/木材收入
- 击杀主怪、Boss、挑战成功都会通过统一奖励入口发放资源
- 羁绊奖励修正通过 `BondSystem.build_reward_with_bonus()` 生效
- 宝物可以追加 `reward_ratio` 与 `passive_income`
- 英雄升级会增加技能点，并触发 `G` 系统输入入口
- 英雄达到 `10/20/30/40` 级时会把烙印节点排进奖励队列
- 宝物挑战成功后会立即给资源，并额外排入一次宝物 3 选 1
- `finish_game()` 会输出胜利/失败文本和本局基础结算摘要
- 单局结束后会把章节、模式、胜负和到达波次写回局外最近结果
- 标准模式通关会解锁本章挑战模式和下一章标准模式

## 当前实现边界

- 资源仍是局内资源，没有局外货币写回
- 经验已兼容引擎等级上限后的自定义进度计算
- 挑战次数是一种随时间恢复的特殊资源
- 高价值奖励已经有排队逻辑，但仍以文本与 HUD 为主要承载方式

## 部分实现与未实现

- 已有章节/模式解锁和最近结果写盘，但没有局外货币、图鉴、成就等奖励写盘逻辑
- 没有结算页 UI 面板，只是基础文本与已有 UI 资源占位

## Agent 修改建议与风险点

- 任何新奖励都优先经过统一奖励入口，避免直接散改金币/木材/经验
- 若新增高价值奖励，先决定是否进入 `reward_queue`
- 不要把局内资源和未来局外资源共用一套字段
