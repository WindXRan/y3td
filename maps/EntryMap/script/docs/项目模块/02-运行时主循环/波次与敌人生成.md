# 波次与敌人生成

## 模块职责

说明 5 波主线如何配置、如何生成主怪与 Boss、以及敌人 runtime 信息放在哪里。

## Source Of Truth

- 波次对象：`maps/EntryMap/script/entry_objects/waves`
- 波次配置汇总：`maps/EntryMap/script/entry_config.lua`
- 生成与推进：`maps/EntryMap/script/entry_runtime_battlefield.lua`

## 关键状态与数据流

- 静态配置：`CONFIG.waves`
- 当前波运行时：`STATE.active_wave`
- 敌人容器：`STATE.all_enemies`、`STATE.total_enemy_alive`
- 敌人运行时元信息：`STATE.enemy_info_map`
- 运行时元信息由 `create_enemy_info()` 和 `spawn_enemy()` 生成并维护
- 区域取点：`random_point_in_area()`

## 当前已实现行为

- 标准模式固定 5 波
- 每波固定 1 种主怪与 1 名 Boss
- `start_wave(index)` 会创建当前波 runtime，并提示 Boss 登场倒计时
- `update_wave(dt)` 按 `spawn_segments` 动态决定刷新间隔
- `spawn_main_batch()` 负责普通怪批次生成
- `spawn_boss()` 在 `boss_spawn_sec` 到点后刷 Boss
- Boss 被击败后会立刻切到下一波；最终 Boss 被击败后直接胜利
- 敌人 runtime 会区分 `main`、`boss`、`challenge` 等来源

## 当前实现特点

- 波次对象已经拆成 `entry_objects/waves/*.lua`
- 总单位数受软上限控制，超限时延后新批次，而不是强制清场
- Boss 登场后仍允许普通怪继续构成压力
- 当前 `entry_config.lua` 里的 `temp_unit_labels` 仍说明不少主怪/Boss 只是临时替身物编

## 部分实现与未实现

- `entry_config.lua` 已有 `boss_timeline_profile_id`、`boss_low_hp_profile_id` 字段，但当前代码层没有完整 Boss 时间轴系统
- 没有模式表、章节表、挑战模式波次表

## Agent 修改建议与风险点

- 改波次节奏优先改 `entry_objects/waves/*.lua`，不要把数值写死进 `update_wave()`
- 改敌人类型时要同时核对物编 ID 是否存在
- 新增“某波独有机制”时，优先挂在波次配置或敌人 runtime info，不要散落成硬编码条件
