# 主循环与状态机

## 模块职责

说明 `entry_runtime.lua` 如何用 `STATE` 和定时循环驱动整局游戏。

## Source Of Truth

- 状态容器：`maps/EntryMap/script/entry_runtime.lua`
- 配置源：`maps/EntryMap/script/entry_config.lua`

## 关键状态与数据流

- 战场状态：`STATE.hero`、`STATE.enemies`、`STATE.enemy_alive_count`
- 主线状态：`STATE.current_wave_index`、`STATE.started_wave_count`、`STATE.active_wave`
- 挑战状态：`STATE.active_challenges`、`STATE.challenge_charges`、`STATE.challenge_recover_elapsed`
- 成长状态：`STATE.hero_progress`、`STATE.skill_points`、`STATE.attack_skill_state`
- 羁绊状态：`STATE.bond_runtime`
- UI/调试状态：`STATE.gm_ui`、`STATE.dev_commands_registered`

## 当前已实现行为

- `reset_state()` 会重建整局必要状态
- `start_runtime_loops()` 至少驱动以下周期逻辑：
  - `update_wave(0.25)`
  - `update_challenges(0.25)`
  - `update_challenge_charges(0.25)`
  - `update_bond_effects(0.25)`
  - 攻击技能与普攻更新
  - 被动资源更新
  - GM 面板刷新
- `finish_game()` 会终止主局状态并显示胜负结果

## 状态机重点

- 启动即进入第 1 波，不存在独立大厅态
- 每波由“主怪阶段 + Boss 登场后并行压力”构成
- 挑战和主线并行，不需要清场切态
- 切到下一波后，旧波残留怪仍可能存在，但不会继续按旧波追加新批次

## 未实现或占位项

- 没有统一的中央决策队列状态机
- 没有成长武器、宝物、烙印等独立 runtime context
- 没有局外成长/存档驱动的跨局状态机

## Agent 修改建议与风险点

- 想加新局内系统，优先先定义 `STATE` 的归属和生命周期
- 不要让 UI 自己保存真状态，UI 只读 `STATE`
- 新增循环时优先归并进现有 loop，避免分散为大量孤立 timer
