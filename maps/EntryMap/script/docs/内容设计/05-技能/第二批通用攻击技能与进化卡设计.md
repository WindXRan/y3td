# 第二批通用攻击技能与进化卡设计

> 文档类型：`系统策划文档`
> 文档目标：统一第二批通用攻击技能的强化卡结构、品质职责与终局规则，作为后续内容生产与文档维护的标准模板。
> 作用范围：只覆盖第二批通用攻击技能的强化卡体系，不覆盖当前运行时抽卡实现、施法实现与 UI 接线细节。

## 1. 系统定位

第二批通用攻击技能，承担首批五个技能之外的中后期内容扩展职责。

本批技能的设计目标不是简单增加技能数量，而是建立一套统一可复制的技能成长模板，让后续所有新技能都能按同一规则生产、审查和扩展。

玩家体验目标：

- 拿到技能后，前两张卡就能看到方向
- 中期通过品质提升形成明确分支
- 后期通过终局牌完成技能毕业
- 每个技能都具备独立成核能力，不依赖特定其他技能才能成立

## 2. 系统边界

本系统负责：

- 定义第二批技能的标准卡牌结构
- 定义四种品质的职责分工
- 定义单技能 `9 张强化牌 + 1 张终局牌` 的固定模板
- 定义终局牌出现前的成长约束

本系统不负责：

- 当前运行时抽卡概率实现
- 当前 Lua 技能施法逻辑
- 当前 UI 面板展示逻辑
- 具体特效、音效与物编资源接线

## 4. 核心规则

### 4.1 单技能固定结构

每个技能固定采用 `10 张卡` 结构：

- `普通`：`3` 张
- `优秀`：`3` 张
- `稀有`：`3` 张
- `传奇`：`1` 张

统一记法：

- `9 张强化牌`
- `1 张终局牌`

### 4.2 四品质职责

`普通`

- 负责基础增幅
- 用于建立稳定成长底座
- 只承接直接、易懂、泛用的强化

`优秀`

- 负责形态变化
- 用于拉开技能手感差异
- 承接范围、数量、轨迹、穿透、持续等中期分支

`稀有`

- 负责流派特化
- 用于建立异常、对单、对群、触发或联动标签
- 让技能从“能用”进入“能 Carry”

`传奇`

- 负责终局定型
- 每技能唯一
- 用于完成最终形态、终盘定位和毕业体验

### 4.3 原子化要求

除传奇牌外，单张卡默认只负责一类成长方向：

- 伤害
- 频率
- 范围
- 数量
- 形态
- 状态
- 特攻
- 触发

不允许把多种核心职责塞进同一张普通、优秀或稀有卡中。

传奇牌允许承担：

- 最终形态替换
- 终局机制升级
- 终盘强度抬升

### 4.4 终局牌规则

- 每个技能只有 `1` 张传奇终局牌
- 终局牌必须代表该技能的最终身份，而不是普通增伤卡
- 终局牌只允许拿 `1` 次
- 终局牌不能独占全部成长价值，技能本体和前置强化仍需承担中期强度

### 4.5 局内技能位规则

- 局内攻击技能位采用 `1+4` 结构：
  - `1` 个固定基础位
  - `4` 个自由装配位
- 固定基础位默认由 `普攻` 占用。
- 自由装配位用于承载局内获得的通用攻击技能。
- 第二批通用攻击技能与首批技能共用同一套自由装配位规则。

### 4.6 局内成长节奏规则

- 单局目标时长按 `30 分钟` 爽局节奏设计。
- 局内成长不设置固定满级。
- 成长节奏采用前快后慢的递增经验结构，而不是固定等级上限。
- 真正限制成长上限的因素为：
  - 技能位数量
  - 技能卡池长度
  - 终局牌唯一性

推荐节奏：

- `0-5 分钟`
  应完成基础输出建立，并开始拿到第 `2` 个技能
- `5-10 分钟`
  应完成第 `3` 个技能装配，主技能开始形成方向
- `10-15 分钟`
  应完成第 `4` 个技能装配，中期构筑进入成型段
- `15-20 分钟`
  应完成第 `5` 个技能装配，并开始出现第一张终局牌
- `20-30 分钟`
  继续升级但不封顶，奖励重心逐步转向补强、重复普通卡和终局后强化

## 6. 单技能标准模板

每个技能的 `9+1` 结构统一按以下模板生产：

- `普通 1`
  基础伤害成长
- `普通 2`
  基础频率成长
- `普通 3`
  基础功能成长
- `优秀 1`
  范围成长
- `优秀 2`
  数量成长
- `优秀 3`
  形态成长
- `稀有 1`
  状态特化
- `稀有 2`
  对单或对高威胁目标特化
- `稀有 3`
  特殊触发或联动特化
- `传奇 1`
  终局形态
