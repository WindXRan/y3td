# 套装总表

> 本文用于管理装备套装方向和套装字段结构，不填写具体套装条目。

## 1. 套装职责

- 套装用于强化职业与 build 的方向感。
- 套装效果允许同时存在：
  - `纯数值强化`
  - `小幅机制强化`
- 套装的机制强化应低于高稀有遗物的机制权重。

## 2. 套装结构建议

| 字段 | 含义 | 说明 |
| --- | --- | --- |
| `set_id` | 套装 ID | 唯一标识 |
| `display_name` | 套装名称 | 玩家可见名称 |
| `theme_tags` | 主题标签 | 例如 `暴击`、`召唤`、`站场` |
| `slot_distribution` | 分布部位 | 套装覆盖哪些部位 |
| `set_effect_2` | 两件套效果 | 基础方向强化 |
| `set_effect_4` | 四件套效果 | 更强的方向强化或轻机制 |
| `generic_pool_flag` | 是否通用套装 | 用于区分卡池通用套装 |
| `status` | 当前状态 | `设计中 / 未实现 / 部分实现 / 已实现` |

## 3. 套装方向建议

- `站场生存套`
- `高速直伤套`
- `持续伤害套`
- `召唤协同套`
- `技能循环套`
- `触发连锁套`

## 4. 设计边界

- 两件套更适合做“泛用基础方向”。
- 四件套更适合做“build 放大器”。
- 通用卡池套装应更泛用，但不能强过刷图专精套装。
