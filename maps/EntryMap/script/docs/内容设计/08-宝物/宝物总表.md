# 宝物总表

> 宝物属于局内构筑系统，以 `maps/EntryMap/script/entry_objects/treasures` 为当前实现事实来源；本表负责记录当前局内宝物池、分类与联动标签。

## 1. 文档职责

- 记录当前已实现的局内宝物条目。
- 区分 `通用常驻`、`路线放大` 与后续要补的 `战术临时宝物`。
- 明确 `宝物` 与 `遗物` 的系统边界：
  - `宝物`：局内构筑
  - `遗物`：局外长期成长

## 2. 建议记录字段

| 字段 | 含义 | 说明 |
| --- | --- | --- |
| `treasure_id` | 宝物 ID | 对应 `entry_objects/treasures/*.lua` |
| `display_name` | 宝物名称 | 玩家可见名称 |
| `quality` | 品质 | 当前 runtime 已实现品质 |
| `treasure_type` | 宝物类型 | `general / route_amp / tactical_temp` |
| `duration_type` | 持续方式 | `permanent / wave / timed / charges / next_boss / next_challenge` |
| `theme_tags` | 主题标签 | 例如 `economy`、`survival`、`basic_attack` |
| `effect_summary` | 效果摘要 | 直接使用运行时摘要或文档摘要 |
| `best_with_tags` | 适配标签 | 技能、羁绊、场景标签 |
| `timing_tags` | 生效时机标签 | 例如 `immediate`、`persistent` |
| `status` | 当前状态 | `已实现 / 设计中` |

## 4. 当前池结论

- 当前 runtime `12` 件宝物全部是 `常驻生效`，没有真正的 `战术临时宝物`。
- 当前池更偏 `通用补强 + 局部路线放大`，这和本轮设计方向一致。
- 后续新增宝物时，应优先补齐：
  - `tactical_temp`
  - `next_boss`
  - `next_challenge`
  - `wave`
  这几类时机性设计。
