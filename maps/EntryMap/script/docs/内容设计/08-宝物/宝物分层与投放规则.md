# 宝物分层与投放规则

## 1. 系统边界

- `宝物` 是局内构筑内容。
- `遗物` 是局外长期成长内容。
- 本文只讨论局内 `宝物`，不覆盖 `遗物`。

## 2. 分层定义

### 2.1 `general`

- 适合大多数构筑的稳定强化。
- 优先提供：
  - 攻击
  - 攻速
  - 技能伤害
  - 续航
  - 经济
  - Boss 增伤

### 2.2 `route_amp`

- 只在特定路线中收益更高，但离开路线也不至于彻底废牌。
- 首轮重点支持：
  - `basic_attack`
  - `chain`
  - `boss`
  - `skill`
  - `element`

### 2.3 `tactical_temp`

- 用于制造“这一波 / 这次挑战 / 下个 Boss”里的临时高潮。
- 推荐的 `duration_type`：
  - `wave`
  - `timed`
  - `charges`
  - `next_boss`
  - `next_challenge`

## 3. 投放占比建议

| 类型 | 建议占比 | 说明 |
| --- | --- | --- |
| `general` | `60%` | 局内保底与泛用补强 |
| `tactical_temp` | `25%` | 节奏强化与波次决策 |
| `route_amp` | `15%` | 少量构筑放大件 |

## 6. 维护规则

- 当前 runtime 已实现宝物必须以 `entry_objects/treasures/*.lua` 为准。
- 后续新增战术型宝物时，必须同时写：
  - `treasure_type`
  - `duration_type`
  - `timing_tags`
  - `best_with_tags`
- 若某件宝物会与 `羁绊` 或 `技能强化` 形成强耦合，应优先写成 `route_amp`，而不是强行写成 `general`。
