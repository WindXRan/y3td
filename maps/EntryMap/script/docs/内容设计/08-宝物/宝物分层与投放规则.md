# 宝物分层与投放规则

> 当前事实：战斗内宝物使用 `maps/EntryMap/script/data_csv/treasures.csv`。旧版按 `general / route_amp / tactical_temp` 划分的运行时宝物池已废弃；本文保留分层规则时，必须明确哪些是 `当前事实`，哪些只是 `历史方案`。

## 1. 文档目标

- 说明当前宝物投放事实与历史分层方案的边界。
- 防止后续继续把已废弃的旧版单文件宝物池当成当前运行规则。
- 为后续重建设计预留稳定口径。

## 2. 当前事实

- 当前战斗内宝物唯一事实源：`maps/EntryMap/script/data_csv/treasures.csv`
- 当前运行时兼容层入口：
  - `maps/EntryMap/script/data/object_tables/treasure_catalog.lua`
  - `maps/EntryMap/script/data/object_tables/treasure_catalog_compat.lua`
- 当前没有独立生效的“旧版 runtime 宝物池”。
- 当前没有正式落表的 `general / route_amp / tactical_temp` 分类字段。

## 3. 当前实现状态

| 项目 | 当前状态 | 说明 |
| --- | --- | --- |
| 宝物主数据源 | `已实现` | 使用 `treasures.csv` |
| 图鉴兼容层 | `已实现` | 兼容旧运行时读取结构 |
| 旧版单文件宝物池 | `已废弃` | 不再作为事实来源 |
| 旧版临时宝物原型 | `未采用` | 未接入当前图鉴 |
| 正式投放分层字段 | `未建立` | 当前仍需后续重建 |

## 4. 当前投放口径

- 当前如需讨论“有哪些宝物”，以 [宝物总表](./宝物总表.md) 为准。
- 当前如需新增投放逻辑，应直接在 `treasures.csv` 或新增附表上扩展，不应恢复旧版 `entry_objects/treasures/*.lua` 结构。
- 当前如需新增运行时标签，应以显式字段落表，不应继续只靠文档中的旧标签名单维护。

## 5. 历史方案入口

- 旧版分层概念与历史临时宝物原型已迁移到 [宝物历史方案归档.md](./宝物历史方案归档.md)。
- 主文档不再直接保留旧 `treasure_id` 清单，避免与现行图鉴口径混淆。

## 6. 后续重建建议

- 若后续要恢复“分层投放”，建议新增正式字段，而不是恢复旧版宝物 ID 清单。
- 若后续要恢复“临时宝物”，应直接落到现行 CSV 体系，并明确：
  - 生效方式
  - 持续方式
  - UI 展示字段
  - 与奖励系统的真实关系
