# 内容设计索引

> 本目录用于沉淀“具体内容池”，和 `design/初步设计`、`项目模块`、`开发进度与计划` 形成分层。

## 目录定位

- `design/初步设计`
  - 负责系统规则、设计原则、数值方向与总纲定案。
- `内容设计`
  - 负责具体内容条目与内容池本身，例如职业、装备、遗物、羁绊、敌人、Boss、掉落池、卡池。
- `项目模块`
  - 负责说明代码结构、模块职责与实现落点。
- `开发进度与计划`
  - 负责说明哪些内容已经落地、哪些还在设计或部分实现。

## 维护规则

- 当 `系统规则` 变化时，优先修改 `design/初步设计`，再同步这里的内容条目规则。
- 当新增或调整 `具体内容` 时，优先修改本目录，再按需要回写 `开发进度与计划`。
- 当内容已经进入真实 runtime 时，再同步 `项目模块` 和 `开发进度与计划` 的实现说明。
- 本目录优先记录“内容本身是什么”，不替代代码实现文档。
