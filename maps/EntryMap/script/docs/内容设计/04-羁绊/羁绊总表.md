# 羁绊总表

> 羁绊属于局内构筑系统，但实际羁绊条目很多，单独放在内容设计目录中管理更方便。

## 1. 文档职责

- 记录所有羁绊条目的内容池。
- 记录羁绊阈值、效果方向、适配 build 与候选池信息。
- 不替代 `项目模块/03-战斗与成长/羁绊系统.md` 的实现说明。

## 2. 建议记录字段

| 字段 | 含义 | 说明 |
| --- | --- | --- |
| `bond_id` | 羁绊 ID | 运行时与配置引用 |
| `display_name` | 羁绊名称 | 玩家可见名称 |
| `theme_tags` | 主题标签 | 例如 `火焰`、`召唤`、`暴击` |
| `thresholds` | 阈值配置 | 例如 `2/4/6` |
| `effects_by_threshold` | 各阈值效果 | 对应具体强化内容 |
| `candidate_pool_rule` | 进入候选池规则 | 用于抽卡与刷新逻辑 |
| `synergy_tags` | 联动标签 | 适配哪些职业、技能、宝物 |
| `status` | 当前状态 | `设计中 / 未实现 / 部分实现 / 已实现` |

## 3. 维护建议

- 当羁绊数量增多后，建议按主题继续拆分子文档，例如：
  - `元素系羁绊.md`
  - `攻击系羁绊.md`
  - `召唤系羁绊.md`
- 羁绊数值或阈值调整后，应同步回看：
  - `攻击技能`
  - `宝物`
  - `烙印`
  - `职业标签`
