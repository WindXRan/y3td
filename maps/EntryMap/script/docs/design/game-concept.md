## 游戏概述

**游戏类型**: Hero Auto-Battle Wave Defense（英雄自动战斗 + 波次防守 + 构筑 Roguelike）

**引擎**: Y3 (魔兽争霸3复刻编辑器)

**一句话描述**: 选择英雄进入战场，在波次推进中通过羁绊抽卡、装备词条、属性选择和进化专精四层构筑打造独一无二的战斗流派，抵挡越来越强的怪物浪潮。

## 三大设计支柱

| 支柱 | 说明 | 实现体现 |
|------|------|---------|
| **策略构筑** | 羁绊 + 装备词条 + 属性选择 + 进化专精四层协同，形成独特构筑方向。玩家的核心乐趣在于"构筑成型后的碾压爽感" | 羁绊抽卡(F)、成长武器词条、属性四选一、猎手专精 |
| **自动战斗观赏** | 英雄自动攻击和释放技能，玩家从战术操作中解放，专注于构筑决策和战斗观赏 | 自动攻击 + 攻击技能自动施放、弹道特效、伤害数字、固定视角 |
| **波次压力递增** | 越来越强的敌人波次，Boss 登场制造高压节点，检验构筑质量。打不过说明构筑方向需要调整 | 多波怪物 → Boss 波、精英怪、挑战试炼 |

## 游戏模式说明

**混合模式**（Roguelike 构筑 + 持久养成）：

- **局内**：每局从初始状态开始，通过波间选择动态构筑（羁绊/装备/属性/进化），构筑在结算后重置
- **局外**：部分资源和进度跨局保留，支持长线英雄养成和属性积累
- **关卡**：多关卡可选，不同关卡有独立的敌人配置和难度模式

## 设计原则

1. **构筑优先于操作** — 玩家的时间花在"选择"上而非"操作"上，自动战斗是手段不是目的
2. **协同大于堆叠** — 羁绊、装备、属性、进化四层系统的协同收益大于单层数值堆叠
3. **波次节奏明快** — 波间选择窗口不宜过长，保持战斗节奏紧凑
4. **失败有反馈** — 战斗失败应能追溯到构筑决策的不足，而非纯粹的数值碾压
