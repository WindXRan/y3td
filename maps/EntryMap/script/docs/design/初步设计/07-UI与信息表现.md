# UI 与信息表现

> 从《项目主 GDD（初版）》拆分而来，便于按专题维护。

## 11. UI 与信息表现

### 11.1 局内 UI 总原则
- 局内 UI 固定按 `顶部战斗信息 + 左侧快捷提示 + 中央弹窗 + 右侧追踪区 + 底部角色操作区 + 右下开关区` 的结构布局。
- 战斗中常驻显示的信息只保留“当前决策必需”的内容，深度信息通过属性面板、技能详情面板、背包面板展开。
- 所有局内 UI 文案统一使用本 GDD 中已经定下的字段名称，不再额外创造别名。
- 首版 UI 以功能完整和信息清晰为目标，不追求复杂装饰。

### 11.2 顶部战斗状态栏
顶部中间固定显示本局核心战斗节奏信息：
- 当前战斗计时。
- 当前波次，固定显示为 `第 X / 5 波`。
- 当前波次对应的 `Boss 来临倒计时 / Boss 信息`。
- 当前波次剩余怪物阶段倒计时。
- 当前关卡节点编号，首版采用 `主线 1-1` 格式。

Boss 信息栏补充规则：
- 顶部同一块区域始终用于显示 `本波 Boss 信息`，不额外切换成独立的 Boss 预警组件。
- 在 Boss 尚未登场前，这块区域显示 `Boss 来临倒计时`，并可附带本波 Boss 的 `名称` 或 `头像缩略图`。
- 当 Boss 实际登场后，这块区域改为显示当前 Boss 的生命信息或简版 Boss 状态，但仍沿用同一块 UI 区域。
- Boss 被击败后，这块区域短暂显示 `Boss 已击败`，随后在下一波开始时无缝切换为下一波 Boss 的来临倒计时。

### 11.3 顶部资源栏
顶部右侧固定显示本局资源栏：
- `金币`：当前持有数量。
- `木材`：当前持有数量。
- `杀敌数`：当前局累计击杀数量。
- 每个资源条目下方额外显示当前局对应的 `固定增益值` 与 `百分比增益值`。
- 资源栏是战斗主 HUD 的一部分，始终可见，不折叠。

### 11.4 玩家统计栏
顶部右侧资源栏下方固定显示简版玩家统计板：
- 玩家名称。
- 当前累计伤害。
- 当前累计杀敌。
- 当前复活次数。
- 多人模式下按玩家逐行展示；单人模式下保留同样结构，不单独另做一套面板。

### 11.5 左侧系统按钮与快捷键提示
左上与左侧固定承担系统入口和快捷键教学：
- 左上固定放置 `退出游戏` 按钮。
- 左上固定放置 `设置` 按钮。
- 左侧固定显示快捷键提示列表。
- 首版固定显示的快捷键提示包括：`F1 选中英雄`、`F2 返回阵地`、`F4 清除集火`、`F 羁绊抽卡`、`G 技能加点`、`B 打开背包`、`TAB 查看属性`。
- 允许玩家关闭快捷键提示显示，但默认开启。

战场控制快捷键补充说明：
- `F1 选中英雄`：将镜头与当前操作焦点快速切回英雄。
- `F2 返回阵地`：将镜头快速定位回左侧核心防线区域。
- `F4 清除集火`：取消当前手动指定的集火目标，恢复默认自动索敌。

### 11.6 攻击技能强化面板、技能详情面板、羁绊抽卡面板、宝物候选面板与烙印选择面板
局内中央弹窗固定分为五套面板：

攻击技能强化面板：
- 英雄升级后不再直接弹出奖励，而是增加 `1 个技能点`。
- 玩家主动消耗技能点时，弹出 `3 张攻击技能强化候选卡牌`。
- 玩家可以点击角色操作区中的 `技能` 按钮，或直接按下快捷键 `G` 打开攻击技能强化面板。
- 当当前技能点小于 `1` 时，`技能` 按钮与快捷键 `G` 均不可用，并提示 `技能点不足`。
- `G` 打开的不是单技能专属抽卡，而是一次通用的 `攻击技能强化 3 选 1`。
- 本次 `3 张候选卡` 中，既可能出现 `新攻击技能卡`，也可能出现 `已有攻击技能强化卡`。
- 若当前仍有空闲攻击技能位，则候选中有概率出现 `奥术箭 / 爆炎箭 / 寒冰箭 / 天雷` 这类 `新攻击技能卡`。
- 若当前 `4 个攻击技能位` 已全部装满，则后续候选只出现 `已有攻击技能强化卡`。
- 若玩家连续多次未看到 `新攻击技能卡` 且当前技能位仍未装满，面板需要按规则触发保底，确保下一次候选中至少出现 `1 张新攻击技能卡`。
- 首版面板表现上默认每轮至多展示 `1 张新攻击技能卡`，其余位置优先展示 `已有技能强化卡`，避免单轮路线分叉过重。
- 若玩家刚刚获得了一个新的攻击技能，则后续数次 `G` 强化面板应更容易出现该技能的强化卡，让玩家快速建立“拿到技能就能继续做强”的认知。
- 攻击技能强化面板与其他中央弹窗互斥；当 `G` 面板已经存在 `未完成选择` 的候选时，再次按下 `G` 或点击 `技能` 按钮，不会再次消耗技能点，而是直接回到这组尚未结算的候选。
- 每张强化卡都必须额外显示 `所属技能标签`，让玩家明确知道它强化的是哪一个攻击技能。
- 每张卡牌必须显示 `名称`、`图标`、`是否为 NEW`、`核心效果描述`。
- 攻击技能强化候选卡的卡面信息固定分为三层：
  `共通字段`
  `新攻击技能卡专属字段`
  `已有攻击技能强化卡专属字段`

共通字段：
- `名称`
- `图标`
- `品质`
- `是否为 NEW`
- `所属技能标签`
- `核心效果描述`
- `类型标签`
- `前置提示`
- `排斥提示`

`新攻击技能卡` 专属字段：
- 需要额外显示 `新技能` 标签，避免与普通强化卡混淆。
- 需要显示该技能的 `基础定位`，例如 `直线穿透`、`范围爆炸`、`控制减速`、`雷击连锁`。
- 需要显示该技能装配后将进入的 `技能位`，首版文案固定为 `将装配到 X 号技能位`。
- 需要显示该技能的 `基础释放方式 / 基础效果` 简述，让玩家知道拿到后会怎么出手。
- 若该技能与当前已选路线存在明显联动，可显示 `推荐联动标签`，例如 `适合普攻联动`、`适合技能爆发`、`适合控制流`。

`已有攻击技能强化卡` 专属字段：
- 需要额外显示该卡当前强化的是哪个技能，例如 `强化：普攻`、`强化：奥术箭`。
- 需要显示该强化在对应技能树中的 `分类标签`，例如 `基础强化`、`形态变化`、`路线强化`、`高阶派生`、`顶级专精`。
- 需要显示 `前置是否已满足` 与 `当前是否为可直接提升` 的状态文案，避免玩家误读成长路径。
- 若该卡会直接补齐某条技能路线关键节点，则卡面需要附带 `可成型` 或 `关键强化` 提示。
- 若该卡与玩家当前主路线明显不兼容，则卡面可以附带较弱的 `偏离当前路线` 提示，但不应强行阻止选择。

攻击技能强化候选卡文案模板：
- 首版统一按 `标题区 -> 标签区 -> 核心描述区 -> 补充说明区` 的顺序排布，避免不同卡牌各写各的。
- 标题区固定显示：
  `卡牌名称`
  `所属技能`
  `品质边框 / 品质色`
- 标签区固定显示 2 到 4 个短标签，优先级为：
  `新技能 / 强化类型`
  `基础强化 / 形态变化 / 路线强化 / 高阶派生 / 顶级专精`
  `可成型 / 关键强化 / 偏离当前路线`
  `推荐联动`
- 核心描述区固定只写 `1 到 2 行`，优先使用“动词 + 结果”的句式，不写长段背景说明。
- 补充说明区用于显示：
  `将装配到 X 号技能位`
  `前置已满足 / 前置未满足`
  `排斥：XXX`
  `当前已拥有`
  这类状态型说明。

文案写法规则：
- 标题只写卡牌名，不在标题中重复数值。
- `所属技能` 单独显示，不和标题混写。
- 数值描述优先写在核心描述区，不塞进标签区。
- 标签区只放分类与决策信息，不放长句。
- 若一张卡的收益一眼不明显，优先在核心描述里说明“它改变了什么”，再说明“加了多少”。
- 若是 `新攻击技能卡`，核心描述优先写该技能的攻击方式与定位。
- 若是 `已有攻击技能强化卡`，核心描述优先写它对当前技能造成的直接变化。

`新攻击技能卡` 示例模板：
- 标题：
  `奥术箭`
- 所属技能：
  `新技能`
- 标签：
  `新技能`
  `技能爆发`
  `适合普攻联动`
- 核心描述：
  `获得奥术箭，射出 1 支奥术箭攻击目标，造成能量魔法伤害。`
  `将装配到 2 号技能位。`
- 补充说明：
  `定位：直线穿透`

`已有攻击技能强化卡` 示例模板：
- 标题：
  `箭矢齐射`
- 所属技能：
  `奥术箭`
- 标签：
  `路线强化`
  `关键强化`
- 核心描述：
  `奥术箭数量 +1，伤害 -20%。`
  `显著提高对群覆盖能力。`
- 补充说明：
  `前置已满足`
- 点击卡牌即完成本次攻击技能强化。

技能详情面板：
- 玩家点击底部角色操作区中的任意一个 `攻击技能位` 时，打开对应技能的 `技能详情面板`。
- 技能详情面板用于查看该技能当前 `等级`、`伤害类型`、`伤害系数`、`攻击范围`、`攻击间隔`、`已解锁卡牌列表` 与当前路线走向。
- 面板顶部固定显示当前已装配的 `4 个攻击技能位` 页签，支持在详情面板内快速切换查看其他技能。
- 若当前没有明确目标技能，则默认打开 `1 号位普攻` 的详情页。
- 技能详情面板不提供 `升级按钮`、`抽卡按钮` 与 `重置按钮`；所有成长入口统一回到角色操作区中的 `技能(G)`。
- 若当前已经存在 `未完成选择` 的 `G` 面板或 `F` 面板候选，则技能详情面板不能覆盖其决策弹窗；玩家需要先完成当前候选选择或先执行 `稍后选择`。

羁绊抽卡面板：
- 羁绊抽卡面板由玩家主动点击角色操作区中的羁绊抽卡按钮打开，用于抽取带有羁绊名称的卡牌。
- 玩家也可以直接按下快捷键 `F` 打开羁绊抽卡面板。
- 每次抽卡固定消耗 `100 木材`。
- 玩家点击 `羁绊抽卡` 按钮且木材足够时，立即扣除本次抽卡所需的 `100 木材`，随后生成本轮 `3 张候选卡`。
- 玩家按下快捷键 `F` 且木材足够时，也按完全相同的规则立即扣除 `100 木材` 并打开本轮面板。
- 固定展示 `3 张羁绊候选卡牌`。
- 每张羁绊卡必须显示 `品质`、`羁绊名称`、`当前进度`、`触发需求`、`基础效果`、`激活后的唯一羁绊效果`。
- 面板底部固定显示 `稍后选择` 与 `刷新` 按钮。
- `刷新` 按当前规则重抽 3 张羁绊候选卡牌。
- `刷新` 按钮旁必须明确显示当前刷新消耗与本轮剩余免费刷新次数。
- 面板内需要常驻显示当前 `7 个未被吞噬的羁绊卡位` 及其占用情况。
- 当玩家已经持有 `7 张未被吞噬的羁绊卡` 时，选择新卡后必须立刻进入 `替换 / 吞噬` 流程，从当前生效卡中选出 `1 张` 被替换卡。
- 替换流程中需要明确标识被替换卡将 `失去基础效果`、`不再计入羁绊进度`，避免玩家误解为只是移入背包。
- 替换流程中需要提供 `系统推荐替换目标`，并展示替换前后对应羁绊的激活状态变化。
- 羁绊抽卡面板默认与攻击技能强化面板一样采用 `一次打开 -> 展示 3 张候选 -> 玩家立即选择` 的节奏，只是消耗资源从 `技能点` 改为 `木材`。
- 若玩家点击 `稍后选择` 关闭面板，则当前这组 `3 张候选卡`、本轮已支付的 `100 木材` 与当前 `刷新次数/刷新费用阶梯` 全部保留；再次按下 `F` 或点击 `羁绊抽卡` 按钮时，直接回到这组未结算候选，不会重新扣费或重抽。
- 在存在 `未完成选择` 的羁绊候选时，玩家不能开启新一轮羁绊抽卡，也不能通过其他入口覆盖这组待选候选。

宝物候选面板：
- 当玩家完成 `宝物挑战` 或获得其他有效宝物奖励时，立即弹出 `宝物候选面板`。
- 宝物候选面板固定展示 `3 件宝物候选`，玩家必须 `3 选 1`。
- 若当前宝物位未满 `3 件`，则选择后直接放入空位。
- 若当前宝物位已满 `3 件`，则选择后立刻进入 `宝物替换` 流程，从当前持有的 `3 件宝物` 中替换 `1 件`。
- 宝物候选面板不消耗 `技能点`、`木材` 或 `金币`，它的成本已经体现在宝物挑战或奖励来源本身。
- 宝物候选面板与 `G`、`F`、烙印面板一样属于中央决策弹窗；存在未结算宝物候选时，其他中央决策入口不可覆盖它。
- 若玩家临时关闭宝物候选面板，则当前这组宝物候选进入 `待选择` 状态；再次点击 `宝物` 按钮时，优先回到该组未结算候选。
- 每件宝物候选固定显示 `名称`、`品质`、`核心效果`、`类型标签`、`推荐路线标签`。
- 若当前选择该宝物会触发满位替换，则卡面需要额外显示 `将触发替换`，并提示系统推荐的替换目标。

宝物候选文案模板：
- 首版统一按 `标题区 -> 标签区 -> 核心描述区 -> 补充说明区` 排布，和技能卡、羁绊卡保持同一阅读逻辑。
- 标题区固定显示：
  `宝物名称`
  `品质边框 / 品质色`
- 标签区固定显示 2 到 4 个短标签，优先级为：
  `普通宝物 / 稀有宝物 / 史诗宝物`
  `普攻 / 技能 / 生存 / 经济 / 终盘`
  `可联动 / 当前适配 / 将触发替换`
- 核心描述区固定只写 `1 到 2 行`，优先说明该宝物改变了什么机制或提供了什么关键收益。
- 补充说明区用于显示：
  `选择后放入空位`
  `选择后需要替换`
  `推荐替换目标`
  这类状态型文案。

羁绊候选卡文案模板：
- 首版统一也按 `标题区 -> 标签区 -> 核心描述区 -> 补充说明区` 的顺序排布，与攻击技能强化卡保持同一阅读节奏。
- 标题区固定显示：
  `羁绊名称`
  `品质边框 / 品质色`
  `当前有效数量 / 触发需求`
- 标签区固定显示 2 到 4 个短标签，优先级为：
  `普通羁绊 / 稀有羁绊 / 史诗羁绊`
  `续航 / 经济 / 普攻 / 技能 / 成长 / 防守`
  `可成套 / 已成型 / 将触发替换 / 高风险替换`
  `推荐流派`
- 核心描述区固定只写 `1 到 2 行`，优先展示：
  `这张卡自己的基础效果`
  `该羁绊成套后的唯一羁绊效果`
- 补充说明区用于显示：
  `当前进度`
  `替换后变化`
  `是否会失去已激活羁绊`
  `推荐替换目标`
  这类状态型说明。

羁绊卡文案写法规则：
- 标题只写 `羁绊名称`，不把具体基础效果写进标题。
- 当前进度统一写成 `当前有效数量 / 触发需求`，例如 `2/3`、`3/4`。
- 核心描述第一句优先写 `基础效果`，第二句再写 `成套收益`。
- 标签区优先告诉玩家“这套羁绊大致是干什么的”，不在标签区塞长数值句子。
- 若该卡能直接补齐一套羁绊，必须显示 `可成套` 标签。
- 若选择该卡后因为满 `7 张` 需要拆掉现有羁绊，且存在明显损失，必须显示 `将触发替换` 或 `高风险替换` 标签。
- 若某张羁绊卡属于当前局已明显成型路线，可显示 `推荐流派` 标签，例如 `适合普攻流`、`适合经济流`、`适合站场流`。

羁绊候选卡字段结构：
- 共通字段：
  `羁绊名称`
  `品质`
  `当前进度`
  `触发需求`
  `基础效果`
  `唯一羁绊效果`
  `类型标签`
  `状态标签`
- 替换相关字段：
  `是否需要替换`
  `推荐替换目标`
  `替换后进度变化`
  `是否会导致已激活羁绊失效`

羁绊候选卡示例模板：
- 标题：
  `连射`
- 进度：
  `2/3`
- 标签：
  `稀有羁绊`
  `普攻`
  `可成套`
  `适合普攻流`
- 核心描述：
  `基础效果：多重伤害 +15%。`
  `成套效果：多重数量 +1，且多重伤害 +35%。`
- 补充说明：
  `选择后将由 2/3 提升至 3/3`
  `无需替换`

满位替换示例模板：
- 标题：
  `成长`
- 进度：
  `3/4`
- 标签：
  `史诗羁绊`
  `成长`

  `高风险替换`

  `成套效果：每击杀 20 个敌人，永久获得攻击力 +12。`

烙印选择面板：
- 英雄达到 `10 / 20 / 30 / 40 级` 时立即弹出 `烙印选择面板`。
- 烙印选择面板固定展示 `3 张烙印候选卡牌`。
- 每张烙印卡必须显示 `品质`、`烙印名称`、`核心效果`、`定位标签`。
- 烙印面板不提供手动刷新，也不提供跳过选项，必须当场完成 `3 选 1`。
- 烙印面板关闭后，所选烙印立即生效，并写入 `吞噬界面`。

背包面板：
- 玩家按下快捷键 `B` 时，打开 `背包面板`。
- 背包面板首版固定拆为 `成长武器`、`宝物`、`奖励记录` 三个页签。
- `成长武器` 页签用于查看当前成长武器的 `等级`、`品质`、`基础效果` 与已获得词缀列表。
- `成长武器` 页签同时承担 `金币强化` 与 `待选词缀处理` 入口，固定显示 `当前等级`、`下一次强化费用`、`距下一个词缀节点还差多少级`。
- 当成长武器当前存在 `待选词缀` 时，`成长武器` 页签需要显示明确的 `待处理词缀` 提示，并暂时隐藏继续强化入口，避免玩家误以为还能越过节点继续升级。
- `宝物` 页签用于查看当前 `3 个生效中宝物位`、宝物效果与已弃置宝物记录。
- `奖励记录` 页签用于查看本局已经获得过的重要奖励摘要，首版优先收录 `宝物`、`高品质词缀节点结果`、`宝物挑战奖励` 与 `精英/Boss 的高价值奖励结算`。
- 当当前存在 `未完成选择` 的 `G`、`F`、`宝物候选`、`烙印` 面板时，`B` 不能覆盖这些中央决策弹窗；玩家需要先完成当前选择或返回该组待选候选。

### 11.7 右侧任务追踪区
右侧中部固定显示任务追踪区，采用两层结构：
- `主线任务` 固定归属于 `爬塔挑战玩法`，用于显示当前关卡目标、完成进度与奖励。
- 主线任务区固定显示 `任务名`、`目标进度`、`奖励内容`。
- 主线任务区下方固定提供 `自动任务` 开关。
- `新手引导` 只在新手期出现，用于提示当前必须完成的教学动作。
- 新手期结束后，`新手引导` 模块永久关闭，不再常驻显示。

`自动任务` 开关规则：
- `自动任务` 只影响右侧任务追踪区的显示逻辑，不代替玩家操作，也不自动帮玩家抽卡、加点、开挑战或移动英雄。
- 开启后，任务追踪区默认自动置顶当前最高优先级目标，并在目标变化时自动展开对应条目。
- 关闭后，任务追踪区保持玩家上一次手动展开/折叠状态，不再因为目标变化强制切换焦点。
- 首版默认 `开启`，帮助新玩家减少信息筛选成本。

`新手引导` 首版流程：
- `新手引导` 只在账号或本地存档的 `首次标准模式对局` 中出现，后续对局默认关闭。
- 首版固定拆为 `4 步`，每次只追踪 `1 个当前必须完成的动作`：
  `第 1 步`：获得首个 `技能点` 后，提示玩家按 `G` 或点击 `技能` 按钮完成第一次攻击技能强化。
  `第 2 步`：首次拥有不少于 `100 木材` 时，提示玩家按 `F` 或点击 `羁绊抽卡` 按钮完成第一次羁绊抽卡。
  `第 3 步`：首次获得烙印节点时，提示玩家完成一次烙印 `3 选 1`。
  `第 4 步`：首次拥有 `7 张未被吞噬羁绊卡` 并获得新羁绊候选时，提示玩家完成一次 `替换 / 吞噬` 决策。
- `新手引导` 只在对应条件第一次满足时出现，不会因为玩家之后再次满足同条件而重复弹出。
- 当 `G / F / 烙印` 这类中央决策面板已经弹出时，新手引导只保留右侧高亮提示，不额外叠加居中弹窗，避免遮挡选择。
- 每一步在玩家实际完成对应操作后立刻结算完成，并切换到下一步；若当前局未走到后续条件，则剩余步骤顺延到下一局继续完成。
- 当 `4 步` 全部完成后，将该存档标记为 `新手引导已完成`，后续标准模式与挑战模式都不再显示该模块。

### 11.8 底部常驻玩法入口
底部左侧固定显示局内常驻玩法入口：
- `金币挑战`
- `木材挑战`
- `经验挑战`
- `宝物挑战`
- 这 4 个入口属于正式局内常驻玩法，不是测试入口，也不是临时活动入口。
- 入口按钮需要显示当前是否可进入、当前挑战次数与下一次恢复信息。

首版统一规则：
- 这 `4 个挑战入口` 都属于 `局内限次支线挑战`，在同一局标准模式中与主线战斗并存，但不会替代 `5 波主线通关` 的胜负判定。
- 这 `4 个挑战入口` 共用同一套 `挑战次数` 资源；玩家点击任意一个挑战按钮时，都会消耗 `1 次挑战次数`。
- 首版建议 `挑战次数上限 = 3`，初始进入对局时持有 `1 次挑战次数`。
- 首版建议每经过 `120 秒` 自动恢复 `1 次挑战次数`，恢复时不能超过 `3 次` 上限。
- 首版允许 `金币挑战 / 木材挑战 / 经验挑战 / 宝物挑战` 多种挑战同时进行，只要玩家仍有剩余 `挑战次数`，就可以继续开启新的挑战。
- 同一种挑战在单次开启后，若该挑战怪物组尚未结算，则该挑战按钮不再重复开启第二组同名挑战怪物，避免同一种挑战无限叠层。
- 挑战入口按钮首版 `永不灰置`，即使当前不能真正开启挑战，也保持同一套可见样式与点击反馈。
- 当中央存在 `未完成选择` 的 `G`、`F`、烙印面板时，点击挑战按钮不直接开启挑战，而是提示 `请先完成当前选择`。
- 挑战入口的默认状态固定分为：
  `可进入`
  `进行中`
  `次数不足`

挑战次数规则：
- `挑战次数` 是局内循环恢复的功能次数，不写入局外存档，也不和 `金币 / 木材 / 技能点` 混用。
- 只要当前 `挑战次数 >= 1`，任一挑战且当前未处于同名挑战进行中的按钮都应显示为 `可进入`。
- 当 `挑战次数 = 0` 时，所有挑战按钮统一显示为 `次数不足`，并附带下一次恢复倒计时。
- 当 `挑战次数` 因时间恢复时，系统需要给出轻度提示，例如 `挑战次数 +1`，提醒玩家可以再次开启支线挑战。
- 当某个同名挑战已经在进行中时，点击该挑战按钮会提示 `该挑战进行中`，但按钮本身不变灰。
- 四类挑战在首版中默认全部开放，差异主要体现在奖励结构、危险标签和刷怪构成，而不是前置解锁。

进入与结算规则：
- 玩家点击某个可进入的挑战入口后，立即扣除 `1 次挑战次数`，并进入该挑战的 `准备中` 状态。
- 按钮点击后，在主战场内立即从该挑战对应的 `挑战刷新区域` 中随机刷出一组 `挑战怪物组`，不额外切换地图场景。
- 首版挑战统一采用 `限时事件` 形式接入主战场，避免单独开房间打断主线节奏。
- 每种挑战都使用自己固定的 `怪物配置` 与 `奖励配置`，不随当前主线波次变化。
- 每次挑战固定只有 `1 波挑战怪物`；不会因为主线来到第几波，就把挑战内容切成第 1 波、第 2 波、第 3 波版本。
- 这 `1 波挑战怪物` 可以按固定批次或固定时间点补入，但本质上仍属于同一次挑战的同一波内容，不再中途切换到第二波怪物。
- 每个挑战开始后固定持续 `60 秒`，期间右侧任务追踪区追加对应挑战目标、剩余怪物数与倒计时。
- 多种不同挑战的 `60 秒` 倒计时允许并行存在，互不打断，分别独立结算。
- 挑战怪物组只在当前挑战事件中存在；若 `60 秒` 到时仍未击败，则这组挑战怪物立即消失，本次挑战判定为失败。
- 挑战成功条件为：在 `60 秒` 内击败本次刷出的整组挑战怪物。
- 挑战成功后立即发放一笔 `大量奖励`，并结束当前挑战状态。
- 挑战失败后不返还已消耗的 `1 次挑战次数`，但玩家后续仍可在次数恢复后再次开启任意挑战。
- 挑战结束后，无论成功或失败，都只清理本次对应挑战的怪物组与未结算状态，不影响其他仍在进行中的挑战。

四类挑战的首版职责：
- `金币挑战`：点击后刷新 `金币挑战怪物组`，在 `60 秒` 内击败后发放一笔大量金币奖励。
- `木材挑战`：点击后刷新 `木材挑战怪物组`，在 `60 秒` 内击败后发放一笔大量木材奖励。
- `经验挑战`：点击后刷新 `经验挑战怪物组`，在 `60 秒` 内击败后发放一笔大量经验奖励。
- `宝物挑战`：点击后刷新 `宝物挑战怪物组`，在 `60 秒` 内击败后必定掉落 `1 组宝物候选` 或等价宝物奖励。

四类挑战的首版怪物模板：
- `金币挑战怪物组`：
  刷新定位：低血量、高移速、数量多的 `运财小鬼` 型怪物
  固定配置建议：`20 只`，分 `3 批` 刷出
  出生规则：在 `金币挑战刷新区域` 内随机分散刷出，表现为一群运财小鬼从侧翼涌入
  战斗目标：检验玩家 `清杂效率`
  风险特征：若放任不管，容易快速穿过火力网
  成功奖励建议：固定 `220 金币`
- `木材挑战怪物组`：
  刷新定位：中等数量、较高生命、带少量护甲的 `伐木巨兽` 型怪物
  固定配置建议：`10 只`，分 `2 批` 刷出
  出生规则：在 `木材挑战刷新区域` 内随机刷出，单只之间保留更大间距，体现体型和压迫感
  战斗目标：检验玩家 `稳定持续输出`
  风险特征：单体更硬，若 build 对单不足会超时
  成功奖励建议：固定 `60 木材`
- `经验挑战怪物组`：
  刷新定位：中速移动、密集成群的 `灵能团簇` 型怪物
  固定配置建议：`24 只`，分 `4 批` 匀速补入
  出生规则：在 `经验挑战刷新区域` 内持续随机补入，区域宽度可略大于金币挑战，制造铺场感
  战斗目标：检验玩家 `范围压制与持续清场`
  风险特征：数量多，容易把场面推满
  成功奖励建议：固定 `相当于 30 只普通怪的经验`
- `宝物挑战怪物组`：
  刷新定位：`1 名高威胁精英首领 + 6 到 10 只护卫怪`
  固定配置建议：`1 名首领 + 8 只护卫`，护卫分 `2 批` 刷出
  出生规则：首领从 `宝物挑战 Boss 区域` 随机一点登场，护卫从其周围的小范围区域随机分散出生
  战斗目标：检验玩家 `对精英 / 小 Boss 的集火能力`
  风险特征：首领高伤害、护卫会干扰锁定和走位压力
  成功奖励建议：
  必定获得 `1 组宝物候选`
  并额外附带 `少量金币或木材`

四类挑战的设计分工：
- `金币挑战` 负责补经济，不负责给构筑分叉。
- `木材挑战` 负责支撑羁绊抽卡节奏，是卡牌系统的重要外部木材来源。
- `经验挑战` 负责追等级与技能点，帮助中后期 build 追赶成型窗口。
- `宝物挑战` 负责给高价值功能组件，不直接给大量白字资源。

挑战奖励强度约束：
- 单次挑战成功奖励必须明显高于一小段常规刷怪收益，玩家才会觉得值得主动点开。
- 但单次挑战奖励不应高到完全碾压主线经济，避免玩法退化为“只等挑战次数”。
- `金币 / 木材 / 经验` 三类挑战采用固定奖励，不按 `当前波次` 提升，定位为稳定补给型玩法而不是波次倍率玩法。
- `宝物挑战` 的核心价值始终来自 `宝物候选` 本身，附带金币或木材只作为额外甜头。
- 由于挑战奖励固定，首版数值建议按 `中局可接受、前期有吸引力、后期不失效但也不碾压` 的原则来定。

挑战配表字段建议：
首版建议额外建立一张 `挑战表`，统一管理 4 种挑战内容：

| 字段 | 含义 |
| --- | --- |
| `challenge_id` | 挑战唯一 ID，如 `gold_trial`、`wood_trial` |
| `challenge_name` | 挑战名称 |
| `duration_sec` | 持续时间；首版固定为 `60` |
| `cost_charge` | 消耗挑战次数；首版固定为 `1` |
| `monster_group_id` | 对应挑战怪物组 ID |
| `spawn_area_id` | 对应挑战刷新区域 ID |
| `challenge_wave_count` | 挑战内部波次数；首版固定为 `1` |
| `spawn_batch_count` | 刷新批次数 |
| `spawn_total_count` | 总怪物数量 |
| `reward_type` | 奖励类型：`gold`、`wood`、`exp`、`treasure` |
| `reward_value_fixed` | 固定奖励值；不随主线波次变化 |
| `danger_tags` | 风险标签，如 `fast`、`tanky`、`elite` |
| `ui_icon` | 按钮与追踪区图标 |

`monster_group_id` 对应展开建议：
- `gold_trial_pack`：`运财小鬼 x20`，第 `0 / 12 / 24 秒` 各刷 1 批，优先从上侧翼切入。
- `wood_trial_pack`：`伐木巨兽 x10`，第 `0 / 18 秒` 各刷 1 批，整体间距较大，强调单体压力。
- `exp_trial_pack`：`灵能团簇 x24`，第 `0 / 10 / 20 / 30 秒` 各刷 1 批，持续铺场。
- `treasure_trial_pack`：`精英首领 x1 + 护卫 x8`，第 `0 秒` 刷出首领与 `4` 只护卫，第 `20 秒` 补 `4` 只护卫。

实现备注：
- `reward_value_fixed` 在 `gold / wood` 类型下直接写固定数值即可。
- `exp_trial` 建议在实现层把 `30_normal_kill_exp` 解释为 `30 只标准普通怪经验总和`，避免经验值改表时文档和数据脱节。
- `treasure_trial` 的 `bonus_resource` 首版建议在 `少量金币` 与 `少量木材` 中二选一，不额外再开一套稀有资源。

UI 表现规则：
- 每个入口按钮都需要同时显示 `名称`、`当前状态` 与当前共享的 `挑战次数 / 上限`。
- 当挑战处于 `进行中` 时，对应入口持续高亮，并在按钮上显示剩余时间。
- 若同时有多种挑战进行中，则每个对应按钮都分别显示自己的剩余时间与进行中状态，不互相覆盖成单一总状态。
- 当挑战可进入且玩家当前战力明显不足时，按钮允许显示弱提示 `高风险`，但不阻止玩家尝试。
- 当 `挑战次数 = 0` 时，入口按钮上需要直接显示下一次恢复倒计时，例如 `下次 +1：87s`。
- 即使处于 `次数不足`、`该挑战进行中` 等不可开启状态，按钮也不改成灰色，只通过角标、描边、状态字和点击后的 toast 提示当前原因。
- 若挑战奖励会溢出或无法立即领取，系统必须把奖励自动写入本局待领取列表，不允许因 UI 关闭导致奖励丢失。

### 11.9 底部角色操作区
底部中间到右侧固定显示角色操作区：
- 英雄头像、名称、等级、生命值。
- 当前经验条。
- 主属性与核心属性简版展示。
- 单成长武器格子。
- 玩家点击 `成长武器格子` 时，默认打开 `背包面板 -> 成长武器` 页签。
- 成长武器格子需要直接显示 `当前等级`，并在可强化时弱提示 `下一级费用`；若存在 `待选词缀`，则改为显示 `待选词缀` 状态。
- `4 个攻击技能位` 的快捷栏展示，其中 `1 号位` 为默认自带的 `普攻`。
- 玩家点击任意 `攻击技能位` 时，打开对应技能的 `技能详情面板`。
- 当前 `技能点` 数量需要在角色操作区可见。
- 额外功能按钮，包括 `宝物`、`技能`、`羁绊抽卡`、`集火`、`已吞噬` 等战斗操作入口。
- 这组按钮的首版固定顺序定义为：`技能(G)` -> `羁绊(F)` -> `宝物` -> `集火` -> `已吞噬`。
- `技能(G)` 与 `羁绊(F)` 作为首版最核心的局内构筑按钮，必须放在同一组按钮的最左侧，并采用更高视觉优先级。
- `宝物`、`集火`、`已吞噬` 作为功能按钮，统一放在构筑按钮右侧，避免和核心成长入口抢注意力。
- `技能` 按钮固定绑定快捷键 `G`，用于主动消耗 `1 个技能点` 打开攻击技能强化面板。
- `羁绊抽卡` 按钮固定放在 `技能` 按钮旁边，作为与攻击技能强化并列的第二个核心构筑入口。
- `羁绊抽卡` 按钮固定绑定快捷键 `F`。
- `宝物` 按钮用于打开 `背包面板` 的 `宝物` 页签；若当前存在未结算的宝物候选，则优先回到这组待选宝物候选。
- `集火` 按钮用于进入 `选择集火目标` 状态；若当前已经存在集火目标，则再次点击会直接清除当前集火目标。
- 按钮文案首版统一采用 `名称 + 快捷键角标 / 资源角标` 的形式：
  `技能 G`
  `羁绊 F`
  `宝物`
  `集火`
  `已吞噬`
- `技能` 按钮右上角固定显示 `G` 角标与当前技能点数量徽标。
- `羁绊抽卡` 按钮右上角固定显示 `F` 角标，并在按钮底部或右下角显示 `100 木材` 消耗标识。
- `宝物`、`集火`、`已吞噬` 默认不显示快捷键角标，首版先保留纯点击入口。
- `羁绊抽卡` 按钮需要直接显示本次抽卡固定消耗 `100 木材`，避免玩家额外打开说明确认成本。
- `技能` 按钮需要直接显示当前可用技能点数量，便于玩家判断 `G` 是否可用。
- `宝物` 按钮需要显示当前 `已持有宝物数 / 3`，例如 `2/3`，帮助玩家快速判断是否会触发宝物替换。
- `技能` 与 `羁绊抽卡` 按钮使用同一套主按钮样式；`宝物`、`集火`、`已吞噬` 使用统一的次级按钮样式。
- 主按钮样式需要明显大于次级按钮，至少在 `宽度`、`描边亮度` 或 `资源信息密度` 三者之一上拉开层级。
- 当当前技能点大于等于 `1` 时，`技能` 按钮恢复可点击状态；当技能点小于 `1` 时，按钮灰置并提示 `技能点不足`。
- 当当前木材小于 `100` 时，`羁绊抽卡` 按钮显示为灰置不可点击状态，并提示 `木材不足`。
- 当当前木材大于等于 `100` 时，`羁绊抽卡` 按钮恢复可点击状态。
- 当当前技能点大于等于 `1` 且存在尚未分配的强化机会时，`技能` 按钮可以出现轻度高亮，提示玩家可以按 `G` 继续构筑。
- 当玩家已经拥有某个可直接补齐的羁绊路线，且当前木材足够抽卡时，`羁绊抽卡` 按钮可以出现轻度高亮，提示玩家当前适合继续追羁绊。
- 当玩家已持有 `7 张未被吞噬的羁绊卡` 且当前木材足够抽卡时，`羁绊抽卡` 按钮需要附带 `将触发替换` 提示，避免玩家误以为还能直接新增卡位。
- 当当前存在 `未完成选择` 的宝物候选时，`宝物` 按钮需要出现高亮并显示 `继续选择` 提示。
- 当当前已经锁定集火目标时，`集火` 按钮需要持续高亮，并显示当前目标的 `名称` 或 `Boss / 精英` 头像缩略信息。
- 当玩家点击 `集火` 按钮后但还未选中具体敌人时，按钮进入 `等待选择目标` 状态，并在鼠标指针与敌方血条上显示集火选择提示。
- 当当前不存在可被集火的敌方单位时，`集火` 按钮仍可点击，但进入选择状态后需要给出 `当前无可选目标` 的弱提示。
- 成长武器需要显示当前 `装备等级`、`品质` 与核心词缀预览。
- `已吞噬` 固定定义为 `吞噬界面`，用于展示本局已经获得的 `烙印`、魔法效果与被替换吞噬的羁绊卡记录。
- `攻击技能` 本身不会被吞噬。
- `烙印` 不占用攻击技能格子，且不可成长。
- `吞噬界面` 首版固定拆为 `烙印`、`生效中羁绊`、`已吞噬记录` 三个页签，避免把当前生效信息与历史记录混在同一页里。
- `吞噬界面` 内的烙印列表固定按获得顺序展示，并显示 `品质`、`烙印名称`、`来源等级节点`、`效果说明`。
- `吞噬界面` 的 `生效中羁绊` 页签固定展示当前 `7 张未被吞噬羁绊卡`、对应 `羁绊名称`、`基础效果`、`当前进度` 与 `是否已激活`。
- `吞噬界面` 中的羁绊卡需要区分 `生效中` 与 `已吞噬` 两种状态，并显示 `羁绊名称`、`基础效果`、`当前是否计入进度`。
- `吞噬界面` 的 `已吞噬记录` 页签固定按时间倒序展示被替换掉的羁绊卡，并补充显示 `吞噬时间点 / 替换来源候选 / 是否导致羁绊失效`，便于复盘替换决策。
- 若本次获得了新烙印，`已吞噬` 按钮需要出现高亮提示，直到玩家打开界面查看为止。

### 11.10 右下战斗功能开关
右下角固定显示战斗功能开关组：
- `屏蔽特效`
- `屏蔽喊字`
- `屏蔽音效`
- `禁用控制`
- 以上 4 项都属于正式局内 UI 功能，不是调试选项。
- 所有开关状态都应即时生效，并在本地保存玩家偏好设置。

### 11.11 聊天、日志与提示
- 左下角保留战斗日志 / 系统通知输出区。
- 战斗日志至少显示系统提示、奖励提示、重要战斗提示。
- 获得高稀有掉落时要有显著视觉反馈。
- 当玩家设置或清除 `集火目标` 时，战斗日志需要输出简短提示，例如 `已集火：Boss`、`已清除集火目标`。
- 获得高价值 `金币 / 木材 / 精英或 Boss 奖励` 时，战斗日志需要输出简短奖励提示，例如 `获得木材 +40`、`获得金币 +120`。
- 小额常规经验与金币收益可以只做数字飘字、资源栏跳动与经验条增长，不必每次都写入战斗日志，避免刷屏。

资源结算表现规则：
- `经验`、`金币`、`木材` 的直接入账需要有明显可区分的颜色、图标或数字动画，避免玩家无法一眼判断刚刚获得了什么。
- 普通击杀奖励默认采用轻量反馈，不长时间停留屏幕中央。
- 精英、Boss、挑战怪物带来的高价值奖励应附带更明显的描边、高亮颜色或短暂停留提示。
- 当短时间内连续击杀大量怪物时，允许系统把小额奖励做合并显示，但顶部资源栏与经验条的最终结算值必须准确同步。
- Boss 的危险技能需要明显前摇与地面提示，但 Boss 来临本身不依赖独立预警弹窗。
- build 成型时需要通过数字、特效、标签点亮给玩家正反馈。

### 11.12 弹窗与结算
- 波次切换时可以显示轻量阶段提示，但不能打断战斗、清场或切出独立过场。
- Boss 来临默认通过顶部 `Boss 来临倒计时 / Boss 信息` 区域持续提示，不强制额外弹出独立预警弹窗。
- 胜利后弹出标准模式结算面板，并提供进入挑战模式或返回结算页的入口。
- 失败后弹出失败结算面板，显示本局核心数据与获得的局外奖励。
- 挑战模式结算面板必须额外显示本次挑战词条、完成情况与写入局外存档的奖励。
