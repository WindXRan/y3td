## 3. 系统定位与设计目标

### 3.1 定位

羁绊系统是战斗内第二条构筑轴，目标是让玩家在“伤害效率、节奏控制、召唤占场、属性成长”之间持续做取舍。

### 3.2 设计目标

- 差异化：每个羁绊有明确战斗辨识度，不做同构换皮。
- 可读性：视觉表现必须与真实结算路径一致（先投递，后命中结算）。
- 可维护：参数集中在 runtime rule 表，便于调优。
- 可回归：每次改动可按标准清单验证，不依赖口头描述。

## 5. 核心运行机制

### 5.1 统一命名与别名归一

运行时会先做别名归一，避免配置名与展示名不一致导致失效。

- 寒冰法师 -> 冰霜法师
- 冰法 -> 冰霜法师
- 电法 -> 雷电法王
- 雷法 -> 雷电法王

### 5.2 统一结算链路

主动类羁绊统一按以下顺序：

1. 触发条件成立（概率/计数/状态窗口）
2. 投射物投递（`perform_visual_delivery`）
3. 到达回调
4. 伤害/召唤/状态二段结算

禁止“视觉命中与逻辑命中分离”。

### 5.3 持续类视觉规则

持续型羁绊优先使用目标点生成投射物（`delivery_mode='spawn_on_target'`），减少长距离发射造成的误读。

当前已按该模式落地：

- 冰霜法师 / 寒冰法师
- 剑宗
- 游侠

## 6. 羁绊效果规格（实机）

### 6.1 普攻触发型

- 枪炮师：概率触发线性光波，可多波，主打直线穿透压制。
- 神射手：主弹命中后分裂追射，多目标补伤。
- 游侠：触发后进入毒雨持续区域伤害模型。
- 狂战士：命中即主伤，延迟血爆形成二段爆发。
- 剑魂：普攻计数达阈值后触发，含回斩线性补刀。
- 剑宗：触发剑阵持续 Tick，并带脉冲/轮转视觉。
- 龙骑士：按暴击阈值触发火龙模板。
- 战斗法师：主爆发 + 外扩脉冲，强调范围清场。
- 魔剑士：仅入魔窗口有效，命中后暗能扩散。

### 6.2 周期触发型

- 火法师：周期火球，命中后主伤+溅射。
- 冰霜法师：周期暴风雪，多段法术 Tick + 冰环氛围。
- 雷电法王：高频多目标点名，逐目标到达后结算。
- 猎人：周期召唤魔鹿，有目标时先视觉投递。
- 骷髅法师：周期召唤骷髅，有目标时先视觉投递。

### 6.3 纯属性型

- 刀锋战士：静态属性增益，不触发主动演出。
- 全能骑士：静态综合增益，不触发主动演出。

## 8. 视觉与投射物规范

### 8.1 统一规范

- 主动演出必须走投射物链路。
- 命中伤害必须在到达回调执行。
- 大范围持续伤害采用“常驻主特效 + 低频脉冲”而非每 Tick 全量重播。

### 8.2 冰霜法师专项规范

- 使用可用投射物资源键（当前为 `201392033`）。
- 模式：`spawn_on_target + line`
- 目的：体现“目标点直接生成”，而非英雄发射长弹道。

## 11. 已知风险与治理

- 风险1：物编删除后 `projectile_key` 失效。  
  治理：每次删物编后做投射物资源存在性校验。

- 风险2：运行时逻辑演进快，文档过期。  
  治理：提交涉及羁绊逻辑时，强制同步本文版本记录。

- 风险3：持续型视觉重叠造成读屏负担。  
  治理：维持“主特效常驻 + 低频脉冲”的策略，不做高频全重播。

## 12. 变更流程（强制）

1. 改规则先改 `bond_effect_runtime_rules.lua`
2. 改演出先改 `bond_visual_editor_ids.lua` 与 `bond_modifier_effects.lua`
3. 改行为再改 `bond_modifier_core_effects.lua`
4. 本文同步更新“参数口径/风险/版本记录”
