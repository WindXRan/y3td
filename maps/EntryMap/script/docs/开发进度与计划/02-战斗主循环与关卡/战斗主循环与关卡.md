# 战斗主循环与关卡

## 文档职责

说明当前战斗主循环、波次、Boss、挑战事件和章节内容差异的完成度。

## Source Of Truth

- `maps/EntryMap/script/entry_runtime.lua`
- `maps/EntryMap/script/entry_runtime_battlefield.lua`
- `maps/EntryMap/script/entry_config.lua`
- `maps/EntryMap/script/entry_objects/waves`
- `maps/EntryMap/script/entry_objects/challenges`
- `maps/EntryMap/script/entry_objects/stages`
- `maps/EntryMap/script/entry_objects/stage_modes`

## 已实现

- 单英雄进入战斗后会创建英雄、初始化资源与成长状态，并启动固定 `5 波` 主线。
- 每波主怪按 `spawn_segments` 刷新，`240 秒` 刷出 Boss，Boss 死亡后切到下一波；击败第 `5 波 Boss` 后胜利，英雄死亡后失败。
- 主怪、Boss、挑战怪可以并场存在；`entry_runtime_battlefield.lua` 已负责主线刷怪、Boss 创建、挑战刷新、单位回收和胜负结算。
- `Q/W/E/R` 四类挑战已接入真实 runtime，共用挑战次数、解锁条件、持续时间和奖励发放链路。
- 主线与挑战奖励已经能写回当前局内资源或高价值奖励队列，例如宝物挑战成功后会入队一次宝物轮次。

## 部分实现

- `1-2`、`1-3` 已有章节入口，但当前战斗内容仍复用 `1-1`，没有章节专属敌人、Boss 或波次差异。
- `entry_config.lua` 当前大量 `unit_id` 仍依赖临时替身物编；`temp_unit_labels` 也明确说明当前主怪 / Boss / 挑战怪并非正式资源。
- `entry_objects/waves/*.lua` 已预留 `boss_timeline_profile_id` 和 `boss_low_hp_profile_id`，但当前 runtime 只消费刷怪、奖励和 Boss 基本生成信息，没有真正的 Boss 时间轴与低血机制。
- `standard` / `challenge` 已有独立模式条目和局外入口，但当前战斗侧并没有真正的挑战词条包、规则覆写或模式差异闭环。

## 未实现

- Boss 时间轴、低血强化、固定技能节奏模板、高危技能提示等正式 Boss 机制。
- `1-2`、`1-3` 的章节专属主怪、Boss、奖励差异和地图 / 布局差异。
- 挑战模式开局词条选择、模式奖励写回和更完整的失败条件覆写。
- 正式怪物 / Boss 资源替换与更完整的精英 / Boss 内容池。

## 后续开发建议

1. 优先让 runtime 真正消费波次对象中的 Boss 预留字段，而不是继续只在文档里保留 `timeline_profile_id`。
2. 替换正式怪物资源时，不要把“临时物编替换”与“Boss 机制接入”完全拆开；两者最好按波次一起推进，避免文档与体验再次脱节。
3. 章节扩展时继续复用现有 `stages` / `stage_modes` 数据驱动结构，但不要再把新章节标成已完成后继续复用 `1-1` 内容。
