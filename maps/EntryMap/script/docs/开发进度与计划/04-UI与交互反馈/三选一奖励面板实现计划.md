# 三选一奖励面板 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 `G 技能强化`、`F 羁绊抽卡`、`宝物候选` 接入统一的三选一奖励面板，支持隐藏、恢复和刷新，并下线旧 merge 中错误的 HUD 决策卡显示链路。

**Architecture:** 首版直接在 Lua 侧动态创建独立的 `choice_panel` 覆盖层，不依赖编辑器静态面板资产。业务状态仍由 `entry_runtime.lua`、`entry_runtime_attack_upgrades.lua`、`runtime_bonds.lua` 和宝物 runtime 持有；新面板只读取当前待选状态并负责展示与交互。图片绑定统一走 `ui/res.lua` 的语义资源槽和业务对象上的 `ui_icon`/fallback 图标。

**Tech Stack:** Lua、Y3 UI API、`ui.factory`、`ui.skin`、`ui.theme`
