# 顶部战斗HUD结构标注示例（AI快速版）

> 这是 [UI结构标注图模板](./16-UI结构标注图模板.md) 的轻量示例。目标不是做传统正式标注图，而是让 AI 和程序快速知道 `怎么搭结构`。

## 1. 一句话结构
- 顶部中间是一组 `中轴仪表盘`，由左侧关卡标签、中央波次徽章、右侧 Boss 胶囊和下方细时间轨组成；右上是 `浮动资源簇`，下方挂一张轻量玩家统计卡。

## 2. 草图

```text
┌──────────────────────── 顶部安全区 ────────────────────────┐
│                                                            │
│                 [关卡标签] [波次徽章] [Boss胶囊]           │
│                        ─── 时间轨 / timer ───              │
│                                              [ 资源簇 ]    │
│                                              [ 统计卡 ]    │
└────────────────────────────────────────────────────────────┘
```

## 6. 程序接法

```md
程序接法：
- 中轴主构图：GameAPI.set_ui_comp_pos_percent(...)
- 资源浮动簇：set_relative_parent_pos('顶部', 24) + set_relative_parent_pos('右侧', 28)
- 玩家统计卡：同资源簇，往下错开
- 主构图内部：set_ui_size + set_anchor + set_pos
- 所有九宫底板：set_ui_9_enable(true) + set_ui_9(...)
```

## 7. 美观优化点
- 中间改成 `中轴仪表盘`，视觉焦点集中，不再像一条普通状态栏。
- 波次徽章上浮，形成 `王冠位`，让玩家一眼先看到当前局势节奏。
- Boss 胶囊拉长并右置，形成不完全对称的构图，比纯平均分栏更有张力。
- 时间轨做细做长，作为第二阅读层，不和标题抢重量。
- 右上资源簇改成窄卡堆叠感，而不是笨重大面板，整体更轻。

## 8. 给 AI 的使用方式
- 先把这份结构清单喂给 AI，让它生成：
  `低保真结构图`
  `第一版资源拆分`
  `顶部 HUD 出图 prompt`
- 如果 AI 回来的是整张效果图，再让它反向补：
  `哪些底板可拉伸`
  `哪些文本必须独立`
  `Boss区和资源栏分别有哪些层`
