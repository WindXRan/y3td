# 底部角色操作区结构标注示例（MOBA端游习惯版）

> 这是 [UI结构标注图模板](./16-UI结构标注图模板.md) 的轻量示例。目标不是画传统施工图，而是让 AI 和程序快速理解：`底栏要像 MOBA 端游那样怎么排`。

## 1. 一句话结构
- 左边是英雄核心区，中间是技能热键区，右边是主功能和次功能按钮，最底部是一条细经验轨。

## 2. 草图

```text
┌──────────────────────────── 底部主条 ────────────────────────────┐
│ [头像/等级/属性] [武器格] [技能1][技能2][技能3][技能4] [技能G][羁绊F] │
│                                                      [宝物][集火][已吞] │
│ ───────────────────────── 经验轨 / EXP ───────────────────────── │
└──────────────────────────────────────────────────────────────────┘
```

## 6. 程序接法

```md
程序接法：
- 底部主条：bottom-center 吸附
- 英雄核心区、技能热键区、右侧功能簇都挂在同一主条容器内
- 技能热键区保持底栏几何中心，不跟右侧按钮数量变化漂移
- 成长武器格夹在英雄区和技能区之间
- G/F 主按钮用更大按钮模板
- 宝物/集火/已吞噬用更小次级按钮模板
- 经验轨贴底栏底边拉满
```

## 7. MOBA 习惯化优化点
- `头像在左，技能在中，功能在右`，这是 PC MOBA 用户最熟悉的底栏阅读结构。
- 技能热键区必须是底栏视觉中心，不被成长按钮挤偏。
- G/F 虽然是构筑入口，但在布局上应像“主动操作键”，而不是普通菜单按钮。
- 次级功能按钮统一缩到右下，形成明显的主次分层。
- 经验轨细贴底边，像传统 MOBA 的进度带，而不是单独大条框。

## 8. 给 AI 的使用方式
- 先把这份结构清单发给 AI，让它出 `低保真底栏草图`。
- 再让 AI 基于同一结构输出：
  `资源拆分建议`
  `底栏出图 prompt`
  `程序 spec 草案`
