# 代码与数据流向

## 模块职责

说明 `script`、配置、物编、UI 资产、触发器之间的数据边界，帮助 agent 判断“应该改哪一层”。

## Source Of Truth

- 配置层：`maps/EntryMap/script/entry_config.lua`
- 对象层：`maps/EntryMap/script/entry_objects`
- 运行时协调层：`maps/EntryMap/script/entry_runtime.lua`
- 奖励运行时层：`maps/EntryMap/script/runtime/rewards.lua`
- 运行时子模块层：`maps/EntryMap/script/runtime/*.lua`
- 羁绊层：`maps/EntryMap/script/runtime/bonds.lua`
- UI 资产层：`maps/EntryMap/ui`
- 物编层：`maps/EntryMap/unit`、`maps/EntryMap/ability`、`maps/EntryMap/projectile`

## 数据流总图

```text
docs/design
  -> 设计目标与未来模块

entry_objects/*
  -> 波次 / 挑战 / 攻击技能 / 羁绊 / 宝物 / 烙印 的静态对象

entry_config.lua
  -> 点位 / 区域 / 资源规则 / 挑战规则 / 对象汇总

entry_runtime.lua
  -> STATE / 轮次互斥 / 子系统装配

runtime/rewards.lua
  -> 宝物 / 烙印 / 奖励队列 / 奖励倍率 / 被动收入修正

runtime/battlefield.lua
  -> 主线波次 / 挑战 / 战场 / 结算

runtime/progression.lua
  -> 经验 / 等级 / 技能点

runtime/attack_skills.lua
  -> 普攻技能实例与施法结算

runtime/bonds.lua
  -> F抽卡 / 羁绊效果 / 奖励修正 / 动态触发

entry_runtime_hud.lua
  -> 运行时 HUD 与按钮状态

maps/EntryMap/ui + ui_res.lua
  -> HUD资产 / 挂载点 / 资源ID

maps/EntryMap/unit|ability|projectile
  -> 实际单位/技能/投射物物编资源
```

## 当前已实现闭环

- `entry_objects/*` 提供波次、挑战、攻击技能、羁绊、宝物、烙印等静态对象
- `entry_config.lua` 汇总点位、区域、资源规则与挑战规则
- `entry_runtime.lua` 负责总协调与子系统装配
- `runtime/rewards.lua` 负责宝物、烙印、奖励队列和奖励修正
- `runtime/bonds.lua` 在击杀、奖励、动态效果刷新中回流到主循环
- `entry_runtime_hud.lua` 读取 `STATE` 展示波次、Boss、资源、技能点、挑战状态与待领奖励
- `treasures`、羁绊配置、`marks`、`stages`、`stage_modes` 与全局基础数值已经迁到 `script/data_csv/`
- 进化本体、进化节点与进化抽取规则现已统一由 `script/data_csv/` 驱动
- `script/data/object_tables/*` 负责把 CSV 装配回 runtime 继续消费的 Lua table
- `entry_objects/*/init.lua` 与 `entry_data/*.lua` 保留为稳定入口
- `entry_objects.mark_nodes` 继续保留为稳定入口，但底层来源已切换到 CSV 装配层

## 当前缺口

- 成长武器、词缀节点和长期局外成长还没有进入上述闭环
- 局外选关、章节/模式解锁和最近结果存档已经接通，但还不是完整长期资源闭环
- 宝物、烙印、奖励队列已经进入 runtime，但仍缺正式中央面板和背包页

## Agent 修改建议与风险点

- 改规则先看 `entry_config.lua` 与 `entry_objects/*`
- 改行为先看 `entry_runtime.lua` 指向的子模块
- 改宝物、烙印、奖励队列优先看 `runtime/rewards.lua`
- 改羁绊必须检查 `runtime/bonds.lua` 是否也要同步
- 改物编 ID 时要核对真实资源目录，不能只改文档或 Lua 常量
