# 战斗事件提示条

## 功能目标
- 在战斗 HUD 顶部追加一组“最近战斗事件”提示条，用来承接当前只会闪过一次的局内提示文案。
- 让玩家在不打开聊天或控制台的情况下，也能回看最近几条关键事件。
- 该组件只服务于 `battle` 阶段，不参与局外界面。

## 当前实现定位
- 运行时模块：`maps/EntryMap/script/runtime/battle_event_feed.lua`
- HUD 接线：`maps/EntryMap/script/ui/runtime_hud_panel1_top.lua`
- 通用消息桥接：`maps/EntryMap/script/runtime/boot.lua`

## 显示规则
- 位置：挂在顶部 HUD 的 `tips` 文本区域下方，与 Boss/波次提示分离。
- 可见条数：默认同时显示最近 `4` 条。
- 保留历史：运行时最多保留 `12` 条，超过后丢弃更早记录。
- 刷新方式：每次 HUD 刷新时按当前 `runtime_elapsed` 重新计算可见条目。
- 过期规则：每条事件默认保留 `8` 秒；也允许事件源传入自定义持续时间。
- 排列顺序：从上到下按“旧 -> 新”显示，避免新事件插入后整组文本频繁跳位。

## 文案来源
- 当前第一阶段先桥接 `message(text)` 在 `battle` 阶段发出的文案。
- 这样已有的挑战开启、奖励获取、技能升级、宝物获得、羁绊解锁等提示，会自动同步进事件条。
- 后续如果要做精细化控制，再逐步把高频低价值消息改成显式调用 `push_battle_event()`。

## 样式规则
- `neutral`：普通白字，用于一般提示。
- `positive`：浅绿色，用于进度增长、次数恢复、状态变好。
- `reward`：金黄色，用于获得奖励、刷新次数、资源追加。
- `warning`：橙红色，用于挑战开始、敌人进攻、风险提示。
- `rare`：浅紫金色，用于高价值获得、稀有触发、关键构筑事件。

## 典型文案示例
- `发起经验挑战，梦魔虎开始进攻`
- `瘟疫散播效果触发，额外获取1份残骸`
- `技能免费刷新次数 +1`
- `获得1张金行技能卡，希文王1星效果触发，金行伤害+1%`

## 运行时接口
- `create_runtime()`
  - 创建提示条运行时状态，内含历史队列和显示上限。
- `push_event(runtime, text, options)`
  - 压入一条事件。
  - `options.style` 可选，默认 `neutral`。
  - `options.duration` 可选，默认 `8` 秒。
  - `options.now` 可选，未传时按 `0` 处理。
- `update(runtime, now)`
  - 按当前时间清理过期事件。
- `get_visible_entries(runtime, now, max_visible)`
  - 取当前仍在有效期内、用于 HUD 渲染的条目列表。

## 接入约束
- 不复用原 `tips` 文本节点，避免和 Boss 状态抢占同一块显示区域。
- 不改已有 `display_message` 行为，事件条只是 HUD 内的补充回看层。
- 不引入滚动动画，首版只做稳定堆叠显示，先保证可读性和可维护性。
