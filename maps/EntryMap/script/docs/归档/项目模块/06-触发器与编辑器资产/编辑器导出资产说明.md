# 编辑器导出资产说明

## 模块职责

说明哪些文件是编辑器导出或地图元数据，agent 需要知道它们存在，但不应把它们当成首要玩法修改点。

## Source Of Truth

- 地图元数据根目录：`maps/EntryMap`
- 业务主线：`maps/EntryMap/script`

## 当前常见资产类型

- 地图元配置：`logicres.json`、`userresource.json`、`desc.json`、`userdata.json`
- 语言与资源映射：`zhlanguage.json`、`userlanguage.json`、`cloudresdependence.json`
- 编辑器表与模板：`editor_table`、`objecteditortemplate.json`
- 地图本体元信息：`header.map`、`gameworld.json`、`terrain.json`

## 关键状态与数据流

- 这些文件主要为引擎和编辑器提供静态资源/元数据
- 当前玩法真状态仍在 `maps/EntryMap/script/entry_runtime.lua`
- agent 通常只在“确认资源存在性”时需要回读这里

## 当前已实现行为

- Lua 运行时依赖这些资产“存在且可被引擎识别”
- 但当前玩法逻辑不直接由这些导出文件表达

## 未实现或占位项

- 没有“设计稿配表 -> 编辑器资源 -> Lua 配置”自动同步链
- 没有针对这些导出资产的文档化校验清单

## Agent 修改建议与风险点

- 不要把编辑器导出文件当成设计文档
- 不要在没有必要时批量修改地图元数据
- 如果文档要引用资源存在性，尽量指向目录职责，而不是细抠导出 JSON 内部细节
