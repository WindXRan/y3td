# 热重载与保留入口

## 模块职责

说明哪些入口只是开发保留位、示例位或编辑器遗留位，避免 agent 误把它们当主逻辑。

## Source Of Truth

- 热重载示例：`maps/EntryMap/script/可重载的代码.lua`
- 项目级保留脚本：`global_script/global_main.lua`
- 项目级保留触发器：`global_trigger`
- 地图级触发器：`maps/EntryMap/global_trigger`

## 当前已实现行为

- `main.lua` 用 `include '可重载的代码'` 引入热重载示例文件
- `可重载的代码.lua` 当前只注册 `R` 键示例，不参与主循环、波次、结算
- 根目录 `global_script/global_main.lua` 仍是占位内容
- 根目录 `global_trigger` 只有空索引，没有玩法逻辑

## 关键状态与数据流

- 真正有用的“地图级 UI 触发器资产”在 `maps/EntryMap/global_trigger`
- 真正有用的“热更新后仍会生效的主逻辑”不在 `可重载的代码.lua`，而在 `entry_runtime.lua`
- `.rd`/`.rr` 更偏开发调试能力，不是玩法模块边界

## 未实现或占位项

- 没有正式的热重载模块分层方案
- 没有把主循环拆成一套专门的 `include` 结构

## Agent 修改建议与风险点

- 修改当前玩法时，不要把新逻辑塞进 `可重载的代码.lua`
- 除非明确要做项目级共享机制，否则不要优先碰根目录 `global_script` 和根目录 `global_trigger`
- 看到 `R` 键逻辑时先确认是热重载示例还是业务热键，当前它不是玩法输入主线
