# 攻击技能系统

## 模块职责

说明英雄自动战斗、普攻技能、攻击技能槽位和施放更新由谁负责。

## Source Of Truth

- 攻击技能定义：`maps/EntryMap/script/entry_objects/attack_skills`
- 攻击技能运行时：`maps/EntryMap/script/runtime/attack_skills.lua`
- 技能强化设计参考：`maps/EntryMap/script/docs/归档/design/初步设计/11-攻击技能与羁绊配表与实现映射.md`

## 关键状态与数据流

- 技能定义：`ATTACK_SKILL_DEFS`
- 表现定义：`ATTACK_SKILL_VFX`
- 技能运行时：`STATE.attack_skill_state`
- 普攻技能同步：`setup_basic_attack_ability()`、`sync_basic_attack_ability()`

## 当前已实现行为

- 英雄会自动索敌并自动执行普攻
- 已内置 `basic_attack`、`arcane_arrow`、`flame_arrow`、`frost_arrow`、`thunder`
- 当前以 `basic_attack` 为核心，其他攻击技能位按现行配置逐步开放
- 技能有槽位、冷却、目标选择、VFX、投射物、控制和命中后处理
- 普攻被单独绑定到引擎技能按钮与描述同步逻辑

## 当前实现特点

- 技能静态定义和表现资源已经拆进 `entry_objects/attack_skills/*.lua`
- 技能实例、自动施法与伤害处理在 `runtime/attack_skills.lua`
- 攻击技能成长由运行时模板与构筑系统共同驱动
- 文档中的更完整卡牌 UI 仍是未来方向，不是当前实现现实

## 未实现或占位项

- 没有正式中央决策面板承载技能候选 UI
- 没有多英雄差异化技能树

## Agent 修改建议与风险点

- 改技能数值或描述时先改 `entry_objects/attack_skills/*.lua`
- 改施法、索敌、投射物或控制时改 `runtime/attack_skills.lua`
- 改普攻逻辑时同时检查 `basic_attack` 与引擎技能描述同步
