# 羁绊系统

## 模块职责

说明 `F` 抽取、链式节点解锁、动态效果刷新和奖励修正由谁实现。

## Source Of Truth

- 主实现入口：`maps/EntryMap/script/runtime/bonds.lua`
- 链式羁绊实现：`maps/EntryMap/script/runtime/bonds_chain.lua`
- 羁绊节点定义：`maps/EntryMap/script/runtime/bond_nodes.lua`
- 接入层：`maps/EntryMap/script/entry_runtime.lua`
- 设计参考：`maps/EntryMap/script/docs/归档/design/初步设计/11-攻击技能与羁绊配表与实现映射.md`

## 关键状态与数据流

- 羁绊节点定义：`BondNodes.list / BondNodes.by_id`
- 当前局羁绊状态：`STATE.bond_runtime`
- 接入入口：
  - `BondSystem.create_runtime()`
  - `BondSystem.try_draw()`
  - `BondSystem.refresh_choice()`
  - `BondSystem.apply_choice()`
  - `BondSystem.refresh_effects()`
  - `BondSystem.update_effects()`
  - `BondSystem.build_reward_with_bonus()`

## 当前已实现行为

- `F` 系统已经真实接入 `runtime/bonds.lua`
- 抽卡消耗 `100` 木材
- 抽卡会生成候选，选择后写入卡槽、进度、激活效果
- 当前最多保留 `7` 张生效羁绊卡，满位后会按推荐自动吞噬 1 张旧卡
- 凑齐并激活整套羁绊后，会把整套卡从羁绊栏移出、释放格子，并把该套羁绊写入“已吞噬列表”
- 已吞噬的整套羁绊会继续保留单卡属性、成套效果和动态 runtime 加成，战斗内可按 `I` 查看
- 羁绊既有静态属性加成，也有动态 runtime bonus
- 击杀奖励、Boss/精英增伤、低血增伤、成长型击杀收益等都已接入
- 已接入统一三选一奖励面板，支持点击 / 数字键选择、暂时隐藏和恢复
- `F` 没有免费刷新；刷新按 `40 / 80 / 100 / 100...` 消耗木材，不会退还原始抽卡的 `100` 木材

## 当前实现特点

- 静态定义已经收敛到 `runtime/bond_nodes.lua`
- `runtime/bonds_chain.lua` 已经覆盖抽取、解锁、效果刷新、奖励修正、动态触发
- `entry_runtime.lua` 主要负责把 `F` 与其他轮次系统做互斥协调
- 文档必须把羁绊写成“已实现系统”，不能再写成占位

## 部分实现与未实现

- 当前权重与替换策略仍写在 runtime 代码里，不是编辑器配表
- 目前已与宝物、烙印、`G` 形成统一的轮次互斥；其中烙印仍未接入新面板

## Agent 修改建议与风险点

- 改羁绊定义时优先改 `runtime/bond_nodes.lua`
- 改抽取、解锁、动态效果、奖励联动时优先改 `runtime/bonds_chain.lua`
- 改 `F` 与其他轮次的并存规则时再回到 `entry_runtime.lua`
- 新增羁绊效果时必须同时考虑静态属性、动态效果、击杀/奖励回调三个落点
