# 宝物总表

> 当前事实：战斗内宝物唯一数据源为 `maps/EntryMap/script/data_csv/treasures.csv`，运行时通过 `data/object_tables/treasure_catalog.lua` 与 `data/object_tables/treasure_catalog_compat.lua` 读取。旧版 `entry_objects/treasures/*.lua` 方案已废弃，不再作为运行时事实来源。

## 1. 文档职责

- 记录当前战斗内宝物图鉴条目。
- 说明当前图鉴字段与运行时兼容层的边界。
- 区分 `当前事实` 与 `后续待补设计`，避免把历史方案继续当成现行实现。

## 2. 作用范围

- 包含：
  - `treasures.csv` 中已经入库的战斗内宝物
  - 当前 CSV 已存在的字段：`id / name / category / rarity / is_set_item / set_id / summary / notes`
- 不包含：
  - 已删除的旧版 runtime 宝物单文件定义
  - 尚未重新立项的旧版 `general / route_amp / tactical_temp` 分类方案
  - 局外长期成长内容

## 3. 当前字段说明

| 字段 | 含义 | 当前状态 |
| --- | --- | --- |
| `id` | 宝物唯一 ID | 当前事实，运行时使用 |
| `name` | 宝物名称 | 当前事实，玩家可见 |
| `category` | 图鉴分类 | 当前事实，策划分类字段 |
| `rarity` | 稀有度：`normal / rare / epic` | 当前事实，运行时会映射到 `common / rare / epic` |
| `is_set_item` | 是否为套装件 | 当前事实 |
| `set_id` | 套装 ID | 套装件时生效 |
| `summary` | 当前摘要描述 | 当前事实，兼容层据此生成部分标签 |
| `notes` | 备注 | 当前事实，主要用于策划说明 |

## 5. 当前实现结论

- 当前战斗内宝物总数为 `22` 件。
- 当前实现已经不再使用旧版单文件宝物定义，也不再维护第二套运行时宝物池。
- 当前图鉴分类以 `category` 为准；旧文档里的 `general / route_amp / tactical_temp` 只可作为历史方案参考，不能再视为现行规则。
- 当前兼容层会把图鉴数据转换为运行时可读结构，但不会反向定义新的策划事实源。

## 6. 后续待补项

- 若后续需要重新建立 `投放分层`，应直接基于 `treasures.csv` 新增字段或新增附表，不再恢复旧版单文件宝物方案。
- 若后续需要恢复“技能适配 / 构筑标签 / 生效时机标签”，应在 CSV 或附表中补正式字段，不应继续依赖旧文档口头维护。
