# Runtime HUD HTML 初版设计

## 背景

本次目标是为 `EntryMap` 的运行时 HUD 制作一个 HTML 初版，用于快速确认整体分区、比例、视觉气质与后续 UI 拆分方向。用户明确要求按提供截图进行高拟真临摹，设计基准分辨率为 `1920x1080`。

项目现状显示：

- 现有运行时 HUD 已有 Lua 版本骨架，核心文件为 `maps/EntryMap/script/ui/runtime_hud.lua`
- 当前 UI 规范指出运行时 UI 由 Lua 动态创建或控制，UI 不重复保存业务状态
- 目前缺口集中在更完整的正式 HUD、决策面板和其他交互面板

本次 HTML 初版不直接接入 Lua 逻辑，仅承担视觉确认与结构预演职责。

## 目标

- 产出一个可直接打开预览的 HTML 初版页面
- 在 `1920x1080` 画布上尽量还原参考截图的 HUD 层级与空间关系
- 用分区化结构搭好后续可拆分的 UI 骨架
- 提供少量演示性交互，增强预览观感，但不引入复杂业务逻辑

## 非目标

- 不在本次实现中接入真实运行时数据
- 不在本次实现中还原完整游戏场景渲染
- 不在本次实现中输出 Y3 JSON 或 Lua UI 创建代码
- 不追求像素级逐点复刻原图素材

## 设计方案

采用高拟真静态临摹方案，并带有轻量演示交互。

整体方案：

- 页面以 `1920x1080` 为唯一设计基准
- 中央为战场背景层，HUD 通过绝对定位覆盖其上
- 所有 HUD 大区按截图布局组织，但代码内部使用独立区块命名，便于后续迁移
- 视觉上保留旧式网游 HUD 风格：深色半透明面板、金属描边、青蓝冷光、黄绿强化数字

## 页面结构

HTML 页面拆为以下区块：

- `hud-root`
  - 顶部栏：退出按钮、设置按钮、倒计时、波次、Boss 面板、资源条
  - 左侧提示列：快捷键提示、作者/玩家铭牌、聊天提示
  - 右侧任务列：玩家列表按钮、快捷键入口、主线任务卡、自动任务与系统勾选项
  - 底部左区：挑战入口、地图卡、抽奖/存档/设置/百科按钮
  - 底部中区：英雄头像、属性卡、经验条、状态栏
  - 底部右区：物品格、技能格、快捷键角标、引导框

## 视觉规范

### 画布与缩放

- 默认画布尺寸为 `1920x1080`
- 使用一个固定比例的主容器承载布局
- 在更小窗口中等比缩放，避免局部错位

### 色彩

- 主底色：深灰黑与棕黑混合
- 强调色：青蓝、荧绿、金黄、警示红
- 描边与分隔：偏冷灰金属色
- 文本层级：
  - 核心标题用高亮白
  - 功能标签用偏黄或偏蓝
  - 增益数字用亮绿
  - 风险/警示按钮用红色

### 质感

- 大面板使用半透明渐变底
- 外框使用细描边和内阴影，模拟金属边框
- 局部高亮使用柔和光晕，不使用过强泛光
- 按钮和技能格采用旧式 RPG HUD 的厚重矩形与压边感

## 交互范围

允许的轻交互：

- 按钮 hover 高亮
- 技能格 hover 描边发光
- 任务卡与引导框轻微脉冲
- 资源数字与增益数字使用强调色区分

不包含：

- 拖拽物品
- 复杂弹窗
- 实时战斗逻辑
- 完整键位监听与状态切换

## 文案与占位数据

页面使用与参考图接近的中文占位文案，例如：

- `退出游戏`
- `00:38`
- `第1/5波`
- `BOSS来临`
- `主线1-1`
- `按G选择技能(0/1)`

资源、属性、技能、任务奖励等均使用演示态假数据，但命名风格与当前项目一致。

## 实现边界

本次建议输出：

- 一个独立 HTML 文件
- 内联或同目录 CSS
- 少量原生 JavaScript，用于演示微交互

不强制要求：

- 构建工具
- 前端框架
- 资源切图

## 与现有项目的衔接

该 HTML 初版用于对齐以下后续工作：

- `runtime_hud.lua` 的视觉重组
- HUD 区块的坐标与层级拆分
- 任务栏、挑战栏、角色区、背包区等模块的边界确认

后续若要迁移到 Y3 UI，应优先按本设计中的区块划分进行映射，而不是直接照搬 DOM 结构。

## 测试与验收

完成后至少检查以下项目：

- 在 `1920x1080` 下整体布局与参考图视觉关系基本一致
- 顶部、左侧、右侧、底部四大区块均已落位
- 关键按钮、任务卡、属性区、技能格具备清晰分层
- 页面可直接在浏览器打开
- 在较小窗口下仍保持整体比例，不出现明显错位

## 风险与取舍

- 由于没有使用原始游戏素材，首版主要依赖 CSS 质感模拟，因此会更偏“高拟真”而非“像素级复刻”
- 若后续需要进一步贴近游戏内效果，应再引入定制图标、边框图和字体资源
- 若后续直接转 Y3 UI，需要针对引擎控件能力再次简化阴影、渐变和装饰层
